Yes you can render old versions of OSM data (see for example https://heigit.org/big-spatial-data-analytics-en/ohsome/ but there are other tools that support this), however old OSM data is not historic data, just old OSM data.

If you want to access historic data https://openhistoricalmap.org/ is a possible resource.

OSM objects have versions and in principle you could determine which version was "current" at a particular time. However this has loads of issues starting with that the editing API is not intended for such use, ending with objects not necessarily maintaining their original object type and id.

However you can restrict an Overpass API query to a date, see

https://wiki.openstreetmap.org/wiki/Overpass_API/Overpass_QL#Date

and

https://wiki.openstreetmap.org/wiki/Overpass_API/Overpass_API_by_Example#OSM_data_at_a_certain_date

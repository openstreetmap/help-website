The "data" dissector in Wireshark is a fallback that just displays the hex values when no other dissector can be found that will dissect the data correctly.

There are a few reasons why the "data" isn't being dissected:

  1. The data is from a protocol that Wireshark doesn't currently support. Someone will need to write a dissector for it.
  2. The data is from a protocol that is supported by Wireshark, but it's on a non-standard port or transport that isn't supported or configured by your copy of Wireshark. If you know what the data is meant to be, then you might try using "Decode As ..." to dissect the data.
  3. Thee data is from a protocol that is supported by Wireshark, but the heuristic detection of it fails for some reason. Again "Decode As ..." might help.
  4. Some other reason.

Do you know what the data is meant to represent? Can you show the packet tree for one of the frames with the hex data, rather than the hex data itself (Hint use File -> Export Packet Dissections -> As Plain Text, set the Packet Range to the Selected Packet and set the Packet Details droplist to "All Expanded") ?

Enhancement Request: Swap A/B On Conversation List

0

Tried to find the proper forum for enhancement requests, and settled on this one. Forgive me if I guessed poorly.

I often use the Conversation List to track conversations between a database server and clients. However, sometimes the server shows up as "PortA" and sometimes as "PortB", which makes it hard to sort the data by the number of database bytes requested. Now, my guess is that WS picks up the FIRST packet in the conversation and makes the source A and target B. Would be really beneficial to be able to right-click on a line item and swap A for B, to then make sorting the data a bit more meaningful. Swapping wouldn't need to be saved -- could be stored only as long as the data is available in the dialog box.

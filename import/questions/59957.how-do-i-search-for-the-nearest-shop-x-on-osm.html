Whenever I challenge friends to find the closest gelato place where to go, or similar, the winner is usually the one who uses OsmAnd, whose search feature seems to work quite well.

Simply go to the point where to start the search; click the magnifying glass; click category; select the category.

I found that the easiest option by far is to search DuckDuckGo for X near Y: a "Places" tab will appear and if you click one of the options a map shows up.

It's proprietary and it mostly seems to use Yelp data, which cannot be edited freely, but at least it doesn't track you and it shows an OpenStreetMap map (via Mapbox), which is an encouragement to improve it. Maybe as the OSM coverage of shops and other points of interests increase DuckDuckGo will be able to ditch Yelp and others will be able to achieve something similar?

All of the above search are good, but please do remember the whole osm.org has limited capacity as running on voluntary donation. So you'll find good answer to your queries but it's not the osm.org core business. - I'm usually doing search on OsmAnd app (my best off line map app): go to the point where start the search; magnifying glass; category; select the category.

  • You can try in the osm.org box search too (e.g.: ristorante, Sestri Levante)

  • When I need search all POI in an area I use overpass-turbo.eu (not the best user friendly interface). For a restaurant search type ameniy=restaurant in "your place" (e.g.: amenity=restaurant in "Wetter (Ruhr)")

  • DuckDuckGo has OpenStreetMap defauul place and POI search

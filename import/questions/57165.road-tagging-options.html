Can you do this? : yes, there is nothing in the OSM data model which prevents it. Should you do this? : in my view NO.

The main OSM data set is a live database used in various ways by millions of people. Transient modifications may live on on apps and copies of the OSM data far longer than you envisage. Transient modifications may inconvenience people who are used to OSM data in the area being reliable.

Modifying it for 'research' purposes is a dubious practice, which should be avoided. If you are based in a research university it is probably that any such project should be reviewed by an ethics committee. In any case the OSM Data Working Group may not accept that your edits are legitimate contributions to OSM.

As others have said:

  • It is far better to work on a copy of the OSM data.
  • Ensure that you avoid uploading any changes you make to the copy.
  • If you do make changes to OSM restrict them to obvious personal use tags only.
  • Document, in detail, what you plan on the OSM wiki.
  • Discuss any plans for changes to OSM with the community: people local to the area you want to investigate, the country mailing list & forum.
  • Do not proceed unless you have positive buy-in to your plans. An email to a list saying "I'm proceeding in X days unless I hear to the contrary" is rarely acceptable.
  • Consider making a donation to the Foundation as you are using their servers for your research.

You don't mention how you'd like to consume the modified data. (What's a "specific tag is toggled"?)

In any case, instead of editing the public map, perhaps you could download the data for your city, modify it as you see fit, and then either distribute it directly to your data consumers, or set up your own server running with that data, etc; the idea being not to clutter a shared resource (the public map and edit history) with private experiments.

From a technical standpoint you can just add whatever tag you want to the streets. You could use something like bgdoo:name which wouldn't get shown by any public styles but could still be visualized using Overpass-Turbo or similar.

From a community standpoint people might not like it (by that I mean they won't like it) if you do that to a huge area of the map, so you'd want to have it all worked out before you started adding such a tag. What it amounts to is that temporary changes and changes that aren't going to be useful to other people are both unpopular.

You cannot change the tiles (map) generated for the default OSM style with a toggle. You will have to obtain the tiles with the specific tag from elsewehere, or overlay the tiles from osm.org. Therefore it is useless to add your specific data to osm.org.

If setting up your own render server that understands the specific tags is too much work for this project, you might consider using umap where you can overlay OSM maps with your own data.

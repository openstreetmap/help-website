Looking at the output I see 2 things:

/usr/bin/perl ./make-version.pl . Version configuration file version.conf not found. Using defaults. svn: The path '.' appears to be part of a Subversion 1.7 or greater working copy. Please upgrade your Subversion client to use this working copy. svnversion.h unchanged.

and

/usr/bin/ld:.libs/libwsutil.ver:2: syntax error in VERSION script

I guess you somehow have an svn 1.7 working copy, but the svn client executable the build process is using is 1.6 or earlier so the version file can't be updated and when it's used in the build its syntax is incorrect.

Check your paths and the version of svn client (svn --version) that will be used by make.

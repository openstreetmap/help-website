Did you prerender any tiles? This would be recommended for lower zooms as the rendering of tiles on e.g. zoom 5 or 6 can take several minutes per metatile. If you zoom in on those levels without any prerendered tiles you will see 404s as Apache is not waiting that long. (But then, if you return several minutes later to that zoom level, they should be finished rendered and displayed.)

I would first check with

tail -f /var/log/syslog | grep renderd

if you do see any rendering activity of your stack when you access the map. If that is not the case, restart renderd and look for any errors/hints in syslog if renderd did start alright or not.

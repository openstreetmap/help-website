I do it this way as well, but it gets ugly results. There are two identically named nodes on map and in results of searches etc, and somethimes it looks like a duplicate. Also in some maps the names are not rendered at all, because there are so close to each other, that the rendered rather shows randomly one of the icons. I know that it is not good to map for renderer, but this is mapping AGAINS renderer, because it has no good way to understand that these are two identical amenities (except somehow guessing from proximity and identical names).

Unfortunately there is no 100% good way to do it. Probably the best is to add another node or way (or the building) beside where the hotel is. In essense treating them as 2 separate, but very close to each other, objects. You can add separate data for each one separately. They might have different websites, phone numbers, names etc.

How about using both tourism=hotel and amenity=restaurant on the same node? It avoids having a duplicated name, IMO it's the best for simple cases. If more specific values are different between the hotel and restaurant (like opening hours or phone number) then create two nodes.

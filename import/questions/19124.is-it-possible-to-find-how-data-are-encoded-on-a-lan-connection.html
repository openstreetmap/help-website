However, the data on the package described on Wireshark cannot be read in ASCII format for example. I have no idea how I can read them.

That means the protocol being used is

  • not a text based protocol

and

  • not a protocol that Wireshark knows how to dissect

so you might have to reverse-engineer the protocol by looking at the binary data, or try to find information on that protocols somewhere, for example by seeing whether the makers of the devices have documented it or have said what it is, or by doing a Web search.

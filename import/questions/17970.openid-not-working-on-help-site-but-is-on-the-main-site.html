Yes this is a problem. Even when regenerating the password (which becomes effective immediately for the OWM website, wiki, or editors, it is not always taken immediately on this Help site which rejects the password just reconfigured on the main web site.

For some unknown reasons, the Help forum rejects the connection, loosing the correct password that still works on the main site, and instead of logging in here, the Help site restarts the process of creation of a new account (but rejects the creation because the email is already registered or the user name is already taken : these usernames and emails are prefilled in the creation form).

The only way to enter is to wait for several days before entering this Help subsite.

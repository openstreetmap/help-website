If your plugin(s) have

#ifdef HAVE_CONFIG_H

# include "config.h"

#endif

You have to change that to

#include "config.h"

Where is your svn working copy connected to, trunk or a release branch (hint use svn info to find out) and what revision number have you updated to? Are all the changes in your working copy as expected?

If you're using the trunk, then occasional breakage does happen. Looking at the buildbot waterfall display will tell you what the last revision to successfully build was, (currently r45948). I've just locally built that version without a problem so it looks like the problem is at your end if you're at that same revision.

Have you changed anything since your last successful compile? Have you tried a make clean or make distclean?

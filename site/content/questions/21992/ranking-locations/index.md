<div class="answer-body">
<p>Quite everything is possible with OpenData and OpenSource software like OSM and most related tools and frameworks are.</p>
<p>But in my personal opinion it is not a good idea to add any ranking information on each or some OSM elements in the main OSM database.</p>
<p>You can try to build up your own local database where special OSM objects and ranking values are associated. Then you can publish a platform where users can do the ranking.</p>
<p>But be aware: the unique OSM element ID can change all time, for example when a POI is in the OSM data as a single node point ... maybe some day an OSM mapper is creating the building silhouette by a way element, and he deletes the (old) node.</p>
<p>You have to find a solution for this, and you are not trhe first one asking for data models like that.</p>

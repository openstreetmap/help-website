<div class="answer-body">
<p>Actually the codes to use are the standard codes from the "BCP 47" standard, which contains:</p>
<ul>
<li>a part of ISO 639-1 (excepting special codes not for languages and obsolete codes, or legacy codes for families),</li>
<li>a part of ISO 639-2 (same remark, but don't use them if there's a code in the previous list),</li>
<li>a part of ISO 639-3 (same remark, but don't use them if there's a code in the two previous lists)</li>
</ul>
<p>These codes (in lowercase only) may be followed by suffixes separated by dashes (-) for</p>
<ul>
<li>a (lowercased) script code from ISO 15924 (e.g. "latn")</li>
<li>a (lowercased) country or region code from ISO 3166-1 for the regional variant of the language</li>
<li>a (lowercased) variant code (e.g. "fonipa" for the IPA phonetics)</li>
</ul>
<p>Those codes are to be used to mark tags that contain translated values, such as</p>
<ul>
<li>"name=*" which can be derived into</li>
<li>"name:de=*" for the name in German.</li>
</ul>
<p>Do not confuse these codes with those used as tag suffixes or prefixes to mark that a tag interpretation is country-specific (in that case the ISO 3166-1 regions codes are all uppercase), independantly of the language, such as:</p>
<ul>
<li>"admin_type:FR=*" for an administrative type whose value is specific for France (giving additional status details not detailed in "admin_level" tags, or</li>
<li>"ref:FR:SIREN=*" for the normative references to organisations, collectivities, and professionals registered in France (a mandatory reference for all accounting and billing purpose).</li>
</ul>

<p>I have currently set both as Feature:Address for now so they're at least listed on the map, but I don't want to leave them like that for long if at all possible.</p>
<p>My question is this: did I miss something, or is there a bug?</p>

<div class="answer-body">
<p>Adding a dentist or doctor using iD both add <code>amenity=doctors</code> tag. I'm guessing that when you deselect and reselect it "dentist" is found alphabetically in the feature list before "doctor". So what you were doing was correct.</p>
<p><code>amenity=dentist</code> seems to be the tag that iD should be using for dentist, though it looks like some people use <code>healthcare=dentist</code> instead or as well as that tag. JOSM editor uses <code>amenity=dentist</code> in its presets.</p>
<p>Edit: it looks like this was <a href="https://github.com/openstreetmap/iD/pull/2121/files">fixed in the code</a> 12 days ago, so I suspect it hasn't yet been deployed as <a href="https://github.com/openstreetmap/iD/releases">latest release was 15 days ago</a>.</p>

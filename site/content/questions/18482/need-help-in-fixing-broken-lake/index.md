<div class="answer-body">
<p>It's normal that things <a href="https://help.openstreetmap.org/questions/178/how-often-does-the-main-mapnik-map-get-updated">take a little while to appear</a> on the main map at Openstreetmap.org, but what's going on here is a little more complicated.</p>
<p><a href="http://www.openstreetmap.org/edit?editor=potlatch2&amp;lat=20.04947&amp;lon=73.67815&amp;zoom=16">This view</a> of the northern end of the dam in the online editor shows the problem. There are actually two things defining the lake. There's <a href="http://www.openstreetmap.org/browse/relation/1924386/history">this "natural=water" relation</a>, which includes <a href="http://www.openstreetmap.org/browse/way/142180332">this way</a> which you can see at the northern end, and also <a href="http://www.openstreetmap.org/browse/way/196543098/history">this "waterway=dam" way</a> which (by mistake I presume) goes all the way around the lake. In addition, there's <a href="http://www.openstreetmap.org/browse/way/196538359">another "waterway=dam" way</a> that duplicates the first (you can see that clearly in the online editor by the highlighted red circles.</p>
<p>In order to fix it, the easiest place to start is probably by reverting changesets <a href="http://www.openstreetmap.org/browse/changeset/14284200">14284200</a> and <a href="http://www.openstreetmap.org/browse/changeset/14284165">14284165</a>, and then by examining problems with the previous "natural=water" relation. If you want to ask someone to do that, the easiest way if probably by <a href="http://wiki.openstreetmap.org/wiki/IRC">IRC</a>. Other <a href="http://wiki.openstreetmap.org/wiki/Contact">contact</a> methods probably won't be as real-time.</p>
<p>Finally, I noticed that on <a href="http://www.openstreetmap.org/browse/way/142180332/history">this way</a> you've added "source=google". Unfortunately, their data isn't one of the sources that can be used in OSM - have a look at the "Don't use copyrighted data" section of the <a href="http://wiki.openstreetmap.org/wiki/Beginners_Guide_1.1">beginner's guide</a> for details.</p>

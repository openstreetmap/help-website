<div class="answer-body">
<p>There are two localization levels:</p>
<ol>
<li>to have just UI strings translated, this is technical task, and possible with OSQA (see <a href="http://meta.osqa.net/questions/94/how-can-i-compilecreate-a-new-language-pack">http://meta.osqa.net/questions/94/how-can-i-compilecreate-a-new-language-pack</a>) for instance.</li>
<li>to have whole Q&amp;A content in German - either similar general questions like in English, or even German-specific questions. This is a bit more complex task technically, possibly requires another separate instance of the help system (help.osm.de?). But is it really good idea, it would generate similar questions duplicated ?</li>
</ol>

<p>I use <a href="http://shapado.com"></a><a href="http://shapado.com">shapado.com</a> for its nice language support. The German version is fully translated. But hasn't been released yet you can check out the translation progress on <a href="http://translatewiki.net/wiki/Translating:Shapado/stats">Translatewiki</a>, or use <a href="http://gitorious.org/shapado/shapado">the dev version</a> if you want to deploy it.</p>
<p>OSQA is free open source software too and supports multiple languages. I know Shapado is trivial to link to setup in their hosted service, and link to domain, so if OSQA doesn't fit the bill OSM may consider moving this facility to <a href="http://shapado.com">shapado.com</a>.</p>

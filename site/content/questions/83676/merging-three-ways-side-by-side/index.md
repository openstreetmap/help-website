<p>I've found a duplicated way <a href="https://www.openstreetmap.org/way/198706213">on this map</a>, different areas with a common border but same name:</p>
<ul>
<li><a href="https://www.openstreetmap.org/way/198706198">Way: Elder Park (198706198)</a></li>
<li><a href="https://www.openstreetmap.org/way/198706213">Way: Elder Park (198706213)</a></li>
</ul>
<p>Both appear to have been created by same user however there are no change set comments explaining why it's been done this way and they have not been active in 7 years so I can't contact them.</p>
<p>Should I merge them? My intention for merging is the park is showing as a duplicate in the search results however I've read in same cases these should be left alone so long as the ways are accurate in their parts because it could cause unforeseen problems.</p>
<p>If merging isn't allowed, what do I do with <a href="https://www.openstreetmap.org/way/200242320">the third segment of the park</a> (also created by the same user) which is currently unnamed. It is part of Elder Park because it contains the <a href="https://www.openstreetmap.org/way/71093853">Elder Park Rotunda</a> within it. Do I add a name to the way (thereby making three ways of the same "Elder Park"), or extend the second way into it?</p>

<div class="answer-body">
<p>This is the result of different opinions in the comunity. The creators of Potlatch have the opinion that a tag should not have several values and theat relations should be used instead. Potlatch are therefore displaying a warning when you include a semicolon in a tag.</p>
<p>The options you have are:</p>
<ol>
<li><a href="https://wiki.openstreetmap.org/wiki/Relations/Proposed/Destination_Signs">Destination Signs relation</a> : <a href="http://taginfo.openstreetmap.org/tags/type=destination_sign">3670</a> uses</li>
<li><em>exit_to=</em>* : <a href="http://taginfo.openstreetmap.org/keys/exit_to">16588</a> uses</li>
<li><a href="https://wiki.openstreetmap.org/wiki/Key:destination"><em>destination=</em>*</a> : <a href="http://taginfo.openstreetmap.org/keys/destination">11342</a> uses</li>
</ol>
<p>I hope that you find a way in the minefield of oppinions in tags that are not commonly used.</p>

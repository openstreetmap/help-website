<p>As <a href="https://help.openstreetmap.org/users/4426/insertuser">InsertUser</a> has said, if each side of a street has a different name, then "name:left" and "name:right" are used for that. An <a href="https://overpass-turbo.eu/s/1mX9">overpass search</a> finds quite a few locally to me, for example <a href="https://www.openstreetmap.org/way/7645100">this one</a>. There <a href="https://map.atownsend.org.uk/maps/map/map.html#zoom=20&amp;lat=53.4809468&amp;lon=-1.60519">are</a> renderers that support this usage.</p>

<p>If different names on each side, then name:forward="one name", name:backward="other name" where forward and backward are based on the direction of the way in OSM. At least this is how many of the side/direction dependent tagging works in OSM. I do not know if any renderer will properly process it though.</p>
<p>If the name simply changes along the way, then split the way and put the appropriate name on each.</p>

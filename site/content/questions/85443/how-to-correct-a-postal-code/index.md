<div class="answer-body">
<p>Firstly the location is B<strong>A</strong>sse-Bodeux and it is contained within the administrative district of <a href="https://www.openstreetmap.org/relation/2436185#map=12/50.3554/5.8811">Trois-Points</a> which has the postal_code of 4980 attached and thus encompassing all relevant sub-divisions. It appears that both Haute- and Basse-Bodeux have a postal code of 4983 and the current mapping of the postal code is incorrect.</p>
<p>In this situation i would contact the Belgian community to help resolve the missing postal code, as this is not merely about adding a new code, but adjusting the existing one in some way.</p>

<div class="answer-body">
<p>You can add a <code>postal_code</code> tag to an administrative boundary if the postal code is valid for the whole area. Otherwise, you can construct a <code>type=boundary</code> + <code>boundary=postal_code</code> object for the area of the postal code and attach the <code>postal_code</code> tag to that. Constructing boundaries is not recommended for beginners though as it can be a bit tricky - you're expected to re-use existing objects to make up the boundary rather than drawing one free-hand.</p>

<p>Hello everyone, I'm new here so excuse me for my potential naivety.<br />
I'm starting a project to display static devices collecting personal data in smart cities, I built a local prototype using OSM/Leaflet, but now I'd like to scale, and I was wondering if it's possible to restrict editing to certain users.<br />
I'm considering the <a href="https://en.wikipedia.org/wiki/General_Data_Protection_Regulation">GDPR</a> which will come in application in May, which requires transparency about personal data collection. This would be open data, but because it is a legal obligation, it would have to be <em>regulated</em>, by people mandated by Data Protection Agencies for example.<br />
I could totally do that on an external database, but is it possible to add it <em>officially</em> on OSM ? If I want some user interaction (<em>e.g</em> clicking on a device to display information about its data collection and processing), is it possible on an official layer ?</p>
<p>Thank you for your answers</p>

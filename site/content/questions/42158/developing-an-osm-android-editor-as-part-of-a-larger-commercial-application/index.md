<div class="answer-body">
<p>only in browser? No, that is wrong. One of our main <span>editors</span> – in fact THE editor – runs not in the browser but locally on the user's computer via the Java platform. Also see <a href="https://wiki.openstreetmap.org/wiki/API">https://wiki.openstreetmap.org/wiki/API</a> and <a href="https://wiki.openstreetmap.org/wiki/Android">https://wiki.openstreetmap.org/wiki/Android</a> .</p>
<p>Regarding "commercial" use, see the <span>older "commercial" questions</span>.</p>
<p>When writing an editor you need to take care to educate your users what they should do/modify/add, what materials they are allowed to use (legal isses, see <a href="https://www.osmfoundation.org/wiki/License/Contributor_Terms">https://www.osmfoundation.org/wiki/License/Contributor_Terms</a> ) … this is to ensure that after all you and your users are not damaging the OSM data but rather enhancing it.</p>
<p>I would recommend to discuss (present your idea and request comments) your more detailed plans <a href="https://wiki.openstreetmap.org/wiki/Contact">with the community</a> (not here, as this platform is not technically suited) before doing major steps to prevent running into a wrong direction.</p>
</div>

<p><em>(Others have mentioned how non-web-browsers are OK)</em>.</p>
<p>There is no problem with commerical applications, or even selling OSM data. So long as you obey the OSM licence.</p>
<p>Remember that the OSM project might have different goals than your project. You may want to store something that OSM doesn't want to store (e.g. ratings). In which case you'd need to have an additional database that stores your things. You cannot and should not use OSM as a generic geographic database that you can stuff any data you want in there.</p>

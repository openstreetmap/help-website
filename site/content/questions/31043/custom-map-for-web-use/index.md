<p>What you ask for is fully possible. You want to put a map on a web site with 10k members; no problem, we have 1.5 million. You want 16:9; no problem, set the containing div for that. You want to choose the center point of the display; not a problem. To make this dream of yours come true, you'll need to either:</p>
<ol>
<li>Learn to implement it for yourself</li>
<li>Hire a consultant to implement it for you</li>
<li>Find an existing web service that is "close enough" and buy access to that service</li>
<li>Some combination of the above</li>
</ol>
<p>You might start by looking at <a href="http://switch2osm.org">switch2osm</a> for some hints on how to put a map on a web site. This site <a href="http://help.openstreetmap.org">Help</a> has several series of questions to go into more detail when you are ready for it.<br />
</p>

<div class="answer-body">
<p>A web map which can be set to a specific location and can be zoomed is this what we have on <span>www.openstreetmap.org</span>. It is called a slippy map. For our default style you can even get a pre-made html snipped in the "share" menu on the right. You can adjust the dimensions easily in HTML. For more info see <span>our wiki about Slippy Maps</span>. It also mentions other tile sources. Please always respect tile usage policies (how much tiles you can download via your website). For the default osm.org style it is: <span>tile usage policy</span>. If your online forum is a small, low-traffic one using osm.org's tiles should be okay. For just testing it is in any way.</p>
<p>If you want a custom map you can make your own tiles (which you can display in a slippy map). See e.g. <span>custom-tagged questions</span>. But I guess you will be satisfied with some other already existing tiles.</p>

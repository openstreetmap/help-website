<div class="answer-body">
<p>OpenStreetMap currently doesn't have an external model repository <a href="http://forum.openstreetmap.org/viewtopic.php?id=28248">however you could help develop one</a>. You can however include a <span>simplified 3d mapping in OSM</span> (but the key word is "simplified"!) which is likely what you would want to do for navigation in any case. As far as I know there is currently no automated way of converting from external formats to OSM simple 3d buildings.</p>
<p>In any case you must own all rights to the model in question and need to check trimble (sketchup is now owned by them) relevant terms of service for allowed use before even considering using it in any form in OSM.</p>

<div class="answer-body">
<p>To get your models shown in OsmAnd you need to have a look which kind of 3D models are supported in OsmAnd. In fact, I am not aware (disclaimer: I use OsmAnd only occasionally) that OsmAnd supports any kind of 3D buildings, currently.</p>
<p>The most popular building modelling is what Simon mentioned, so I would suggest to use it. All OSM-based <span>3D maps</span> which <span>support this</span> kind of modelling will then show your buildings. However, it is not easy and doing it right needs some experience / time to learn it.</p>
<p>Simply adding the number of levels and specifying the building type of the already existing (in our database) buildings would be the first <em>easy</em>, but high-impact step into the 3D world. <a href="https://www.openstreetmap.org/welcome">Begin</a>! :-)</p>

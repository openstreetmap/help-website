<div class="answer-body">
<p>This looks a lot like <a href="https://github.com/openstreetmap/iD/issues/3973">https://github.com/openstreetmap/iD/issues/3973</a> - which is a Chrome bug that we've already added a workaround for.</p>
<p>I did see "entity not found" errors in the console, and I was able to step through and see that the code was throwing the error for an entity that was definitely loaded already.</p>
<p>We probably won't see an update to the Strava fork of iD, so I think the best thing you can do is use another browser instead of Chrome.</p>

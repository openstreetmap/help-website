<p>I'm mainly a consumer of OSM data and noticed a changeset breaking an app of mine. It turned out to be a bulk removal of a tag I was relying on. The changeset was commented on, asking the submitter to please not do this again. In the meantime, I found there are work-arounds for me.</p>
<p>But this experience made me wonder: <strong>How do changes make it into OSM in general?</strong></p>
<p>For instance, is there a review process for changeset, like Wikipedia has with its <a href="https://en.wikipedia.org/wiki/Wikipedia:Pending_changes">pending changes protection</a>? Or is any change going live right away?</p>
<p>(Note: I'm not looking for technical advice here, this is more of a process question.)</p>

<div class="answer-body">
<p>There is no approval process for normal edits, all edits go live as soon as the changeset is uploaded. <a href="https://wiki.openstreetmap.org/wiki/Import">Imports</a> and <a href="https://wiki.openstreetmap.org/wiki/Automated_Edits_code_of_conduct">automated</a> edits require prior discussion or tend to see speedy reversal.</p>
<p>There is a <a href="https://wiki.openstreetmap.org/wiki/Data_working_group">Data Working Group</a> that handles disputes and reverts harmful edits and vandalism. Damage doesn't normally last very long.</p>

<p>In this instance please do drop an email to the Data Working Group at data@osmfoundation.org . Some tags (include some associated with some imports) have been widely discussed and removed en-masse, whereas other tags sometimes get removed because "people don't see the point of them" or think that the last person to edit a particular wiki page had accurately summarised the OSM community's views about something (this can be difficult when there is a divergence of views).</p>
<p>-- Andy (from the DWG)</p>

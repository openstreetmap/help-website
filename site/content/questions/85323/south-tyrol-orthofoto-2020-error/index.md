<p>Please let me know if I should report this issue elsewhere. When trying to edit, for example <a href="https://www.openstreetmap.org/edit#map=17/46.84214/12.23228">https://www.openstreetmap.org/edit#map=17/46.84214/12.23228</a> and selecting any of the "South Tyrol" options, the screen goes black. <a href="https://osmlab.github.io/editor-layer-index/">https://osmlab.github.io/editor-layer-index/</a> Here one can look up the existing URLs. <a href="http://daten.buergernetz.bz.it/de/organization/provbz?q=orthofoto&amp;sort=score+desc%2C+metadata_modified+desc">http://daten.buergernetz.bz.it/de/organization/provbz?q=orthofoto&amp;sort=score+desc%2C+metadata_modified+desc</a> Here one can look up what the correct URLs should be, it appears there are always 2 options but one of them (the one used by OSM) broke. Example for 2020 Orthofoto: <a href="http://daten.buergernetz.bz.it/de/dataset/ortofoto-20201">http://daten.buergernetz.bz.it/de/dataset/ortofoto-20201</a> which leads to <a href="http://geocatalogo.retecivica.bz.it/geokatalog/#!home&amp;layer=p_bz%3AOrthoimagery%3AAerial-2020-RGB">http://geocatalogo.retecivica.bz.it/geokatalog/#!home&amp;layer=p_bz%3AOrthoimagery%3AAerial-2020-RGB</a> as the first option, where the second option (<a href="https://geoservices.buergernetz.bz.it/mapproxy/p_bz-Orthoimagery/ows?service=WCS&amp;version=1.1.0">https://geoservices.buergernetz.bz.it/mapproxy/p_bz-Orthoimagery/ows?service=WCS&amp;version=1.1.0</a>) broke for OSM. Thanks in advance everyone!</p>

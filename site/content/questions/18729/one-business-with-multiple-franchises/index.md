<div class="answer-body">
<p>If, like in your example, the single shops/"franchises" are in the same room and not spatially separated I would just add the multiple tags to the one node. If necessary with ";"-separated values for one key. But... see the wiki section: <span>When NOT to use a semi-colon value separator</span> (this mentions quite your example problem, by the way). So, it is a trade-off. If there is some franchise which is more important (hopefully to all ploeple) or which includes the other franchises somehow, then only tag the most important franchise.</p>
<p>Regarding your comment at <span>the question "Tagging more than one company in one building"</span> ("Will give it a go with multiple tags on the one node and then will confirm if it works in osmand when it updates next"): the tagging may not produce any visible results (did you mean that by "if it works"?) depending on the renderer/application which you use, that is expected by me. In case you did not know, please have a look at <span>our wiki article about "Tagging for the renderer"</span>. If there is a logical problem (or some better solution exists) with the <em>multiple shop/amenity tags on one node approach</em> then, of course, another tagging scheme should be chosen. I am curious for comments by others.</p>
</div>

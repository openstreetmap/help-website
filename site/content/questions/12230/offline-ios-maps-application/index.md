<div class="answer-body">
<p>There are many applications like that already, sadly the idea of Open Source seems to be virtually unheard of in the IOS world so you'll probably have to re-invent the wheel and code what lots of others have coded before you.</p>
<p>Read <a href="http://bennolan.com/2011/06/03/offline-mapping.html">this article</a> for a basic analysis of the options available to you. On our Wiki there is a <a href="https://wiki.openstreetmap.org/wiki/IOS">list of IOS applications</a> that use OSM, and the list also tells you whether these opted for tile or vector storage. A few of the programs mentioned make their source code available so you can look there for inspiration.</p>
<p>If your needs are relatively simple then you might get away with using the JavaScript library OpenLayers that we use on web pages and run that locally on the browser. Note however that any solution based on raster tiles, whether OpenLayers or other, requires that the tiles for the area of interest have to be created and downloaded onto the device first, and that you cannot download them from OpenStreetMap's tile server due to its <a href="https://wiki.openstreetmap.org/wiki/Tile%20Usage%20Policy">Tile Usage Policy</a>.</p>

<div class="answer-body">
<p>MapBox ios sdk (which is also based on Route-me, more specifically, the Alpstein fork) has a fairly good implementation of offline cache. You can modify it to make it store permanently. However, the license prohibit you from re-distribute the tiles you cache from MapBox for yourself and transfer them to users. So you would have to change the tile source if you wish to offer offline maps directly to users.</p>

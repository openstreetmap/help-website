<p>Should islands/islets be the inner border of natural=bay areas?</p>
<p>Recently, Gulf of Bothnia (<a href="https://www.openstreetmap.org/relation/8421537">https://www.openstreetmap.org/relation/8421537</a>) between Sweden and Finland was changed from a single node into a huge multipolygon area. This has lead to some problems on several map services. For example both Mapbox and Wikimedia Maps seem to treat natural=bay as a water area, resulting in all of the islands turning invisible on their maps.</p>
<p>Due to the sheer amount of islands, to me it seems almost impossible to add all of them as inner boundaries of the natural=bay area.</p>
<p>I am familiar with the concept of "don't tag for the renderer", so I wonder what is correct mapping in this case? Should Gulf of Bothnia be changed back into a node or should it stay as an area?</p>

<p><a href="https://wiki.openstreetmap.org/wiki/Tag:natural=bay">https://wiki.openstreetmap.org/wiki/Tag:natural=bay</a> says:</p>
<p>"There is currently no definitive rule if small islands within a bay should be mapped as part of the bay or if they should be excluded of the bay area as inner rings of the corresponding multipolygon relation."</p>
<p>Hence, I doubt we will get a good "answer" here. I can think of pro and con for each solution. I think this question should rather be discussed in the <a href="https://wiki.openstreetmap.org/wiki/Contact_channels">forum or mailing list or wiki talk page</a>. If you take this discussion elsewhere, please mention a link here.</p>

<p>The answer is an obligatory YES no matter whether a related discussion is going on or on which forum the question has been asked. In the mentioned example there are over 3120 islands missing in the uploaded bay/fiord feature in an MP format like here (when clicking, please wait, it is a huge MP to render) <a href="https://osm.org/go/0eMBO--?relation=8421537.">https://osm.org/go/0eMBO--?relation=8421537.</a> Rendering this object will cause a serious damage to a map like here <a href="https://goo.gl/n5XJga">https://goo.gl/n5XJga</a> even if it is used only in lower scales. Besides the missing islands there are other bad side effects like here <a href="https://goo.gl/QfFPHi">https://goo.gl/QfFPHi</a> roads over water, strange stripes like here <a href="https://goo.gl/Nqdc6w">https://goo.gl/Nqdc6w</a> probably caused by unsynchronized generalisation, and so on. To get a correct map, most of the map-makers ignore the bay feature but then we just have a huge data redundancy in the OSM source database. By the way, if a user insists on the fiord area feature, it could be resolved by only two nodes. A program can create the perfect MP area on-the-fly in no time, just like I did to detect the mentioned missing islands.</p>

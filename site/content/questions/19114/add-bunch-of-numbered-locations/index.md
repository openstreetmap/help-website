<p>But single Google Map is unable to take a bunch of addresses as input and doesn't offer a way to number locations, it's very time-consuming. I had to find and download numbered thumbtacks. <a href="http://wholelottalove.free.fr/prague.sights.png">Here</a>'s an example of Prague I started today.</p>
<p>Does OSM provide a better solution? Ideally, I could just paste a list of addresses, it would add all the locations on a map, each with a number, and let me re-order the locations easily.</p>
<p>Thank you.</p>

<div class="answer-body">
<p>This is only if you want to use the phone you mention; what I do is add the stuff I want to see into OSM database, then I download the OSM data in <a href="http://osmand.net/">Osmand</a> and use that for searching and figuring where I want to go. Usually I don't go where I planned because you always find better places when you are there.</p>
<p>You can also create a GPX file with favorites and place them on the SD card in a file named osmand/favorites.gpx. There are many tools to create GPX files, you can search for convert KML to GPX to find something that works.</p>

<div class="answer-body">
<p>OsmAnd is a good option as emj mentioned.</p>
<p>If you really want a printed map, you'll need some coding experience.</p>
<p>With <a href="http://wiki.openstreetmap.org/wiki/Nominatim">Nominatim</a>, you can ask for an address (or POI name), and get the position back. So if you put this in a script, you can create a list with positions and names.</p>
<p>Afterwards, you can create a simple <a href="http://wiki.openstreetmap.org/wiki/Leaflet">leaflet</a> webpage which displays these locations on a map. This map can be printed (or at least via a screenshot) on paper.</p>
<p>Of course, the quality of the reply from Nominatim depends on the data. If there are no housenumbers present in the data, it won't give you the exact position.</p>

<td><div class="item-right">
<div class="answer-body">
<p>En el mapa debes editar o añadir nodos, vías y elementos de uso peatonal o preferencial como ser aceras (footways), plataformas (platforms), escaleras (steps), obstáculos (access), pasos de peatones (crossing), vías peatonales o preferenciales (pedestrian/living_street), senderos y caminos (path/tracks) y otros. La superficie y rugosidad a cualquier elemento anterior ayudará. Todo esto será insumo para el servicio de ruteo. (ver como editar <a href="http://wiki.openstreetmap.org/wiki/Tag:highway%3Dfootway">aquí</a>)</p>
<p>El proyecto por si mismo no provee nada más que acceso a los datos, es decir, consultar o descargar un elemento, área, frontera o región. El servicio que necesitas proviene de proyectos o empresas como ser osrm, graphhopper, mapzen, yours, mapquest, etc (ver más <a href="http://wiki.openstreetmap.org/wiki/Routing">aquí</a>). El que ves en la página osm.org es sólo demostrativo y para feedback de mapeadores.</p>
<p>Según con qué lenguaje de programación realices tu proyecto será más o menos sencillo obtener la ruta según el perfil, en este caso, peatonal. Casi todos tienen interfaces de usuario que puedes probar una vez que hayas editado en el mapa. Incluso hay para celulares.</p>
<p>A grandes rasgos, el servicio de ruteo funciona obteniendo el punto inicial y final, luego a partir de algún algoritmo (distancia corta, rápido, etc) obtiene la mejor ruta utilizando los datos del mapa, y luego devuelve la ruta para mostrar con las indicaciones, y otros datos.</p>

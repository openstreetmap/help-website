<div class="answer-body">
<p>You should use umap for this. See <a href="http://umap.openstreetmap.fr/en/">http://umap.openstreetmap.fr/en/</a>.</p>
<p>The core OpenStreetMap data is not a suitable place for relatively ephemeral information. Currently there are projects to build on OSM to provide open mechanisms for storing such ephemeral data. These are in relatively early stages of developments:</p>
<ul>
<li><strong>OpenEventMap (1)</strong>: <a href="http://www.openeventmap.tum.de/">http://www.openeventmap.tum.de/</a> looks to be not under development any more (clue is imagery provided by Cloudmade in credits). Was a student project at Technische Universität München (TUM) by Bibek Shresta. Some slides <a href="http://www.slideshare.net/bibstha/presentation2-29113481">here</a>, and summary on the wiki.</li>
<li><strong>OpenEventMap (2)</strong>: <a href="http://www.openeventmap.net/">http://www.openeventmap.net/</a>. I don't know what the relationship between this instance &amp; the previous instance, but this has current events visible, a logon/registration mechanism and is published by Daniel Nyfeler. I haven't been able to logon &amp; edit so I cant add more.</li>
<li><strong>OpenEvent Database</strong>: <a href="https://github.com/openeventdatabase/backend/wiki">https://github.com/openeventdatabase/backend/wiki</a>. Christophe Quest (cq94) is presenting this at SotM-16 <a href="http://2016.stateofthemap.org/2016/what-where-when-lets-add-another-dimension-with-openeventdatabase/">today</a>.</li>
</ul>
<p>Although it should be possible to integrate OpenEvent Database with umap I doubt that this has been done yet. With only 30 specific objects to add it is certainly not a major task to add them directly, and this is what I would recommend you do.</p>

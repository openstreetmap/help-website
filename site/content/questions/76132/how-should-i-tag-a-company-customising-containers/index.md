<p>There is currently a node (<a href="https://www.openstreetmap.org/node/7816587720">n7816587720</a>) named Containertech surrounded by a building (<a href="https://www.openstreetmap.org/way/401247612">w401247612</a>) named Containertech and a surrounding area (<a href="https://www.openstreetmap.org/way/747868640">w747868640</a>) named Containertech. I would ordinarily recommend that the node be deleted as it contains no more information than the area, but it appears that there is a desire in the Norwegian community to keep address nodes for <a href="https://wiki.openstreetmap.org/wiki/Import/Catalogue/Address_import_for_Norway">synchronisation with official databases</a>, so I would only remove the Containertech tags from it in this instance. The name tag on the building is probably redundant unless it is also the building name in which case it should probably be migrated to <code>addr:housename</code>.</p>
<p>The building down the road could be tagged separately.</p>

<div class="answer-body">
<p>OSM tagging conventions for businesses primarily selling to other businesses aren't nearly as mature as those that sell to the general public.</p>
<p>Factories are normally tagged as <a href="https://wiki.openstreetmap.org/wiki/Tag:man_made%3Dworks"><code>man_made=works</code></a> and/or <a href="https://wiki.openstreetmap.org/wiki/Tag:industrial%3Dfactory"><code>industrial=factory</code></a>. I don't think there is anything more specific for this particular use case. You could add a short sentence about what they make to a <code>description</code> tag.</p>
</div>

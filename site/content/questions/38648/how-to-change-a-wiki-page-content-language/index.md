<div class="answer-body">
<p>I guess this answer needs an update. Although the information about prefixes is still correct, it <strong>is</strong> possible to set the page content language now.</p>
<p>To change the content language click the link "Page information" in the menu on the side (either second or third from bottom). The language is shown in the section "Basic Information". You can change the "Page content language" by clicking on the link "change".</p>
<p>This setting is now editable for users, so that "right-to-left" languages will appear with their letters in the correct order without using some complicated wiki templates. Of course, the setting can also be used for languages like Turkish.</p>

<p>There is the <strong>"move article"</strong>-function at the top that allows you to alter the name. I guess in your case you would like to move it to "Tr:Istanbul". Please take care, that you pick an international (here:english) base name for the article.</p>
<p>If you add the text {{languages}} at the source of the article, the page gets an nice translation menu. Further tricks can be found here: <a href="http://wiki.openstreetmap.org/wiki/Wiki_Translations">http://wiki.openstreetmap.org/wiki/Wiki_Translations</a></p>

<div class="answer-body">
<p>On That wiki, there's no support for multiple wikis per language. Interlanguage links actually use only prefixes for language codes, using lowercase. But for legacy reasons, 7 languages have been created with dedicated namespaces for their language codes that have been capitalized: DE, ES, FR, IT, JA, NL, RU</p>
<p>For all other languages, there's no dedicated namespaces and the prefixes are lowercased.</p>
<p>However because the initial of the prefix is in fact the initial of the pagename, and that wiki capitalizes the first letter of all pagenames, the first letter of the language code (and only that letter) is capitalized automatically. So you'll use "Tr:" for Turkish (not "tr:", not "TR:").</p>
<p>You'll also use "Zh-hans:" for simplified Chinese, "Zh-hant:" for traditional Chinese, "Pt:" for standard Portuguese, "Pt-br:" for Brasilian Portuguese.</p>
<p>Note that variants of the 7 languages are also using lowercase with first letter uppercased.</p>
<p>The supported language codes can be in those forms:</p>
<ul>
<li>"LL:" (only for the 7 languages above)</li>
<li>"Ll:" (e.g. "Tr" for Turkish)</li>
<li>"Lll:" (e.g. "Vec" for Venitian): note that some 3-letters prefixes such as "Way:" or "Tag:" are not usable as language codes (they are detected as English; they were not created as MediaWiki namespaces)</li>
<li>"Ll-rr:" (e.g. "En-gb:" for British English)</li>
<li>"Lll-rr:"</li>
<li>"Ll-ssss:" (e.g. "Zh-hans")</li>
</ul>
<p>where "L" or "l" are place holders for a letter of the main language code, "rr" is a country/region code, "ssss" is a standard script code such as "latn", "cyrl", "hans", "hant".</p>
<p>You should NEVER use any code for English, and not use country/region codes (except in "-rr" suffixes), e.g.</p>
<ul>
<li>don't use "JP" or "Jp" or "jp": use only "JA" for Japanese</li>
<li>don't use "GE" or "Ge" or "ge": use only "Ka" for Georgian</li>
</ul>
<p>Contents for specific countries should not use the namespace-like prefix with colon, but should preferably name the country itself, or use the code elsewhere: those pages will remain translatable and navigatable across languages.</p>
<p>It would have been better if all languages used the same convention for capitalization, but those 7 languages are already detected and managed in the languages navigation bar and lot of templates. The project to use the same lowercase code with a capitalized initial was proposed but never implemented: it would not just require renaming the 7 namespaces but changing many links (and reversing lots of redirects).</p>
<p>So please the convention and avoid creating new unneeded redirects that are hard to fix later when pages need to be moved/renamed: this causes extra maintenance.</p>
<p>Look at the navigation bar: you can unhide the list of languages with missing translations: all red links displayed use the correct capîtalization: follow these red links to create a translated article (after that you may rename your article in your language, but please keep the prefix unchanged.</p>
<p>There's no restriction in BCP47 codes you can use as suffixes (after a colon) for naming OSM tags such as "name", but they follow the standard BCP47 convention (lowercase only for language codes, uppercase for country/region codes in country-specific tags).</p>

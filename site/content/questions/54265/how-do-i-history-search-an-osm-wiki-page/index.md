<p>See <a href="https://lists.openstreetmap.org/pipermail/tagging/2015-May/024339.html">https://lists.openstreetmap.org/pipermail/tagging/2015-May/024339.html</a> : "actually wikiblame apparently works out of the box if you know which parameters to enter".</p>
<p>So, we can use <a href="http://wikipedia.ramselehof.de/wikiblame?user_lang=en&amp;lang=blank&amp;project=wiki.openstreetmap">Flominator's WikiBlame (with parameters set for the OpenStreetMap wiki)</a>. You just need to enter the "page" and "search for" boxes and press start.</p>
<p>Example: <img src="/upfiles/Screenshot_20170125_082220.png" alt="screenshot" /></p>
<p>It found the <a href="https://wiki.openstreetmap.org/w/index.php?title=ID&amp;diff=prev&amp;oldid=1038817">correct addition diff</a> for the example search.</p>

<div class="answer-body">
<p>You could look at the full page history and select the version that is half way between the oldest and newest page to see if the specific sentence you are looking for was added before or after that version. Then repeat the process in the half of the history where the specific sentence appears to have been added. This can be a bit of a laborious process if there are hundreds of edits but with this sort of binary search you should only need to look at a dozen versions of the page even if it was edited a thousand times.</p>
<p>Also, if you click on the "cur" link for a version you can see how the historic version differs from the current version. If you click on the "prev" link for a version you can see what changes were made to create that version. If you click on the radio buttons of two versions and then click "Compare selected revisions" you can see the changes made between two versions of a page. Helpfully, the difference pages allow you to step back and forth to see the differences between earlier and later revisions too. You might also find the edit comments will assist you identifying the version, as you might find when a section was added.</p>
<p>I know this might still seem a rather tedious method but it is faster than stepping though each revision of a page, revision by revision.</p>

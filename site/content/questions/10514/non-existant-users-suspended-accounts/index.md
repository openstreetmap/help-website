<p>Hi all,</p>
<p>hope this is the right place for this issue.</p>
<p>On monday a user I contacted (concerning the new license) via the OSM messaging system answered me via normal e-mail: When he wanted to accept the CTs, he got the following message (in German):</p>
<p><em>Benutzerkonto gesperrt Dein Benutzerkonto wurde aufgrund verdächtiger Aktivitäten automatisch gesperrt, um potentiellen Schaden von OpenStreetMap abzuwenden. Diese Entscheidung wird in Kürze von einem der Administratoren überprüft. Du kannst dich aber auch direkt an den Webmaster wenden, sofern du diese Angelegenheit klären möchtest.</em></p>
<p><em>Short translation: Your account has been suspended due to suspicious activity.</em></p>
<p><a href="https://www.openstreetmap.org/user/NaviComputer">User link</a></p>
<p>It's just a guess: In the moment some users are very busy in "re-mapping" hugh areas. The edits of the user I contacted are already totally re-mapped. Perhaps there is an automatism like: If a specific quantity of a users edits are deleted -&gt; he might be suspicious -&gt; suspend his account, when he tries to accept the CTs. In this Wiki list you can find also a lot of "none existing" user, I don't think that they are all misspellings: <a href="https://wiki.openstreetmap.org/wiki/Asking_users_to_accept_the_ODbL">Wiki</a> Maybe some of them wanted to accept the CTs.</p>
<p>I experience another increasing phenomenon: user links in the OSM Inspector (license change view) are leading to none existing accounts. Since it's not easy to delete an account, I suppose that there might be an interaction between an automatism to protect the OSM database and the deletions of data due to re-mapping actions.</p>
<p>Two examples in the same area (OSM Inspector) <a href="http://tinyurl.com/7f9vvfw">example1</a> <a href="http://tinyurl.com/7e7yjgc">example2</a></p>
<p>If I am right OSM might loose a lot of valuable data. And last but not least, how can we help user "NaviComputer"?</p>
<p>Cheers<br />
Carsten<br />
[meridianeum]</p>

<p>Looking at example one are you looking for <a href="https://www.openstreetmap.org/user/Aloisius78">this user</a>? If so, there is a chance that they have renamed their account. The ID will remain the same, but the username will be different, and without any suitable data sources to say what ID links to what username, OMSI etc won't be able to link to the current username.</p>
<p>This might change when a new full history update is generated as I think that will show user id and current name; I think one is planned for next week. But at present, if they've renamed their account and haven't edited recently there is no easy way to link a user id to a username as far as I know. To be honest, if I didn't care about the licence but were annoyed about all the emails I was receiving about it the first step I'd take would be to rename my account to make myself harder to find (I'd then change my linked email address to one I don't check any more).</p>
<p>I can't comment on your guess, but hope the account can be unsuspended so they can accept.</p>
<p>And finally, I'm not sure what your question about NaviComputer means - they are one of those users whose <a href="https://www.openstreetmap.org/user/NaviComputer">user page says no user</a>.</p>

<p>The "Aloisius78 " user actually has a space at the end of their user name. I found this out by looking at the raw XML of a way that they had modified, from one of your "osmi" links above.</p>
<p>You can look at their user page by using %20 instead of the trailing space, like <a href="https://www.openstreetmap.org/user/Aloisius78%20">this</a>. That won't show the %20 as you hover over the link but the link will work - you can type it in manually of course.</p>

<div class="answer-body">
<p>I suspect that the red bugs that you're seeing in Potlatch 2 are actually <a href="http://www.mapdust.com/">Mapdust</a> bugs and not <a href="http://openstreetbugs.schokokeks.org/">OpenStreetBugs</a>.</p>
<p>The procedure to reopen is as follows:</p>
<ul>
<li>Go to <a href="http://www.mapdust.com/">http://www.mapdust.com/</a></li>
<li>Close the large box in front of the map.</li>
<li>Zoom in to the area of interest</li>
<li>Select "Recently fixed" and "Recently closed (non reproducible / software bug)" but not "Open" at the left.</li>
<li>Click "apply"</li>
<li>After zooming in you may have to click "apply" again.</li>
<li>Click on the bug, and click "bug details"</li>
<li>Click on "reopen bug".</li>
</ul>

<div class="answer-body">
<p>The "bug symbols" that show up in potlach are not actual OSM data, so it is normal that you dont see them anywhere in your changeset. When you edit or close a bug in potlach, this action goes to a different database.</p>
<p>That database is <a href="http://openstreetbugs.schokokeks.org/">OpenStreetBugs</a>. If you go there instead of potlach, you should still see your bug (marked as "fixed") for a while. I do not know of a way to reopen the bug, but you should be able to create a new bug at the same spot with the same description.</p>
<p>Alternatively, you can do the actual bugfixing, so that the bug should indeed be closed :p</p>

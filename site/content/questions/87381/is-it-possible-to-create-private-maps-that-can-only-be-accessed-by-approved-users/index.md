<div class="answer-body">
<p>Yes, check out umap.openstreetmap.fr which lets you draw semi-private stuff on top of the public OSM map. You can choose between letting "anyone who knows the link" view the map, or "only people who also have edit permission", and you can list the user accounts that should have this permission. Careful, these advanced authorization controls are only available when you log in to umap with your OSM account - since umap also allows creating maps anonymously, it is easy to accidentally create an anonymous map which is then more difficult to control.</p>

<p>Hi,</p>
<p><a href="https://facilmap.org/">facilmap.org</a> is easier to use than umap (IMHO) and works well on mobile device.</p>
<p>The security is only through obfuscated links, so I wouldn't use it for really sensitive data, except if you host it yourself, with appropriate protection.</p>
<p>Be aware that the same is true for uMap, even if permissions are better handled, it is still hosted by volunteers, without special focus on security and privacy.</p>

<p>Not really OSM, but if it needs to be access controlled, then it might be something more suited to <a href="https://merginmaps.com/">Input/Mergin</a> or <a href="https://qfield.org/">QField</a> depending on how you want to sync things with the forms/layers set up in <a href="https://qgis.org/en/site/">QGIS</a>. These are meant for field data collection and display and I think they can be used without publishing anything outside of pre-approved people (barely played around with either of them myself). QGIS does have quite a learning curve though as it's aimed at people with a more traditional GIS background.</p>
<p>IIRC the people behind mergin are UK based have talked about special pricing for non-profits.</p>

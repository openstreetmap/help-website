<div class="answer-body">
<p>The wiki fulfils quite a different function e.g. for presenting longer more detailed documentation on various topics. However <em>some</em> bits of the wiki could perhaps disappear</p>
<p>This new <a href="http://help.openstreetmap.org">help.openstreetmap.org</a> Q&amp;A site functionally overlaps with the FAQ pages on the wiki (the main '<a href="http://wiki.openstreetmap.org/wiki/FAQ">FAQ</a>', the '<a href="http://wiki.openstreetmap.org/wiki/Legal_FAQ">Legal FAQ</a>', and the more recent '<a href="http://wiki.openstreetmap.org/wiki/Developer_FAQ">Developer FAQ</a>') This is clearly a better way to cover more question topics while still being nicely findable with tags etc. We may also arrive at better answers to questions this way. Then again an 'FAQ' page is a much more widely used and understood format (for the site visitor)</p>
<p>The <a href="http://wiki.openstreetmap.org/wiki/Help">'Help' page</a> on the wiki has always sat a little uneasily alongside other wiki content, perhaps because a busy wiki is a bad place to try to create simple concise user help, and also because in a way <em>all</em> wiki content should be help. That page is a bit of a backwater, receiving little attention (certainly I haven't thought about tidying it up in a while) and it's kind of a naming clash with <a href="http://help.openstreetmap.org">help.openstreetmap.org</a>, so I wonder if we should phase out that page somehow.</p>
<p>Currently we have a 'help &amp; wiki' link on the <a href="http://OpenStreetMap.org">OpenStreetMap.org</a> front page. Should that link the word help to <a href="http://help.openstreetmap.org">help.openstreetmap.org</a> perhaps?</p>
<p>All in all I'd say the systems have their place, we should aim to just cross link them as much as possible. Also I could look into making a plugin to present content from here within wiki pages.</p>
<p>To me the overlap between this and the forum is much closer. That's dealt with in a separate question: <a href="http://help.openstreetmap.org/questions/37/what-about-forumopenstreetmaporg">What about</a> <a href="http://forum.openstreetmap.org"></a><a href="http://forum.openstreetmap.org">forum.openstreetmap.org</a>?</p>

<p>Diary could be made much more attractive. Here's some ideas:</p>
<ul>
<li>provide readership statistics to posters (views, backlinks) [EDIT: you can do this yourself, using an image provider that tracks views]</li>
<li>provide notification if new comments on article you commented about [EDIT: this has been implemented 13/10/2016]</li>
<li>make adding images easier [EDIT: just use something like imgur - also provides stats. Syntax for images is explained next to the editing screen]</li>
<li>provide a reddit style view of entries, with most popular entries on top</li>
</ul>

<p>yes, as separate "issues", IMHO (since each can be tackled separately from a programming POV) … like as often users here should post separate questions. But mention your overall goal each time, so others know where you are coming from. <div class="answer-body">
<p>There is a basic misunderstanding somewhere in the question. There is no, really no, shortage of good ideas for OSM in general and for the website specifically. Matter of fact every self respecting OSMer can probably produce a list with multiple man-centuries worth of work of nice-to-have items in the time frame of a couple of minutes.</p>
<p>The scarce resource is developer time and attention and the practical consideration of limiting complexity to a (with the available resources) manageable level.</p>
<p>As a consequence feature requests tend to be taken far more serious if:</p>
<ul>
<li>you provide code yourself</li>
<li>you convince somebody else to code</li>
<li>you pay somebody to code</li>
<li>you convince somebody else to pay somebody to code</li>
</ul>
<p>I'm not aware of a feature that has been coded and tested in a reasonable way being turned down to date, however the additional complexity and resource consumption considerations might still lead to a feature not being integrated.</p>

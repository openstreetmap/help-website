<p>The sign-up form shows some strange copy-paste behavior -- it seems to allow pasting of some strings, but not others. None of secure passwords I generated (I used KeePass) could be pasted into the password form, but simple minded texts could be.</p>
<p>I can understand disabling copy-paste on the login form, but in the sign up form where the user has to enter it for the first time, preventing this might cause the user to select insecure passwords. Mine had about 80+ bits of entropy and I just couldn't type it in correctly -- definitely not <em>twice</em>.</p>
<p>Is there another way to do this? Also applies to password change form.</p>

<p>As a node:</p>
<pre><code>amenity=shelter
shelter_type=shopping_cart</code></pre>
<p>As an area/relation:</p>
<pre><code>building=roof
note=shopping_cart</code></pre>
<p>Although another suggestion is to use <a href="https://wiki.openstreetmap.org/wiki/Tag:amenity%3Dtrolley_bay">trolley_bay</a>.</p>
<p>So what is finally the correct way?</p>

<p>For the avoidance of doubt, "shopping trolley" is English and "shopping cart" is American.</p>
<p>You can use <a href="https://taginfo.openstreetmap.org/search?q=shopping_cart#values">taginfo</a> to investigate the popularity of these and other options.</p>
<p>There's not really a "correct" way to map very-low-volume features - I personally tend to pick something that's a compromise between reasonable English and reasonably popular for these sorts of things.</p>

<div class="answer-body">
<p>As <a href="https://help.openstreetmap.org/users/387/someoneelse">@SomeoneElse</a> says, there's no official answer. So I'll give you my personal opinion, I like amenity=trolley_bay because:</p>
<ol>
<li>Works for British English.</li>
<li>Not all of these are covered, so subtagging amenity=shelter isn't ideal (and you can always add other tags such as covered=yes if it is covered).</li>
<li>Same for using building (and adding data with the note key is not a good practice).</li>
<li>This is vague enough to work for shopping carts, luggage carts, and any kinds of carts/trolleys</li>
<li>It's used more than the other options I see in taginfo</li>
</ol>

<div class="answer-body">
<p>The point of uploading GPS tracks is that you and others can download them and display them overlayed on a base layer (OSM data, OSM renderings or explicitly allowed aerial imagery (e.g. bing's)) in an editor.</p>
<p>I don't know which <span>editor</span> you are using. However, in most OSM editors you can display your own tracks. Some rare editors can only display tracks which are opened from your local computer (uploading still benefits the other OSM mappers) or only uploaded traces. All top three editors iD, Potlatch2 and JOSM can display uploaded tracks and local tracks.</p>
<p>A short how to:</p>
<ul>
<li><span>iD</span>: For uploaded traces (aggregated(!) from all users): Click on the "background settings" (on the right side) and then enable "OpenStreetMap GPS traces" (note that it may take some time – maybe months – until a newly uploaded trace appears there). For local traces: drag and drop your gpx file (a file symbol on your computer's graphical user interface) to the editor area. See the GPS section of the help inside the editor (bookmark icon on the right side).</li>
<li><span>Potlatch 2</span>: for your uploaded traces use the drop-down menu of the button "GPS" and select "my traces" (or similar). For local traces use Menu "background" → "Vector file" → "Open". See <span>P2 Primer</span>.</li>
<li><span>JOSM</span>: For uploaded traces use the "download data" button (or menu File) (that download all traces in the area, no highlighting of your own, but manual filtering is possible). For local traces use the "open" button or menu file→open. See <span>guide</span> and that <a href="/questions/1931/how-do-i-see-past-gps-traces-in-a-specific-area">other question</a>.</li>
</ul>

<ol>
<li>create track using OSMtracker</li>
<li>upload track to OSM with OSMtracker</li>
<li>download track from OSM to local computer</li>
<li>click edit on the OSM track page</li>
<li>click layer icon in editor</li>
<li>click "local gpx file"</li>
<li>open local file dialog</li>
<li>drag and drop gpx file onto editor</li>

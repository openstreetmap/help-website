<div class="answer-body">
<p>I do some work on the Navit project, so I can speak to your problem with Navit:</p>
<ul>
<li>Did you check that your new street is actually included in the map you use with Navit? Can you see it in Navit? If not, get a new map.</li>
<li>The street you added is tagged as a pedestrian street. By default, Navit uses the "Car" vehicle (=routing profile), which obviously will not use pedestrian streets. Make sure you selected another vehicle, e.g. "pedestrian". In the internal GUI, it's under "Settings / Vehicle".</li>
</ul>
<p>If you still, can't get it to work, it might be a bug :-). Post a bug report on the Navit trac: <a href="http://trac.navit-project.org/">http://trac.navit-project.org/</a></p>
</div>

<p>Once you solve the above two points (is it in the map, pedestrian/car use), you may still find the street is not used because it is not the "fastest" route. When routing, Navit calculates a "time" to travel that road based on the road type (e.g. <code>street_0</code>) and the <code>speed</code> and <code>weight</code> assigned to that type of road. These come from the currently active <code>vehicle_profile</code> (see: <a href="http://wiki.navit-project.org/index.php/Configuring_Navit#The_Vehicle_Profile)">http://wiki.navit-project.org/index.php/Configuring_Navit#The_Vehicle_Profile)</a></p>

<p>Often (but not in your case) a common reason for routing not working properly is that roads haven't been joined up properly. The picture attached to the top answer to <a href="http://help.openstreetmap.org/questions/1022/what-are-the-most-common-mapping-mistakes-that-other-users-make">this help question</a> shows the problem - if a router thinks that roads don't join then it won't route you from one to the other.</p>

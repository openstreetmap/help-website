<div class="answer-body">
<p>The data is presented exactly as it is in the database; no effort is made by the OSM web site to parse the information because the OSM web site is not intended to be a public-facing POI directory. Other websites that use OSM data can (and do) parse the data and present it nicely. The data you entered looks correct to me.</p>

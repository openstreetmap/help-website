<p>My Project Requirement:</p>
<p>I have a project requirement where I need to create a database for the country of the United States of America containing the following columns: Zip code</p>
<p>Latitude(of zip code)</p>
<p>Longitude(of zip code)</p>
<p>Zip code 1 ( which is at a distance of 10 miles from zip code)</p>
<p>Zip code 2 (which is at a distance of 25 miles from zip code)</p>
<p>Zip code 3 (which is at a distance of 50 miles from zip code)</p>
<p>My approach:</p>
<p>1.Download the OSM file (in PBF format ) for the country of united states of America 2.Convert the PBF file into a format which contains the zip code, latitude, and longitude information of USA 3.Extract the information into the database</p>
<p>My Research : I tried using the site <a href="https://mygeodata.cloud/converter/osm-to-xlsx">https://mygeodata.cloud/converter/osm-to-xlsx</a> to convert pbf file into excel file.However, it had size limitations and I could not use it for free. I also searched the internet for various other formats in which the PBF file could be converted. I haven't got any such file format.</p>
<p>My Question: I have downloaded the OSM(PBF) files for the entire country of the United States of America from <a href="http://download.geofabrik.de">http://download.geofabrik.de</a></p>
<p>Do these files contain the following information :</p>
<p>All zip codes in the USA</p>
<p>Latitude</p>
<p>Longitude</p>
<p>In which format can I convert the PBF file so that I can get the above information? Can I get a converter for free which allows the conversion so that I can extract all the relevant information for my database?</p>
<p>Any help will be appreciated</p>

<p>OpenStreetMap is unsuitable for your project because:</p>
<ol>
<li>It does not have complete coverage of US ZIP codes, only patchy coverage</li>
<li>Where OSM has ZIP codes they will either be attached to individual addresses, hence you will find more than one occurrence of a ZIP code in the data, or they will be mapped as polygons, which would mean you'd have to construct these polygons first and compute their centroid</li>
</ol>
<p>Also, you seem to assume that ZIP codes are tied to one particular lat/lon pair when in reality they cover a larger area, hence it is unclear what you mean by the distance between two zip codes.</p>
<p>I suggest you use a web search engine to search "US zipcodes download" and you'll find a few providers giving you simple tables with zip code and lat/lon of the central point, which will make life much easier for you.</p>

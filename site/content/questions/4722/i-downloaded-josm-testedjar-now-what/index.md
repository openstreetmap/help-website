<div class="answer-body">
<p>Generally spoken, JOSM is a java application (hence cross platform), and not a windows binary, so the .jar should contain everything you need. You can make the JOSM-shortcut point to java with the location of the jar-file as parameter (and usually associating more memory to JOSM then java default).</p>
<p>A typical call would be:</p>
<pre><code>java -Xmx1024 -jar c:\dirtojosm\josm-tested.jar</code></pre>
<p>(you must have an entry to java in your system-PATH to be able to call it like this).</p>

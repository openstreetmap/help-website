<p>No not really. You can keep an eye on the OpenTraffic effort to see if it gets interesting.</p>
<p>You could also do something like produce a csv giving some sort of derived traffic information about each OSM way you have data for. Then people could merge that info with OSM and take it into account for routing or whatever. I suggest doing a dump like that instead of trying to add the data to OSM directly because the information is relatively subjective and there is no widely used scheme for it in OSM yet. There is an import process, but subjective information using novel tags isn't a great candidate for it.</p>
</div>

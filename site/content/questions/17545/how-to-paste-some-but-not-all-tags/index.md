<p>I select a building in JOSM, then use "copy all key values" in the properties box of the way. I then paste them into a text editor. I remove one or two and copy the result back into the paste buffer. I then wish to paste them into another way which has already some tags which clash with the (now deleted) tags of the original way. I can certainly paste them back into the text editor. When I try to paste them into a new way using ctrl-shift-V, I do not get the edited tags, I get the original tags, presumably in a non-system paste buffer of JOSM.</p>
<p>I have looked for shortcuts which say "paste from the system buffer" but am unable to find any, nor any obvious menu entries.</p>
<p>So, sorry to be so naive, but I can copy tags so how can I paste them back into a JOSM entity?</p>

<div class="answer-body">
<p>I feel that I am answering the wrong thing, because this seems to be too obvious (or is it a new – years later – JOSM feature?)</p>
<ol>
<li>select the source object</li>
<li>click into the tags dialog</li>
<li>mark all (ctrl+A)</li>
<li>hold ctrl and click the ones you do not want</li>
<li>ctrl+c</li>
<li>select your target object</li>
<li>ctrl+shift+v</li>
</ol>
<p>(likely you can replace the keyboard shortcuts with menu actions)</p>

<div class="answer-body">
<p>Oddly enough we don't seem to have a well-defined way of doing this. Normally there are 2 approaches:</p>
<ul>
<li>use an adjectival key (I.e., one which does not standalone) with the same name as the main tag (i.e, bar=<em>). This is</em> not recommended* as this key is already in use to indicate that other types of amenities (restaurants, hotels etc) have a bar on the premises.</li>
<li>use a similar key postpended with ":type", i.e., bar:type=*. This currently has a <a href="https://taginfo.openstreetmap.org/keys/bar%3Atype">minimal usage</a> with a single object tagged. I think this is the better option.</li>
</ul>
<p>The low usage of this type of tag makes it quite hard to recommend one or the other. This is certainly something worth discussion on the tagging list: off the top of my head I can think of wine bars, whisky bars, gin bars, enotecas and cafe-bars which might all benefit from some kind of subtagging.</p>

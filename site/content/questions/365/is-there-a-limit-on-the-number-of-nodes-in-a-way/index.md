<div class="answer-body">
<p>The number of nodes in a way is limited to 2,000. Just like other API limits, this is coded in the application.yml file in the rails port:</p>
<p><a href="http://git.openstreetmap.org/?p=rails.git;a=blob;f=config/application.yml;hb=HEAD"></a><a href="http://git.openstreetmap.org/?p=rails.git;a=blob;f=config/application.yml;hb=HEAD"></a><a href="http://git.openstreetmap.org/?p=rails.git;a=blob;f=config/application.yml;hb=HEAD">http://git.openstreetmap.org/?p=rails.git;a=blob;f=config/application.yml;hb=HEAD</a></p>
<p>There is currently no limit to number of elements in a relation, but it is possible that one could be added at a later time. You should certainly avoid ever having more than 2,000 members in a relation even if it is currently possible. Relations that big slow down editing and cause unnecessary waste of bandwidth.</p>
<p>For route relations or boundaries, make cascading relations if you're approaching a four-digit number of members. For multipolygon relations, split them in several parts.</p>

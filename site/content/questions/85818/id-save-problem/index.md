<div class="answer-body">
<p>1) abort this upload and save your edits as osmChange file (blue text at the bottom on the left hand side panel in the upload panel). This stops you from losing your edits if the issue cannot be fixed in iD (you can load them in to JOSM or Vespucci and upload them from there).</p>
<p>2) As this seems to be a clear bug open an issue (including the screen shot) on <a href="https://github.com/openstreetmap/iD/issues">https://github.com/openstreetmap/iD/issues</a> if an appropriate issue doesn't exist (it would likely make sense if you could translate the Turkish text for the iD maintainers).</p>
<p>3) If the issue can't be resolved in a reasonable time frame, upload with an editor that supports loading change files (JOSM has the most extensive conflict resolution handling so should be a good choice).</p>

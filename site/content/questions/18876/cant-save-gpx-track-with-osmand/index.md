<div class="question-body">
<p>Since an update to OsmAnd, about 3 months ago, I've no longer been able to save GPX tracks. My current version is 1.0.0 beta.</p>
<p>If I go to Menu -&gt; Define view, select GPX track, then "Show current track" I get the message "Please enable 'Log tracks to GPX file' settings". But I can't figure out how to enable that. This is what I've tried:</p>
<p>Under "Settings -&gt; Plugin Manager" I've selected "Logging services" and "Sleep mode functionality. Both these options show up with a red circle on the right and green text underneath.</p>
<p>Under "Navigation" the option "Log track to GPX file" is selected and "Logging interval" is set to 5 seconds.</p>
<p>My GPS receiver is on and working (I can see my position on the off-line map and my coordinates with GPS Status).</p>
<p>Am I missing something or doing something wrong? If this functionality no longer works in OsmAnd, can anyone recommend an alternative for saving tracks? I've read good things about OSMtracker. Can it continue to save tracks with the screen blanked and locked? That's pretty essential.</p>
<p>Thanks.</p>

<div class="answer-body">
<p>The default is to start recording after making these changes, however you may have disabled it an another way.</p>
<p>From the configuration menu (gear in the upper right) look in the "Right Panel" for "GPX recording." If this is checked, great, if not check it.</p>
<p>On the upper right side of the screen you should now see either a grey circle with the word "start" or a red circle with some distance measurements. If it says start, press it and it will now be recording. Otherwise it is a bug and should be filed with the OSMAnd people.</p>

<p>Is it <a href="https://www.openstreetmap.org/node/10053903305">this club</a>?</p>
<p>As far as I can see you did everything correctly. The club is now part of the OSM data. You can even search for it: <a href="https://nominatim.openstreetmap.org/ui/search.html?q=Rubicon%2C+Dnipro">https://nominatim.openstreetmap.org/ui/search.html?q=Rubicon%2C+Dnipro</a></p>
<p>I guess you are confused that the club doesn't appear on the map. This is expected since club=sport is not rendered at OSM currently. This is not an error, the data is correct. The reason is just that club=sport hasn't been added to the rendering stylesheet yet. See for example the <a href="https://github.com/gravitystorm/openstreetmap-carto/issues/4199">ticket for club=scout</a>.</p>

<p>Не стоит ориентироваться на сайт osm.org как эталон для проверки отображения объектов карты. Там множество объектов просто не отображается, как например, этот спорт-клуб. В то же время он прекрасно отображается в программах-навигаторах, например, Osmand. Так что можете смело добавить недостающие данные на эту точку. Кстати, если клуб занимает все здание, можно привязать его теги на полигон здания, а точку удалить.</p>

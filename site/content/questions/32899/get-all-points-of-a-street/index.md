<div class="answer-body">
<p>As the others already say, you should have a look about our <a href="https://wiki.openstreetmap.org/wiki/Elements">object primitives</a>. Then you will learn that any kind of road is a <a href="https://wiki.openstreetmap.org/wiki/Way">way object</a> with a attached <a href="https://wiki.openstreetmap.org/wiki/Key:highway">highway=* tag</a> and consisting of <a href="https://wiki.openstreetmap.org/wiki/Node">nodes</a> that hold the geoposition itself.</p>
<p>Then you need to find out how you want to generate your data. You can work offline on <a href="https://wiki.openstreetmap.org/wiki/Planet">planet extracts</a> (e.g. using <a href="https://wiki.openstreetmap.org/wiki/Osmosis">osmosis</a> and <a href="https://wiki.openstreetmap.org/wiki/Xml_schema">XML files</a> or PostGIS DBs) or if you want to query an API (e.g. <a href="https://wiki.openstreetmap.org/wiki/Overpass_API">OverpassAPI</a>). After filtering, you can create an import script for your particular application that adds the line objects in your desired format/DOM.</p>
<p><strong>Hint:</strong> There are already existing solutions that do <a href="https://wiki.openstreetmap.org/wiki/3D_Development">3D processing</a> with OSM.<br />
Keep also in mind, that tagging schemas might differ from place to place and are just somewhat 'semi-standards'. Also know that there might be 'unexpected' constellations as closed ways, area=yes full shapes and other suprises as multipolygones. So it's better to check your algorithm for a lot of different areas :)</p>

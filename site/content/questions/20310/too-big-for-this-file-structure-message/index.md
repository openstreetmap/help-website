<p>Yesterday I downloaded a gmapsupp.img from the <a href="https://wiki.openstreetmap.org/wiki/All_in_one_Garmin_Map">All-in-One</a> map page for my Garmin Dakota 20. The file is <a href="http://dev.openstreetmap.de/aio/europe/">6.6Gb</a>, uncompressed, and I stored it on my computer's C: drive. My Dakota has a 16GB SD card, empty except for a &lt;garmin&gt; folder. When I tried to copy the downloaded file into the SD card I got a message saying it was "too large for this file structure" and said that gmapsupp.img was a DISK IMAGE FILE, which is not true, is it? Should I try changing the img extension for the move? Is there a 'file association' that I should change? There really shouldn't be a problem loading a 6.6GB file into a 16GB chip, should there? Any advice?</p>

<p>I guess your SD card is formated with a <a href="http://en.wikipedia.org/wiki/Fat32">FAT32 filesystem</a>, in which case you will not be able to put a file larger than 4GB on it. Possible workarounds:</p>
<ul>
<li>Format your SD card with <a href="http://en.wikipedia.org/wiki/Ext2">Ext2/3/4</a> or <a href="http://en.wikipedia.org/wiki/Ntfs">Ntfs</a>. The problem with that is that your Garmin device is likely to only support FAT32 filesystems.</li>
<li>Use multiple smaller gmappsupp.img files, each showing a smaller area of the world. One advantage of this is that you can deactivate some of the map files, so that your device will feel faster. Another advantage is that you do not have to redownload the full 6GB if you just want an update of your area.</li>
<li><a href="https://wiki.openstreetmap.org/wiki/Mkgmap">Generate</a> a smaller map file, by removing some features. This is more technical, but can be an interesting solution.</li>
</ul>
<p>Concerning the "gmappsupp.img is a disk image file" question, this is just Windows's silly habit of guessing a file's type from its name. Ignore it.</p>

<p>As Vincent says, I would be very surprised if your GPS supports a filesystem that supports individual files larger than 4Gb. I'd leave your filesystem formatted as FAT32 and find a download that's smaller than 4Gb instead.<br />
</p>
<p>I presume that the download that you're trying to use is <a href="http://dev.openstreetmap.de/aio/europe/">from here</a>. According to the <a href="https://wiki.openstreetmap.org/wiki/All_in_one_Garmin_Map">wiki page</a>, you can contact the creator of these files via OSM. In fact a number of people have already asked him <a href="https://wiki.openstreetmap.org/wiki/User_talk:Master">exactly this question</a>, but with no reply yet. I suspect that what has happened is that an automatic process is used, and the file size (before all those delightful French buildings were imported!) was &lt; 4Gb, but now it isn't.</p>
<p>If you don't want the whole of Europe, then you can try one of the <a href="https://wiki.openstreetmap.org/wiki/OSM_Map_On_Garmin/Download">other Garmin maps</a>, such as <a href="http://garmin.openstreetmap.nl/">from here</a>, and choose which areas you want.</p>
<p>Another option (which is a bit more work, but would work if you did want the whole of Europe) which be to download the OSM data for Europe, filter out what you don't want, and then use <a href="https://wiki.openstreetmap.org/wiki/Mkgmap">mkgmap</a> to create a map in a style of your choice.</p>

<p>exFAT is a solution, yes, but only if the Garmin device has a filesytem driver in its firmware updated to support it. It's not clear (without performing tests) which file systems it will support and will accept to mount. NTFS support is unlikely, ext2fs could work but not sure that Garmin will automount it if it expects to mount only FAT-like systems. So the best you can do is to try formatting the SD card and store a small file on tit to see if it can see it (e.g. a small JPEG photo if it has a feature to render photos, or any other small map image file). You may need a firmware update on your device to accept new SD cards.</p>

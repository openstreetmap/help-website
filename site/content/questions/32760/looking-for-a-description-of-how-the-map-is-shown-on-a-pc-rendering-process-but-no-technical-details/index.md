<div class="answer-body">
<p>I'll have a go...</p>
<p>At it's heart, OpenStreetMap is just a <a href="http://planet.openstreetmap.org/">big pile of data</a> that represent things in the physical world such as <a href="http://www.openstreetmap.org/way/4437062">roads</a>, <a href="http://www.openstreetmap.org/node/455984558">trees</a> and <a href="http://www.openstreetmap.org/node/2251425855">all</a> <a href="http://www.openstreetmap.org/node/401948880">sorts</a> of other things.</p>
<p>The way that the data is organised is described <a href="http://wiki.osm.org/wiki/Elements">here</a> and data items have <a href="http://wiki.openstreetmap.org/wiki/Beginners_Guide_1.4">tags</a> with values that say what the data represents. Lots of information about the sort of data that is stored is available <a href="http://wiki.openstreetmap.org/wiki/Map_features">on the OSM wiki</a> and information about what tags people have used is also available on pages such as <a href="http://taginfo.osm.org/keys/elephant">this</a> one.</p>
<p>When you point a web browser at <a href="http://www.openstreetmap.org/">http://www.openstreetmap.org/</a> you're seeing a representation of that data (one of the the five that are available there) - click the "layers" button at the right to see the others. Behind the scenes the data is converted to an actual map by a process called <a href="http://wiki.openstreetmap.org/wiki/Rendering">rendering</a>. The actual maps that you see on OpenStreetMap.org are made up of individual tiles such as <a href="http://c.tile.openstreetmap.org/17/65094/41945.png">this one</a>. Most of these are actually created only when your web browser asks for them, from the latest OpenStreetMap data.</p>
<p>If you want to set up your own map server that you can point a web browser at you can, if you follow the instructions <a href="http://switch2osm.org/serving-tiles/manually-building-a-tile-server-12-04/">here</a>. You can use <a href="https://github.com/gravitystorm/openstreetmap-carto">the same style as the "standard" map on osm.org</a>, or <a href="https://github.com/hotosm/HDM-CartoCSS">the humanitarian style</a> or you can use these or others to <a href="https://www.mapbox.com/tilemill/docs/crashcourse/introduction/">create your own web map style</a>.</p>
<p>Lots of people have created lots of <a href="http://www.openwhatevermap.com/">maps in different styles</a> showing different sorts of data.</p>
<p>A web map is only one one sort of place where you'll see OpenStreetMap data. You can <a href="http://garmin.openstreetmap.nl/">download maps for handheld GPS devices and car sat-navs</a>, (or <a href="http://wiki.openstreetmap.org/wiki/Mkgmap">create your own</a>), for <a href="http://osmand.net/">mobile</a> <a href="http://wiki.openstreetmap.org/wiki/MapsWithMe">phones</a> and some services using OpenStreetMap data <a href="http://developer.mapquest.com/web/products/open/nominatim">don't involve maps at all</a>.</p>

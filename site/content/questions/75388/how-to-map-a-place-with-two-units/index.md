<td><div class="item-right">
<div class="answer-body">
<p>If the units are essentially separate buildings with separate entrances, exits and services then you can split the building as is done for e.g. <a href="https://wiki.openstreetmap.org/wiki/Tag:building%3Dterrace">terraced houses</a>.</p>
<p>For cases where the different units are just internal divisions of a large building then the <a href="https://wiki.openstreetmap.org/wiki/Simple_Indoor_Tagging">simple indoor tagging</a> scheme can be used. (There are some simple indoor tagging presets for JOSM to same on typing)</p>
<p>If there are more significant structural divisions between part of the building then <a href="https://wiki.openstreetmap.org/wiki/Key:building:part"><code>building:part=*</code></a> can be used on the individual parts.</p>
<p>For both the indoor tagging scheme and the building parts method the outline of the building should be retained as a <code>building=retail</code> way or multipolygon.</p>
<p>The <a href="https://wiki.openstreetmap.org/wiki/Key:addr#Detailed_subkeys">address scheme</a> has tags for unit number if this is displayed somewhere.</p>
<p>In most cases getting reasonably accurate internal dimensions is very difficult and nodes are used instead. If this approach is used it can still be useful to map entrance nodes on the perimeter.</p>
</div>

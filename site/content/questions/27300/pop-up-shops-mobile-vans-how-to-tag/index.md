<div class="answer-body">
<p>I would just create a node and tag the relevant amenity/<span>shop</span> type (in the example case it may be <span><code>cafe</code></span>) and the <span>opening hours</span>. <span><code>takeaway=only</code></span> might be useful to tag if it offers no seats (<span>terrace</span> maybe if only outside seats?). "building" is not here, so nothing to be tagged as "<code>building</code>". You could also add <code>building=no</code> to make clear that there is really no building. Some users used <a href="http://taginfo.openstreetmap.org/keys/temporary_amenity"><code>temporary_amenity</code></a> before - I think that is not really a good idea as it is a completely new key (no subkey or specification of a existing one). <span><code>toilets=no</code></span> might be good, if there are no toilets. Add a <a href="https://wiki.openstreetmap.org/wiki/Key:description"><code>description</code></a> tag - e.g. <code>description=regular coffee van each day at this same location</code>.</p>
<p>So, for your example case:</p>
<ul>
<li>amenity=cafe</li>
<li>opening_hours=07:30-18:00</li>
<li>takeaway=only</li>
<li>building=no</li>
<li>toilets=no</li>
<li>description=coffee van each day at this same location</li>
<li>name=Mary's mobile coffee paradise</li>
</ul>

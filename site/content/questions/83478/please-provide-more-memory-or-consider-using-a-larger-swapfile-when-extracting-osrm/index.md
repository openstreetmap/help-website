<div class="answer-body">
<p>For an osrm installation covering the planet (based on the used lua profile, car is the smallest, biking and walking are much larger in memory needed) you'll need at least 256GB of memory (and with the data in 2022, even that won't be enough anymore. The sample installation by routing.openstreetmap.de cuts up the planet in smaller areas and uses 3 servers with 256gb each - one wouldn't do a full planet anymore).</p>
<p>In general: vps won't help much if you want a full planet install (be it for a map, nominatim or osrm). And you won't be able to install more than one service on a dedicated server.</p>
<p>E.g. see here: <a href="https://github.com/Project-OSRM/osrm-backend/issues/6146">https://github.com/Project-OSRM/osrm-backend/issues/6146</a> where 64gb of memory is not enough to do an Europe extract.</p>

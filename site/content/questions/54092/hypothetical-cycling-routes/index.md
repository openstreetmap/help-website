<div class="answer-body">
<p>If it is an officially proposed cycle route then you might add it to the map with lcn/rcn/ncn=proposed or in relation state=proposed, see <a href="https://wiki.openstreetmap.org/wiki/Cycle_routes#Tagging_cycle_route_networks">https://wiki.openstreetmap.org/wiki/Cycle_routes#Tagging_cycle_route_networks</a></p>
<p>However if it is in a very early stage of planning and you are evaluating several variations you should not yet add it to the public database - you would either have to set up your own database and combine the real world data with your planning data there (as described in your first link) - or as an alternative, if you just want to visualize the proposed routes to somebody, you might just create a route in KML/GPX format in bikemap.net for example (there are other services that offer this too) and then print a map from that or even export the GPX-file and load it on a GPS-device or smartphone to follow the track in real world.</p>

<div class="answer-body">
<p>A reasonable way to proceed would be to extract suitable OpenStreetMap data and add <strong>OFF LINE</strong> projected cycle routes. This could be using JOSM, but be <strong>very careful</strong> not to upload the data, or you could transform the data into another format (shape, geojson etc) and edit it in a GIS tool such as QGIS.</p>
<p>Keeping the data in OSM format probably improves your options for routing analysis. To perform routing analysis on your modelled options you can use any one of a number of options: Graphhopper, OpenTripPlanner, OSRM, or pgrouting (in a PostGIS database).</p>
<p>A GIS tool such as QGIS would meet your needs for vector output in various forms.</p>
<p>For straightforward consultation as distinct from modelling of routes you can add GPX or KML in umap which will give you a suitable online map.</p>

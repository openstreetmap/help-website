<div class="answer-body">
<p>If it's in the OSM data, you can make a map that displays any name you like. <a href="http://toolserver.org/~osm/locale/ga.html?zoom=7&amp;lat=53.52725&amp;lon=-8.16284&amp;layers=BT">Here</a>, for example, is an example map that displays Ireland with Irish placenames.<br />
</p>
<p>The tricky bit is knowing that "in place X the names to display are languages Y and Z, in that order". Many places don't have "official" languages (most of the UK is an example of that) and even some places that do (e.g. Finland) aren't bilingual in the normal sense.</p>
<p>Regardless of how you define it, I suspect that one way forward might be to do a one-off transformation of the data that you're loading into the database that you're displaying from (and similarly to any applied change files). It's probably easier than trying to do the same thing with style sheets (although I'd welcome someone to prove me wrong on that). You'd need to write some code though.</p>
</div>

<p>In Belgium, a trilingual country, in some places we have bilingual street names. What do we do: under the name tag we put both names separated with a dash ('-'). Furthermore we use name:fr and name:nl to put the respective French and Dutch names. See e.g. <a href="http://www.openstreetmap.org/way/64487885">http://www.openstreetmap.org/way/64487885</a></p>
<p>This is something that was agreed upon by the Belgian community. Other countries might have different conventions.</p>

<p>I wouldn't add non-local-language names for a place when there's no evidence that the non-local name is ever used, except as a transliteration of the original. The relevant section of the wiki is here:</p>
<p><a href="http://wiki.osm.org/wiki/Names#Avoid_transliteration">http://wiki.osm.org/wiki/Names#Avoid_transliteration</a></p>
<p>However, the original question was about bilingual areas, where multiple names are in use - there of course it makes sense to add all those names that are used locally.</p>

<div class="answer-body">
<p>Yes, you still need to credit OSM, since you're using OSM's data. However, you may not legally be able to overlay OSM data over a Google Map due to Google's terms and conditions, but you would need proper legal advice to be sure.</p>
<p>Alternatively, you could avoid any legal problems by mapping the streets over an OSM-based map. The <a href="http://switch2osm.org/">Switch2OSM</a> site provides all the information you need to do this.</p>

<p>Remember that you need to comply with all your sources' term&amp;conditions when you mashup multiple sources together. If Google is ok with overlaying osm data on a google basemap, you still need to credit osm when you display the mashup.</p>
<p>If by "transfered [...] in google maps" you mean to submit the osm streetnames to google using googlemapsmaker or similar, then this is definitely not ok (unless you can convince google to credit osm on its map... good luck with that).</p>

<p>Even if you convince Google to credit OSM it's not OK. If data gets merged, all the sources need to be under an ODBL-compatible license, so Google would also need to free its data.</p>
<p>But overlaying data from one source with data from an other source isn't merging data, so that can be done if the other licenses accept it (it's OK for ODBL license).</p>

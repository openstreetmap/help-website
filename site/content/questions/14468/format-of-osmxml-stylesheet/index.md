<p>I've set up a tileserver per <a href="http://switch2osm.org/serving-tiles/building-a-tile-server-from-packages/">switch2osm</a>, and it runs as expected. I'd like to experiment with different map styles, so I set up Mapbox's <a href="https://github.com/mapbox/osm-bright/">OSMBright</a> and exported it from from TileMill, using TileMill's <code>Export to Mapnik XML</code> option.</p>
<p>I expected to be able to replace the default osm.xml file with the one generated by TileMill, but they have different formats, and <code>renderd</code> rejects the TileMill-generated file.</p>
<p><a href="http://pastebin.com/iX1uwuZa">This</a> is the original osm.xml</p>
<p><a href="http://pastebin.com/xwd0KHcV">This</a> is the TileMill-generated file.</p>
<p><code>renderd</code> objects straight away to the <code>&lt;Parameters&gt;</code> section in the TileMill file</p>
<p><code>Running in foreground mode... renderd[22588]: Starting stats thread renderd[22588]: An error occurred while loading the map layer 'default': Unknown child node in 'Map': 'Parameters' renderd[22588]: An error occurred while loading the map layer 'default': Unknown child node in 'Map': 'Parameters' renderd[22588]: An error occurred while loading the map layer 'default': Unknown child node in 'Map': 'Parameters' renderd[22588]: An error occurred while loading the map layer 'default': Unknown child node in 'Map': 'Parameters'</code></p>
<p>Could anyone explain what I'm missing, and how properly to use the file generated by TileMill?</p>
<p>Thanks, Keith</p>

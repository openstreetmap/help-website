<p>Hello, Today i set out investigation on how to start embedding OpenStreetMap instead of Google Maps in my blog. The blog is about travel in <strong>Iran</strong> where the language is <strong>Farsi</strong> which uses Arabic script. But the audience of the blog are <strong>English</strong> speakers. My browser's preferred language is English only. When I go to Google Maps and look at the map of Mashhad, Iran, all or most of the labels are in English (which is what I want because my readers are English speakers).<a href="https://maps.app.goo.gl/pgfYbVaoejoviHbM8">https://maps.app.goo.gl/pgfYbVaoejoviHbM8</a> But in OSM, everything is in Farsi. <a href="https://osm.org/go/zY1k90l-">https://osm.org/go/zY1k90l-</a> I already changed my preferred language in my OSM account (SETTINGS &gt; MY PREFERENCES) to en-US. I read another post on OSM forum that explains that local contributors are asked to put the labels in their own language.</p>
<p><strong>My question: there is no English version of locations in Iran that for some reason is not displaying for me, correct?</strong> I just want to make sure before I go back to using Google Maps.</p>

<div class="answer-body">
<p>OpenStreetMap is a database. The map at osm.org you see is just one rendering of that data.</p>
<p>Several third party tile renderers offer English as a label language in Non-latin-script local language areas. Have a look at the layers switch at osm.org as well as the list at <a href="https://wiki.openstreetmap.org/wiki/Raster_tile_providers">https://wiki.openstreetmap.org/wiki/Raster_tile_providers</a> and <a href="https://wiki.openstreetmap.org/wiki/Vector_tiles#Providers">https://wiki.openstreetmap.org/wiki/Vector_tiles#Providers</a></p>
<p>E.g. I do run a map at <a href="https://www.osmap.uk/#13/36.2893/59.6118">https://www.osmap.uk/#13/36.2893/59.6118</a> where you will see English labels for Mashhad, Iran (but please note that this map uses transliteration if no English label data is present. Transliteration works well for several non-latin languages but not that well for Arabic languages like Farsi).</p>

<p><a href="http://wiki.openstreetmap.org/wiki/Talk:Tag:amenity%3Dschool">amenity=school</a> should include the grounds, per the wiki. <a href="http://wiki.openstreetmap.org/wiki/Proposed_features/landuse%3Dschool">landuse=school</a> was a proposal that never was voted on, or even had much discussion, so I'd tend not to use it.</p>
<p>For the names:</p>
<ul>
<li>You could tag both names on the amenity=school way bounding the grounds, separated by a semicolon</li>
<li>You could leave the names untagged on the amenity=school way bounding the grounds, and create two amenity=school nodes within the grounds with name information</li>
</ul>

<p>How about tagging the grounds as <em>two</em> ways with the same nodes? Supposing the perimeter of the schools' grounds is nodes ABCD, the first way would have nodes ABCD and tags amenity=school and name="First school", the second way would have nodes ABCD and tags amenity=school and name="Second school", the building would be tagged just building=school, and there wouldn't be separate nodes for the schools. (Then there's no need for a landuse tag; of course, there may be a <a href="https://wiki.openstreetmap.org/wiki/Tag:leisure%3Dpitch">leisure=pitch</a> in the yard.)</p>
<p>I might be overly creative here.</p>

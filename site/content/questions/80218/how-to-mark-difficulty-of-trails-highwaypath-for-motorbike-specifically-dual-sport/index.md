<div class="question-body">
<p>For instance in Northern Thailand, single tracks (highway=path) can be either:</p>
<ul>
<li>A) steep mountainous trails with hard bends, cliffs, rocks/roots and used by locals on foot only. Only very skilled enduro riders with lightweight dirt bikes can attempt these.</li>
<li>B) flat or hilly terrain along rivers, lakes, fields, and forests: locals can navigate these on a small motorcycle, so perfect for any MTB or dual-sport motorbikes.</li>
</ul>
<p>Since I focus only on B) and want to avoid A) when scouting and planning itineraries, I am looking for a way to tag these trails (highway=path) so they are rendered differently.</p>
<ul>
<li>tracktype is only relevant for highway=track and to describe its softness</li>
<li>smoothness is good to describe the required clearance but not the steepness/tight bends requiring potentially pivot turns</li>
<li>highway=footway to describe A) would be ideal but I believe this tag should be used instead in urban areas</li>
</ul>
<p>Additional came across this proposal <a href="https://wiki.openstreetmap.org/wiki/Proposed_features/motorcycle:scale">https://wiki.openstreetmap.org/wiki/Proposed_features/motorcycle:scale</a> I like the idea of a rating system however not the fact that this would require some separate viewer.</p>
<p>Any ideas on how this could be solved? Thanks!</p>

<p>If you are on a bike you might have a Garmin unit. For these it is relatively easy to adjust the rendering style. Data volume in Thailand or Laos allows this.

<p>I have to confess: many years ago I made a noob tagging mistake - one that I have been perpetuating ever since! Shoot me now ;-)</p>
<p>I started custom tagging buildings around my university campus with 'building_number', corresponding to our organisational numbering of buildings. I realised much later, that I should have done something with a namespace that at least made some sense.</p>
<p>I'm thinking 'unimelb' (our main domain name) would be an appropriate namespace that we may reuse in other ways, and therefore the tag I'd use would be unimelb:building_number or something like that. Is this making sense so far? Is this the right way to go about it?</p>
<p>My next problem is, that there are now quite a few of these buildings, and being a point and click type of person, it represents quite a bit of work to change - a bit haphazard, too. I'm sure there must be a way of making a bulk change using some sort of command line/API magic. Am I right? Should I attempt this, or should I talk to someone with stronger query fu?</p>
<p>I have been tagging the buildings concerned (about 95% completed) with operator="University of Melbourne", partly so they're easier to isolate and style in Tilemill, but I figure it might also make it easier to perform the change I'm intending.</p>
<p>Any input/suggestions/help gratefully received!</p>

<div class="answer-body">
<p>My answer is similar to jgpacker's, but can be done more quickly and all within JOSM.</p>
<ul>
<li>Download the area containing all of the buildings in JOSM</li>
<li>Search (Ctrl+F) for <em>building_number</em>. This will select all objects in the area you downloaded that have this tag</li>
<li>Scan down the list of selected objects to make sure it contains only the things you want to change, namely the university buildings you tagged with <em>building_number</em></li>
<li>In the tag box at the top right, it will have a line for the tag <em>building_number</em> with the value <em>&lt; different &gt;</em></li>
<li>Click edit for this tag and change the key field from <em>building_number</em> to whatever new key you want to use, like <em>ref</em>, but don't touch the value field, leaving it as <em>&lt; different &gt;</em></li>
</ul>
<p>What this will do is retain the respective value field for each of the objects, but change the key for all of them. For example, <em>building_number</em>=100 would become <em>ref</em>=100. This saves having to edit each one individually or edit an .OSM file.</p>

<div class="answer-body">
<p>In JOSM, you can simply search(ctrl+f) for <em>building_number</em> or <em>building_number:*</em> and you will find them. That's probably the first thing you want to do to verify that only the buildings you added that actually have this tag.</p>
<p>If you want to automate this process, you may do the following:</p>
<p>After downloading the university via JOSM, save it as a <code>.osm</code> file. Close JOSM and open the <code>.osm</code> file in a text editor, and use the text editor's "replace all" functionality on <em>building_number</em> to the tag you desire to use. Then open this file in JOSM, download the data from OpenStreetMap, and merge the layers of the file with the OSM Data Layer. Send Changes.</p>
<p>In this case, you probably want to use <a href="https://wiki.openstreetmap.org/wiki/Ref"><code>ref=*</code> or <code>ref:unimelb=*</code></a> instead of <code>unimelb:building_number=*</code>.</p>

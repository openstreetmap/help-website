<div class="answer-body">
<p>"entity n1578754703 not found" is probably related to <a href="https://github.com/openstreetmap/iD/issues/2142">issue #2142</a>, which I fixed only a few days ago. My guess is you restored some changes after closing and reopening your browser.</p>
<p>As a workaround, you can try this:</p>
<p>After you click on the save button, iD will ask you to enter a save comment and show you the list of your changes in the left sidebar. <em>Click on each change</em> and wait until the map has loaded around that location (this is the workaround to load the entities that are missing). After you have looked at all of the changes, then click the big blue save button.</p>
</div>

<blockquote>
<p>Is there any way to generalise this answer? Seems a very useful thing to know &amp; may help in other situations.</p>
</blockquote>
<p>"entity not found" means that a way exists and iD is trying to do something with it (render it, or edit it, or a presave conflict check), but one or more of its child nodes are missing (not loaded from the openstreetmap API). If a user sees this, it's really an error in the iD code (and I fixed the last one that I know of), or the openstreetmap API misbehaving.</p>
</div>

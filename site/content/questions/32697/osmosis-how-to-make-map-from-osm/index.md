<div class="answer-body">
<p>I think this looks fairly straightforward. The --bounding-box command is not receiving any input.</p>
<p>The osmosis options need to be applied in order as each flagged parameter is a step in a pipe. So:</p>
<ol>
<li>Read the input file &amp; output to a pipe</li>
<li>Read an input pipe, clip the boundaries to a given bbox, output to a pipe</li>
<li>Read an input pipe, write to an output file in a given format</li>
</ol>
<p>When I first used Osmosis I often actually used to run each step separately, saving the intermediate result as file until I got used to the need to manage process order. You can name piped input &amp; output , but this tends to make the osmosis command line even more unwieldy.</p>
<p>So I think you command line should read:</p>
<pre><code>C:\OSM\osmosis\bin\osmosis.bat -plugin org.mapsforge.map.writer.osmosis.MapFileWriterPluginLoader --read-xml file=&quot;C:\OSM\south-africa-and-lesotho-latest.osm&quot; --bounding-box top=-29.85 left=30.9 bottom=-29.9 right=31.1 --mapfile-writer file=&quot;C:\OSM\durban.map&quot;</code></pre>

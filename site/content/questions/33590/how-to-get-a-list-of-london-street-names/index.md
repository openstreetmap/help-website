<div class="answer-body">
<p>Currently we don't provide an easy to use way to get plain lists. Thus you need to process it on your own:</p>
<ol>
<li>Open a OSM Planet extract <a href="http://download.geofabrik.de/europe/great-britain/england/greater-london.html">of London</a></li>
<li>Parse .XML file with a SAX parser</li>
<li>Trigger on highway=<em>, name=</em> items</li>
<li>Add this name to a dictonary</li>
<li>Save this dictonary list to a file</li>
</ol>
<p>That can be done easily in a Python file but as I say, AFAIK we don't provide a ready-to-use service.</p>

<p>The above can also be achieved with existing software like osmconvert/osmfilter, yielding a CSV file with London streets. However the post codes are another matter entirely; they will not be recorded as attributes to the street so deducing them will require spatial computations if it is possible at all. -- Also note one street can consist of a number of individual parts in OSM and therefore will show up multiple times in your result list.</p>

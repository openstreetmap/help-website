<div class="answer-body">
<p>Hi Andy, your changes ("Brierley Close" -&gt; "Briery Close") are visible to me!</p>
<p>The map displayed at <a href="https://www.openstreetmap.org">https://www.openstreetmap.org</a> updates according to its own schedule. Usually it's pretty fast, sometimes it can take a while. Don't fret -- the important thing is to get the correct data into the database!</p>
<p>Of course it's great to see the rendered map change, and it's a good way to double-check your work. Two tips:</p>
<ul>
<li>Your web browser might have the old map tiles cached, and won't even try to load new ones until the cache expires. You can force your browser to load all new images, cache notwithstanding, by holding down Shift while clicking the reload button.</li>
<li>The different zoom levels sometimes update at different times, so for a while the old data might be displayed on some zooms and the new data on others. This is normal; all zooms will get in sync eventually.</li>
</ul>
<p>Good luck &amp; happy mapping!</p>

<p><a href="https://help.openstreetmap.org/users/18153/ericjrw">@EricJRW</a> - in this case yes. The map at each zoom level of the maps at openstreetmap.org is made up of small tiles that are an individual file hosted on a website and recreated when things change. <a href="https://a.tile.openstreetmap.org/17/64784/43019.png">https://a.tile.openstreetmap.org/17/64784/43019.png</a> and <a href="https://a.tile.openstreetmap.org/18/129568/86039.png">https://a.tile.openstreetmap.org/18/129568/86039.png</a> are examples of those, and the "17" and "18" respectively is the zoom level each case. Each tile is created from the same map data, but a higher zoom level tile might show more information.</p>

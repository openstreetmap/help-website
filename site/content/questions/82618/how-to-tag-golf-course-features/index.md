<p>Hi kaptinkenny14 - what to use for grass on golf courses is a bit of an open issue; there are at least two approaches. I'm assuming here that in OSM you're <a href="https://www.openstreetmap.org/user/kaptinkenny14">this user</a>; apologies if I've got that wrong.</p>
<p>The editor that you're using (iD) does indeed suggest "landuse=grass" for a golf fairway as your screenshot shows. Some people think that is incorrect, and have raised that with the iD editor <a href="https://github.com/openstreetmap/iD/issues/8748">here</a>, and <a href="https://github.com/openstreetmap/id-tagging-schema/issues/203">here</a>. Discussion on the first of those explains a bit about why the wiki page <a href="https://wiki.openstreetmap.org/wiki/Tag:leisure%3Dgolf_course">here</a> says things like "Do not use the tag landuse=grass" on fairways.</p>
<p>The comments that you have received <a href="http://resultmaps.neis-one.org/osm-discussion-comments?uid=12752739">here</a>** are from one person who seems to be on a <a href="http://resultmaps.neis-one.org/osm-discussion-comments?uid=115894&amp;commented">singular crusade</a> in GB to rid GB courses of "landuse=grass" for fairways.</p>
<p>For completeness, I'm from OSM's <a href="https://wiki.osmfoundation.org/wiki/Data_Working_Group">Data Working Group</a>, and we got drawn into golf tagging because a couple of people editing golf courses in OSM complained about "automatic edits" to their previous tagging and the rudeness of some of the comments they'd received.</p>
<p>Editor maintainers always need to treat the wiki with care as some of the things that it says are self-contradictory. That said, there have been times where popular editors push forms of tagging that are very much in the minority in OSM. I don't have a personal view about which tagging is correct here, beyond "both are, because both are widely accepted". iD has a <a href="https://blog.openstreetmap.org/2021/11/05/self-introduction-of-the-new-id-developer/">new maintainer</a> and I'm sure that issues such as this are among the ones that they will look at.</p>
<p>Regards, Andy</p>
<p>** and somewhat confusing, containing as they do "The landuse of golf features is 'golf' not grass." and "landuse=golf is not a valid tag".</p>

<div class="answer-body">
<p>The iD editor presets for fairways &amp; greens comes with landuse=grass.</p>
<p>This is believed by some, including me, to be <a href="https://github.com/openstreetmap/iD/issues/8748">incorrect tagging</a> ("tagging for the renderer"). The history if long and complicated. Golf features have only recently been rendered on the main CartoCSS map, and people have used various subterfuges to make the mapped features appear: these include: <code>natural=beach</code> for bunkers, <code>leisure=pitch</code> for greens, and <code>landuse=grass</code> on fairways, etc. The result of this is that statistically these look like very popular combinations, which I presume is what the iD developers picked up. The presets, of course, reinforce it.</p>
<p>In practice the individual <code>golf=*</code> tags now actually work fine as standalone tags for rendering purposes, just as they have for other uses since they were introduced. Unfortunately there does not seem to be a way to avoid iD nagging for these tags.</p>
<p>The use of these tags on <code>golf=*</code> elements is totally uncomplicated if one is just considering golf courses, however <code>landuse=grass</code> is already a tag with multiple meanings and this usage adds another. This means anyone consuming <code>landuse=grass</code> for purposes other than straightforward rendering will need to provide special rules for 'golf=*` features.</p>
<p>Ultimately it is a matter of choice as to whether one ignores the message in iD, but be aware that others might not, and 'correct' your original decision.</p>

<p>You wouldn't be unique in thinking that "the landuse tag should mean the actual use of the land"; that point occasionally gets argued on the <a href="https://lists.openstreetmap.org/listinfo/tagging">tagging mailing list</a>. People have suggested "<a href="https://taginfo.openstreetmap.org/search?q=landcover">landcover</a>" in place of what landuse is often used for in OSM, but that is, I think, a minority view.</p>
<p>Lots of tags don't quite mean the same in OSM as in the outside world, often confused by English words meaning different things in different dialects of English, so to an extent we need to live with that.</p>

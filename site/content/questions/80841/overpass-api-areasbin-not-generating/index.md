<div class="question-body">
<p>I'm trying to setup overpass on a local server and everything is working except for areas. Whenever I try an area query it gives the following error:</p>
<pre><code>No such file or directory /opt/overpass/db/areas.bin File_Blocks::File_Blocks::1</code></pre>
And, in fact, that file does not exist. The only files in db with area in the name are (all of which are like 8 bytes):
<pre><code>area_blocks.bin.idx.shadow
area_blocks.bin.shadow
areas.bin.idx
areas.bin.idx.shadow
areas.bin.shadow
areas_shadow.lock
area_tags_global.bin.idx
area_tags_global.bin.idx.shadow
area_tags_global.bin.shadow
area_tags_local.bin.idx
area_tags_local.bin.idx.shadow
area_tags_local.bin.shadow
area_version
area_version.shadow</code></pre>
I know that areas have to be generated by another process, and to me it seems like it's not making them and I cannot figure out why. From <a href="https://wiki.openstreetmap.org/wiki/User:ZeLonewolf/Overpass_Installation_Guide#Area_Database_Initial_Load">here</a>, the process <code>./osm3s_query --progress --rules</code> has been running for 5 days, which is more than expected. The log in <code>area_update.log</code> just says this same thing over and over (&lt;'s removed):
<pre><code>2021-07-06 10:52:02: update finished
2021-07-06 10:52:05: update started
xml version=&quot;1.0&quot; encoding=&quot;UTF-8&quot;?&gt;
osm version=&quot;0.6&quot; generator=&quot;Overpass API 0.7.56.9 76e5016d&quot;&gt;
note&gt;The data included in this document is from www.openstreetmap.org. The data is made available under ODbL. /note&gt;
meta osm_base=&quot;&quot;/&gt;
/osm&gt;</code></pre>
I've tried restarting everything, deleting what 'area' files are there, using launching methods from both <a href="https://wiki.openstreetmap.org/wiki/User:ZeLonewolf/Overpass_Installation_Guide">here</a> and <a href="https://wiki.openstreetmap.org/wiki/Overpass_API/Installation">here</a>, and nothing has made a difference. I can't find many posts online about others having this issue which makes me think I'm just missing something painfully obvious, but either way any help or ideas would be very much appreciated!
<p><strong>EDIT:</strong> Fixed this, solution in comment below.</p>

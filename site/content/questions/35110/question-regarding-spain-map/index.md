<p>According to the Google Group, the place node in question seems to be <a href="https://www.openstreetmap.org/node/21323935/history">this one</a>.<br />
</p>
<p>Nominatim seems happy searching for Alicante - it finds <a href="https://www.openstreetmap.org/relation/342792">this relation</a>, with an admin_centre node. Regardless of whether that's "correct" or not, if Osmand can't find somewhere using that, I'd suggest that it's a bug in OsmAnd and should be fixed there.</p>
<p>That said, it may be worth contacting the mapper who removed the name from the former place node. There may have been a deliberate reason for the removal (which if so, you can pass on to the OsmAnd people) or it may have been an accident (in which case you or they can add it back).</p>

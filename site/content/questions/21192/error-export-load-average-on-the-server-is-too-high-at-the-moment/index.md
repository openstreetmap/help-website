<div class="answer-body">
<ul>
<li>No, you are not doing anything wrong. The osm.org server only does exports when it has free capacity to do so (we only have few resources, <a href="http://donate.openstreetmap.org">donations</a>!). You may use a different service do create a output (see stephan75's answer).</li>
<li>Concerning retrying on osm.org's servers:
<ul>
<li>Except than in the very busiest periods you should be able to do exports. I twice tried to export a whole small city to pdf (server load is at 170 - quite a busy time): works. The server <a href="http://munin.openstreetmap.org/openstreetmap/yevaud.openstreetmap/load.html">load</a> number is no direct limit any more (details, see <a href="/questions/42332/server-busy-error-when-its-really-not">there</a>), but you may use it to to spot not-that-busy periods in case it does not work at a specific point in time.</li>
<li>Old: <del>For server load on different daytimes, see this entry here: <span>rush-hour-on-osm-servers</span> (that is about the tile server, but I am pretty sure that the load graph looks similar for the pdf export function). According to this older answer (May 2011; <span>server-too-busy-to-export-a-map/1280</span>) the pdf (and other formats) export is disabled if the load (<a href="http://munin.openstreetmap.org/openstreetmap/yevaud.openstreetmap/load.html">graph</a>) is over 25. And according to this old answer yevaud is the server which does serve exports (and the tiles according to the status page and <span>server overview</span>).</del></li>
<li>→ <em>In short:</em> try again between about 23:00 and 5:00 UTC - that was the most quiet time yesterday.</li>
</ul></li>
</ul>

<p>This may depend on the size of the area you want to have as a PDF output.</p>
<p>The export tab on osm.org is the one that is very easy to use, but it has limited capacities.</p>
<p>So if you are willing to learn something about getting OSM based maps in other or similar formats, read the OSM wiki about <a href="https://wiki.openstreetmap.org/wiki/OSM_on_Paper">OSM on paper</a>, <a href="https://wiki.openstreetmap.org/wiki/SVG">SVG</a>, <a href="https://wiki.openstreetmap.org/wiki/Rendering">Rendering</a>, <a href="https://wiki.openstreetmap.org/wiki/Maperitive">Maperitive</a> etc.</p>
<p>There are also many hints about this topic on this FAQ site, try the search feature.</p>

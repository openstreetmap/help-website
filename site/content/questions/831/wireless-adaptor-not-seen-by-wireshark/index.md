This intermediate driver is also responsible for managing association/disassociation, BSSID scans and such. And this intermediate driver is also responsible for filtering the requests coming from the upper protocols (like WinPcap) for the underlying device description, and always returning "Microsoft" instead of e.g. "Intel Wireless 4965 Adapter".</p><p>I haven't looked if there is a possible workaround to the problem, yet.</p></blockquote><p>Do you have an adapter named "Microsoft"?

<p>If there is verifiable information (signposts etc) for the reason you could invent a value for <a href="https://wiki.openstreetmap.org/wiki/Proposed_features/hazard">hazard</a> Regardless of that what you probably want to do is use access like tags to document that the water shouldn't be entered etc there (swimming=no, fishing=no or similar).</p>
<p>You do have to realize that there is not going to be an application "right now" that uses the information except if you write one.</p>

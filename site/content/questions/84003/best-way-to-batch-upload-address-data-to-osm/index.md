<div class="comment-text">
<p>The community does entertain offers like that, but there are caveats that need to be addressed before such an import can be undertaken, such as:</p>
<p>-Whether the license on that data is compatible with the OSM database</p>
<p>-The accuracy of the location information</p>
<p>-How to convert the city's address data into appropriate OSM tags</p>
<p>-Conflation, to make sure imported data doesn't duplicate already-existing addresses in the OSM database</p>
<p>...plus a number of other factors. That's why imports like these need to follow the Import guidelines linked to by Frederik. It isn't as simple as taking a bunch of data and blindly importing it.</p>

<p>We don't do "batch uploads". Even if there really isn't any address data in OSM for your city (which I find unlikely - I would guess there is <em>some</em> at least), a "batch upload" would risk being at odds with other data that is there already - address points you upload might end up in the middle (or on the wrong side of) a road, or in a lake, or something like that. So there is no way around a manual review of any data you import into OSM. Generally, we also recommend involving local mappers in the effort. See <a href="https://wiki.openstreetmap.org/wiki/Import">https://wiki.openstreetmap.org/wiki/Import</a> for more information.</p>

<blockquote>
<p>Does OSM actually believe having a group of volunteers piece meal uploading city data will somehow be more accurate and efficient?</p>
</blockquote>
<p>Different OSM communities around the world have different approaches to imported data. Some (US, NL) love it; some (DE, GB) don't. Data to be imported can vary hugely in quality; nothing is 100% correct (even if it is allegedly "authoritative"), it can still be verifiably wrong if it says "there is a house at XYZ" and there isn't.</p>
<p>With an address import there is less margin for error than with something more complicated; like landuse. Many communities around the world have imported address data, included in the US, and there are several possible approaches that you could take (address nodes, interpolated address information, apply address information to buildings...) - you'll need to discuss what's best in your area with your local OSM community.</p>
<p>That discussion process is part of what <a href="https://wiki.openstreetmap.org/wiki/Import">https://wiki.openstreetmap.org/wiki/Import</a> takes you through. Follow that, including talking to your local OSM community (see <a href="https://openstreetmap.community/">https://openstreetmap.community/</a> to find that) - I'm sure you'll find people who can help.</p>

<div class="answer-body">
<p>This is easily the most asked and most answered question on the OSM help site.</p>
<p>A good visual rendering of the geo-data (commonaly called a "map") needs to be selective on what it actually displays, particularly when so much data is available as in OSM. You will either need to ask the maintainer of the "standard" map style to add a rendering rule for castles to the style or use/create a custom style for your purposes.</p>
<p>See for example <a href="http://geschichtskarten.openstreetmap.de/historische_objekte/?zoom=16&amp;lat=43.28327&amp;lon=5.31107&amp;layers=B0000FFFT">http://geschichtskarten.openstreetmap.de/historische_objekte/?zoom=16&amp;lat=43.28327&amp;lon=5.31107&amp;layers=B0000FFFT</a> (it seems that it doesn't render the polygon at this point in time).</p>

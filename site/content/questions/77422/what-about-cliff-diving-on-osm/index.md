<div class="answer-body">
<p>Sports in the "wild" (excuse my terminology) are only represented by a <code>sport=*</code> node at its location, here <code>sport=cliff_diving</code> (other similar <code>sport=</code> include <code>=fishing</code>, <code>=scuba_diving</code>, <code>=surfing</code>, etc). No other tag to express the "venue" part. <a href="https://wiki.openstreetmap.org/wiki/Tag:sport%3Dcliff_diving">https://wiki.openstreetmap.org/wiki/Tag:sport%3Dcliff_diving</a> suggests tagging on the <code>natural=cliff</code> directly. (although as an outsider I wonder whether there should be multiple surveyed diving spots around a cliff in addition)</p>
<p>[edit] I mistyped <code>leisure=fishing</code> there.</p>

<p>As a rule the "sport" tag is just an attribute of an existing object and does not indicate an object itself, with other words a node with just a sport tag, is not really correct.</p>
<p>But there is at least one notable exception, to this sport=scuba_diving which is widely used to indicate diving sport. So a sport=cliff_diving is probably in order here too. However as a rule I would only add this in locations where there are actual facilities enabling or documenting the use, for example a platform or a sign posted location.</p>

<div class="answer-body">
<p>In general use the original purpose of the building in the building tag: when a building is designed for a given role this tends to significantly affect the design. So I would retain <code>building=greenhouse</code>. For its new usage tag this with <code>building:use=warehouse|caravan_store</code> etc. (Personally warehouse doesn't seem quite right)</p>
<p>This issue often presents itself with disused churches. These will clearly be seen to be church buildings long after their religious usage has ceased; and their use can be diverse: residential accommodation, pubs, offices. Other examples are old warehouses, mills &amp; cinemas.</p>

<div class="answer-body">
<p>You could use a <span>lifecycle prefix</span> to describe the former building tag: <code>disused:building=greenhouse</code> (plus <code>building=warehouse</code>). I guess it still looks quite like a greenhouse, so this information may be useful (e.g. as navigation aid, or for (3D) rendering).</p>
<p>I even would tend to keep the <code>building=greenhouse</code> tag, because it still is ("on the ground") a greenhouse – it is just not used as greenhouse. The building tag describes not the use of a building, but the building itself.</p>
<p>The landuse tags should be on the land which is used for some purpose… usually bigger than the building.</p>
</div>

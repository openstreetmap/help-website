<div class="answer-body">
<p>I've reverted the changeset <span>36248847</span> in changeset <span>36279139</span>, and also made a few additions which I think mirror what you wanted to achieve (i.e., refine the shoreline). The problem was with a <span>relation</span>.</p>
<p>For future reference there are a number of useful tools for examining issues with multipolygon relations:</p>
<ul>
<li>Firstly one can see a relation directly as shown in the link above.</li>
<li>Secondly, <span>Geofabrik's OSM Inspector</span> provides a layer for <a href="http://tools.geofabrik.de/osmi/?view=water&amp;lon=90.84424&amp;lat=15.65447&amp;zoom=1&amp;overlays=bodies_of_water,broken_bow,vmap0_rivers,long_rivers,waterways_river,waterways_stream,waterways_drain,waterways_canal,waterways_riverbank,waterways_other,waterways_width,waterways_in_tunnels,waterways_on_bridges,waterways_without_names,waterway_nodes,coastline,simple_islands,coastline_nodes,rivermouths,coastline_not_simple,coastline_unconnected">finding gaps</a> (and other problems with multipolygons), but many of the multipolygons in this area appear to be poorly formed.</li>
<li>Thirdly, <span>OSM Relation Analyser</span> can be used to check individual relations.</li>
</ul>
<p>Together they can be used to spot what the problem in a relation might be and then it is often relatively easy to fix that directly in the editor of your choice.</p>

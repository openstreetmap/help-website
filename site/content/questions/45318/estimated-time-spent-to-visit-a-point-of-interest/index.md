<div class="answer-body">
<p>As others have mentioned such info is usually not (explicitly) recorded in our data. Although our open tagging system makes it possible that there are exceptions:</p>
<p><a href="http://overpass-turbo.eu/s/bxI">Some tourism objects which have a "duration"</a> tagged too – I guess this is the value you're looking for. And I've also found a test field for project: that <a href="https://www.openstreetmap.org/#map=16/48.8724/2.7730">Disney park near Paris</a>; many objects there have a duration tagged. <span class="small">Note: this (my text) is not a recommendation to use <a href="https://wiki.openstreetmap.org/wiki/Key:duration"><code>duration</code></a> on such objects. <em>IF</em> we would want to tag such a visit duration maybe a more specific key (<code>duration:visit</code>?) would be more useful to say what is meant.</span></p>
<p>Alternative: I could imagine that you could work with an own (to be done by you!) estimation based on our <span>classes of "POI"s</span> and their sub/accompanying tags. This would be similar to how usual/oldschool routing engines/profiles calculate the time to travel a specific route on streets. The wiki, taginfo and overpass turbo will be your friends in that case.</p>
<ul>
<li><code>tourism = museum</code> → 2 h</li>
<li><code>tourism = theme_park</code> → 7 h</li>
<li><code>tourism = viewpoint</code> → 0.25 h
<ul>
<li>+ <code>man_made = tower</code> → +0.1 h (a viewpoint which is on a tower)
<ul>
<li>+ <code>height = x</code> → +0.007*x h (e.g. <code>height=33</code> → + 0.231 h) (a tower which is 33 meters high)
<ul>
<li>…</li>
</ul></li>
<li>…</li>
</ul></li>
</ul></li>
<li>…</li>
</ul>

<p>There is no way you can extract that information out of osm data. Even if the POI was indoor-maped and you could devise a most-likely path through the place, it wouldn't account for how long visitors spend at each spot, which ones they skip, etc.</p>
<p>I suggest you ask for visitor statistics from the POI administrator themselves.</p>

<div class="question-body">
<p>For a presentation I would like to export the recorded GPX tracks of my sailing trips. The map should show the track together with the relevant POIs and use a nautical map style. Optimally, the export is a SVG so that I can post-process it.</p>
<p>The GPX tracks have been recorded with OsmAnd+. Please note, that this question does not ask how I get the GPX file from my smartphone to my PC. I know how to do that. The question addresses the problem which program or web service allows me to properly select the relevant POIs, a nautical map style and define the correct scale.</p>
<p>A already considered the trivial approach which means to take multiple screenshots of OsmAnd on my smartphone and then stitch them together. This doesn't provide me with an SVG, but I have the power to define which POIs should be shown and I get a correct, nautical map style. Nonetheless, this is a very tedious work.</p>
<p>Hence, I thought there should be better ways to do that.</p>
<p>My first approach was look for a proper Linux application from <a href="https://wiki.openstreetmap.org/wiki/Software/Desktop">this list</a>. My first choice was <a href="https://wiki.openstreetmap.org/wiki/KDE_Marble">Marble</a>, because it is the native application for my KDE desktop. The experience was disappointing. Marble only provides a very limited selection of map styles and Marble does not allow to select which layers of the OSM database are shown. In particular, Marble always shows all commercial and social POIs (i.e. stores, public amenities, etc.) but no nautical sea marks. :-(</p>
<p>Then, I tried some web services: <a href="https://www.outdooractive.com">Outdoor Active</a> and <a href="https://maposmatic.osm-baustelle.de/new/">MyOSMatic</a>. I know the former from my hiking tours, the latter has been recommended in some forum. However, Outdoor Active is only meant for for hiking, cycling and such things. The rendering results of MyOSMatic are ways below being usable. The result is very coarse, does not show many details and the labeling of the start and destination point covers 50% of the map. :-(</p>
<p>I actually cannot believe, that there is no (desktop) application which allows what I need besides the Android app OsmAnd+.</p>
<p>Does anybody know an proper desktop application or web service which</p>
<ol>
<li>allows to import a GPX track</li>
<li>provides a nautical map style</li>
<li>allows to define which POIs are shown and which are not shown</li>
<li>allows to define the proper scale and map section</li>
<li>allows to export the result to SVG?</li>
</ol>
<p>SVG export is not a hard requirement. I could also accept PNG if the resolution is sufficiently high.</p>

<p>The general jumping off page for this sort of thing is the bewildering array of options on the <a href="https://wiki.openstreetmap.org/wiki/OSM_on_Paper">OSM on paper</a> page of the wiki.</p>
<p>As a rather labour intensive option, <a href="https://www.qgis.org/en/site/">QGIS</a> covers all of those except for point 2. You would probably have to <a href="https://learnosm.org/en/osm-data/osm-in-qgis/">create the style yourself</a> or adapt it from <a href="https://anitagraser.com/2014/05/31/a-guide-to-googlemaps-like-maps-with-osm-in-qgis/">one available online</a>. It can handle raster layers though, so if you can find an appropriate pre-rendered style it should be possible to do your overlay that way too.</p>

<div class="answer-body">
<p>You mention "render an image from .osm files". We should probably make sure you're aware that is maybe over-complicating things. It depends on the kind of interface you want, but you might be better off taking a look at something like <a href="http://wiki.openstreetmap.org/wiki/JMapViewer">JMapViewer</a> which is a "slippy map" component embeddable within java projects, which works by bringing in tiles into your application from tile server, a lot like a conventional web map display. The tile server could be OpenStreetMap's, or <a href="http://wiki.openstreetmap.org/wiki/Tile_usage_policy#Alternative_OpenStreetMap_Tile_Providers">alternatives</a></p>
<p>However having said all that...</p>
<p>If you do create an application which "renders an image from .osm files" this is better in many ways. It could be the best way to make a super-fluid mapping application harnessing the power of OpenStreetMap's raw data. It is however, a more technical undertaking. <a href="http://wiki.openstreetmap.org/wiki/Mapsforge">Mapsforge</a> (in java) is quite new, but promises to make this easier in a pre-packaged embeddable way, so this may be the closes thing to what you're asking for here.</p>
<p>If you didn't already, have a play with <a href="http://wiki.openstreetmap.org/wiki/JOSM">JOSM</a>. With a small area of map data loaded in, you can zoom and pan around very fluidly. Try doing this with different <a href="http://josm.openstreetmap.de/wiki/Styles">Style settings</a> configured in preferences. That's all written in java, and supports MapCSS stylesheets. It may be possible to rip out the rendering logic from the rest of the editor application. But if you're delving into other codebases, there's lots of other java projects which do vector rendering, particularly for mobile e.g. <a href="http://wiki.openstreetmap.org/wiki/GpsMid">GPSMid</a> and <a href="http://wiki.openstreetmap.org/wiki/Navit">Navit</a></p>

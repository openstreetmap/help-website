<p>The Stollenwanderweg in Bad Bleiberg is a signposted hiking trail along which are historic mining features from Roman times to the early 19th Century (see <a href="http://members.a1.net/bl.kreuth/bb/stollenww.html">http://members.a1.net/bl.kreuth/bb/stollenww.html</a> ) One or two of the many adits have been recorded on the Data layer as "Man Made Features - Adit" but of course this doesn't show on the public map. Other features are mineral processing sites, water pumping features etc.</p>
<p>As I guess this is all strictly Industrial Archaeology, would it be appropriate to mark the features on the trail with the "historic - archaeological site" marker?</p>

<div class="answer-body">
<p>I don't know what you understand by "strictly Industrial Archaeology" but I have always assumed <code>historic=archaeological_site</code> refers to something that is ancient. So an old Roman adit that has been re-discovered could be marked by that tag in my opinion. I wouldn't consider an adit from 100 years ago that has probably always been there in the knowledge of the locals an archaeological site, though.</p>
<p>In the end it's a gray area but please avoid tagging something just to make it appear on the standard map on openstreetmap.org.</p>

<p>The hiking maps I make show adits. :)</p>
<p>I would not worry about the renderer: If the feature is there (i.e. visible), map it the best you can. The <a href="https://wiki.openstreetmap.org/wiki/Lifecycle_prefix">life cycle prefix</a> tagging scheme may be of help for those things that are visible but not used and probably in various states of decay. If there are informational <a href="https://wiki.openstreetmap.org/wiki/Tag:information%3Dboard">signs describing the features</a>, definitely map those.</p>

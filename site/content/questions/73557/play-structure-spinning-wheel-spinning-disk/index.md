<div class="answer-body">
<p>I would tag it as playground=spinningwheel, as this would follow the scheme playground=basketrotator, found the on the wiki page for <a href="https://wiki.openstreetmap.org/wiki/Key%3Aplayground">playground</a>. If you are editing in iD, you will find a header "All Tags" in the left pane, where you can add any tag you want.</p>
<p>I would remove the name:en tag, as it is not name Spinnning Wheel, it a spinning wheel. We also do not map benches with name=Bench. (explained better in <a href="https://wiki.openstreetmap.org/wiki/Names#Name_is_the_name_only">name is the name only</a>.</p>
<p>Normally you can use <a href="https://wiki.openstreetmap.org/wiki/Any_tags_you_like">any tag tag you want</a>, but in order to get traction, it is best to document new tags that you invent. If you are sure spinning wheel is the correct Britsh-English word for the toy you could simply add it to the <a href="https://wiki.openstreetmap.org/wiki/Key%3Aplayground">playground</a> wiki page.</p>
<p>However, for more complex proposals or in case you are not sure about the British English word, you could contact the <a href="https://lists.openstreetmap.org/listinfo/tagging">tagging mailing list</a> for help.</p>

<p>Nothing "prevents" you from using a "new" value. There are 20 ambiguous <code>=spinner</code>, and 1 <code>=spinning_dish</code> (and 1 <code>=spinningdish</code> and <code>=spinningbowl</code> in the approved style). We can think about whether to use <code>=spinner</code> + something like <code>spinner=*</code> (or spinner:<em>=</em><code>), or</code>playground=spinning_*` directly. Before that, it may be more important to clarify what are they actually called. Both "wheel" and "disk" may be confused with merry-go-rounds.</p>
<blockquote>
<p><code>name:en=</code></p>
</blockquote>
<p>Use <code>description=</code> and <code>description:en=</code> for generic terms.</p>

<p>it's not on wikipedia, so I don't know, so far adding things from wikipedia seemed to eventually get added as an option.</p>

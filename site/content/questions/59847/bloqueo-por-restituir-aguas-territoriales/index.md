<div class="answer-body">
<p>The message was <a href="https://www.openstreetmap.org/user_blocks/1485">this one</a> - sent as a message that you have to read before continuing mapping. It was done that way because you hadn't reacted to <a href="http://resultmaps.neis-one.org/osm-discussion-comments?uid=670163">previous messages</a> about your edits.</p>
<p>OpenStreetMap is a collaborative community - we need to work together, especially around things over which there is conflict in the physical world. If you think that a boundary in OpenStreetMap is incorrect I suggest that you discuss it with all communities involved. There are a number of links in that <a href="https://www.openstreetmap.org/user_blocks/1485">message</a> to help you do that.</p>
<p>A useful document that describes how borders are mapped in OSM is <a href="http://wiki.osmfoundation.org/w/images/d/d8/DisputedTerritoriesInformation.pdf">here</a> (pdf).</p>
<p>Andy Townsend (from OSM's <a href="http://wiki.osmfoundation.org/wiki/Data_Working_Group">Data Working Group</a>)</p>

<p>I guess it is really country dependent.</p>
<p>It looks like SIRET in France, for which we have a dedicated tag : <a href="https://wiki.openstreetmap.org/wiki/FR:Key:ref:FR:SIRET"><code>ref:FR:SIRET</code></a>.</p>
<p>In <a href="https://taginfo.openstreetmap.org/keys/organisasjonsnummer#overview">taginfo</a> you'll see that there is 3 instance of the tag <code>organisasjonsnummer</code> already. But maybe it would be best to use <a href="https://wiki.openstreetmap.org/wiki/Namespace">namespace</a> like <code>ref:NO:organisasjonsnummer</code> to make to meaning clear to foreign data users.</p>
<p>Anyhow, I think this would need discussion on a local communication channel, and documentation in the wiki ! ;-)</p>

<p>There's a developing consensus to use the format suggested by <a href="https://help.openstreetmap.org/users/13231/h_mlet">@H_mlet</a> for country specific references, i.e. <code>"ref":&lt;country_code&gt;:&lt;description of ref/number&gt;</code>. Recent examples, which were discussed in detail on mailing lists include <a href="https://wiki.openstreetmap.org/wiki/Key:ref:GB:uprn"><code>ref:GB:uprn</code></a> and <code>ref:GB:usrn</code>. The advantage of this approach is that it identifies identifiers which are of national interest only, and avoids potential clashes where the same abbreviation is used in different countries.</p>

<p>As we <a href="http://zverik.osm.rambler.ru/whodidit/?zoom=11&amp;lat=1.43666&amp;lon=102.29199&amp;layers=BTT&amp;age=187">see here</a> is seems that <span>the island</span> was <a href="http://osmhv.openstreetmap.de/changeset.jsp?id=26109834">deleted</a> (apparently).</p>
<p>I guess it was accidentally deleted when the user tried to delete a <span>newly added supermarket</span> which was accidentally added into the coastline (or even fully accidentally added). More guessing: Being slight off the node when selecting likely caused the coastline to be selected and then the delete command was issued without looking properly which object got selected.</p>
<p>I <a href="https://www.openstreetmap.org/changeset/26709391">have undeleted</a> the island's coastline. <span>It is now existing</span> again (with the same id as before). I did not undelete the supermarket (<span>its old position</span>).</p>
<p>Since costlines are treated specially in the rendering process, it may take <a href="/questions/276/why-do-the-changes-i-have-made-to-coastline-not-appear-on-the-map">longer</a> to show up again (a system admin says: nearly daily). By the way: it still is there on some other <span>layers</span> (some apparently had not updated their coastline yet).</p>

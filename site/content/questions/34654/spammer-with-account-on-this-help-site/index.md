<div class="answer-body">
<p>The OSM help site currently uses the <a href="http://www.osqa.net/">OSQA</a> software. I believe (based on my recollection of IRC conversations from some time ago - which might not be 100% accurate) that it's essentially a standard installation - we haven't customised it extensively. A search of other OSQA sites will reveal that the spam here isn't unique - the same spammers crop up there too. <a href="http://meta.osqa.net/tags/spam/">OSQA's own Q&amp;A site</a> describes some of the options available for dealing with it, and <a href="http://jira.osqa.net/secure/IssueNavigator.jspa">their issues list</a> shows people raising issues about how to deal with spam too. "Shouldn't we change our way and just remove their messages and block their account in one single action?" would as I understand it need a change to the OSQA software itself rather than the configuration, and I suspect that the reason that it hasn't happened is that no-one has volunteered to do it. Whether it'd be a good idea to put a lot of work into that I'm not sure - there may be better Q&amp;A systems out there.</p>
<p>Blocking spam accounts does happen (look at the <a href="https://help.openstreetmap.org/users/?sort=karma&amp;page=262">lowest karma users of this site</a>) but as you say spammers tend to create new user accounts for each message, so while this tidies the questions list up it doesn't necessarily reduce the flow of new spam. I suspect that the admins won't be suspending any spammers today because they're probably <a href="https://lists.openstreetmap.org/pipermail/talk/2014-July/070119.html">very busy with the server move</a>.<br />
</p>
<p>One thing that would reduce spam would be to prevent new users from creating links (they already can't post images) but that would severely reduce the usability of help.osm.org to people asking questions.</p>

<p>Hi ğŸ‘‹</p>
<p>I was editing today and I noticed many nodes with similar edit messages : "<em>Added a cafe</em>", "<em>Added a restaurant</em>", and the like (turns out this is the default edit message when adding a node via Maps.me)</p>
<p>It made me wonder if there were any good practices when writing edit messages on OpenStreetMap.</p>
<p>In a way, it reminds me of writing commit messages in git.</p>
<p>There are many guidelines and good practices for writing git commits, like <a href="https://chris.beams.io/posts/git-commit/">here</a> or <a href="https://medium.com/@steveamaza/how-to-write-a-proper-git-commit-message-e028865e5791">here</a>, covering everything from message length (50 chars for the subject line) to formatting (separate subject and body with a blank line) to the mood to use (imperative).</p>
<p>But I've never read any for OpenStreetMap.</p>
<p>Are there any? What should be said in these edit messages?</p>
</div>

<p>Ich <em>denke</em>, ich verstehe dein Problem nun (danke für deine Kommentare, deren klärender Inhalt nun in deine Frage <span>eingebaut</span> ist):</p>
<p>Unsere <span>Tile usage policy</span> könnte es erklären, falls du (bzw. dein Programm) direkt vom osm.org Server herunterlädst. Siehst du solche Platzhalterbildchen? Das liegt daran, dass die osm.org-Organisation sehr wenige Mittel hat, die Server sind gespendet und um den Dienst für alle verfügbar und schnell zu halten müssen Massendownloader gesperrt werden. Alternative Server zu dem Server direkt von osm.org sind am Ende der Seite genannt. Es geht hier übrigens nur um fertige Kartenkacheln (Rechtecke).</p>
<p>Der Entwickler von Mobile Atlas Creator sollte die Downloadfunktion wohl deutlich einschränken, damit die Leute nicht so viel herunterladen oder auf einen anderen Server ausweichen (eventuell <a href="http://switch2osm.org/">seinen eigenen</a>). Auf <span>Blocked applications</span> ist das Programm übrigens samt Erklärung aufgelistet. Dein Problem ist also anscheinend weniger Cachewolf, als der Mobile Atlas Creator, der sich (wahrscheinlich je nach Bedienung) nicht an die Spielregeln hält.</p>

<div class="answer-body">
<p>Um es einfach zu sagen: Das Ausrechnen der "Tiles", also der Kartenkacheln zur Anzeige der Karte, kostet Rechenzeit und damit Geld. Jemand, der online in der Karte herumsurft, dem schenkt das Projekt diese Rechenzeit. Ein Offline-Tile-Downloader geht weniger verantwortungsvoll mit dieser knappen Ressource um; mit Mobac hat man leicht das Tausendfache davon runtergeladen, was man live anschauen würde (und das meiste davon wird man auch nie anschauen - man lädt es "auf Vorrat" herunter). Das Projekt hat nicht genug Ressourcen, um auch diese Art der Nutzung unbeschränkt zuzulassen. Aktuelle Versionen von Mobac erlauben das daher schon gar nicht mehr; ältere sind vom OSM-Server aus gesperrt.</p>
<p>Alternativen: Entweder man bezahlt jemanden fuer die Rechenzeit, die man in Anspruch nehmen möchte (es gibt eine Anzahl von Anbietern, bei denen man Tiles abonnieren oder kaufen kann), oder man bringt die Rechenzeit selber auf, indem man sich seine eigenen Kacheln erzeugt. Dazu gibt es verschiedene Möglichkeiten, und wenn man nicht gerade ganz Europa ausrechnen will, reicht der heimische PC dafür auch völlig aus. Für kleinere Gebiete kann man mit <a href="https://wiki.openstreetmap.org/wiki/Maperitive">Maperitive</a> Kacheln berechnen, für größere Gebiete sollte man sich eine PostGIS samt Mapnik aufsetzen (Anleitungen dazu z.B. auf <a href="http://www.switch2osm.org"></a><a href="http://www.switch2osm.org/">http://www.switch2osm.org/</a>).</p>

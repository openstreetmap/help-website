<p>First select <em>File/Download from OSM...</em> or press the icon in the menu bar (with the green arrow pointing downwards). The <em>Slippy map</em> tab should now be selected.</p>
<p>Then use the mouse:</p>
<ul>
<li>right button to move the map</li>
<li>the scroll wheel to zoom</li>
<li><strong>the left button to do the actual selection</strong></li>
</ul>
<p>Then press <em>Download</em>.</p>

<p>A good way to diagnose connection problems might be to try a URL like this in a browser (click the link) : <a href="http://api.openstreetmap.org/api/0.6/map?bbox=-0.108157396316528,51.5073601795557,-0.107599496841431,51.5076406454029"></a><a href="http://api.openstreetmap.org/api/0.6/map?bbox=-0.108157396316528,51.5073601795557,-0.107599496841431,51.5076406454029"></a><a href="http://api.openstreetmap.org/api/0.6/map?bbox=-0.108157396316528,51.5073601795557,-0.107599496841431,51.5076406454029">http://api.openstreetmap.org/api/0.6/map?bbox=-0.108157396316528,51.5073601795557,-0.107599496841431,51.5076406454029</a></p>
<p>What does it tell you? Well if you get an error, then it tells you there's something definitely wrong with your internet access. If you see XML, then it hasn't helped you so much, but it still narrows the problem down a little. This is a worthwhile step to diagnose the problem and worth mentioning that you've tried this when asking people about it.</p>
<p>More technical details: What this actually is, is a direct request to the OpenStreetMap API from your browser, request an area of map data. It should give you back XML data like in the <a href="https://wiki.openstreetmap.org/wiki/.osm#Example">example here</a>. JOSM will do almost exactly the same. Only difference is the HTTP request will have a different user agent (<em>possibly</em> blocked by a firewall) and maybe you're requesting a larger area of the map.</p>

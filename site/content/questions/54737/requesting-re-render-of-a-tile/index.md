<p>As <em>workaround</em> you can try it over there: <a href="https://mc.bbbike.org/mc/?num=1&amp;mt0=mapnik">https://mc.bbbike.org/mc/?num=1&amp;mt0=mapnik</a> (it uses tiles from openstreetmap.org).</p>
<p><em>In general</em>, how-to get the browser's original right-click context menu:</p>
<ul>
<li>do a right click while pressing shift</li>
<li>or hit ESC after the right click</li>
<li>or do a double right click</li>
<li>or tell your browser to not respect scripts hijacking the right click menu (for Firefox: <code>dom.event.contextmenu.enabled</code> see <a href="http://kb.mozillazine.org/About:config_entries">http://kb.mozillazine.org/About:config_entries</a> )</li>
</ul>
<p>However, we do not get the clicked tile's (image) address via that original context menu... due to the way leaflet is programmed.</p>

<div class="answer-body">
<p>As said at <a href="/questions/60223">https://help.openstreetmap.org/questions/60223</a>, another way to get the tile URL is, with Firefox, to access the page information (which may be in the browser Tools menu or in the non-Javascript right-click menu, which you can open with Ctrl-Shift-right click).</p>
<p>The page information contains a Media tab with the URL of all images. It's not very handy, but it's feasible.</p>
<p>Of course it would be more sensible to have a "Request rendering" option somewhere.</p>
<p>However the best way may be to use the latest version of JOSM, where you may see the tile status or force rendering (right-click on the map).</p>

<div class="answer-body">
<p>The answer to this question very much depends on which GPS unit you have, for some it is easier, some it is possible and other you currently can't at all.</p>
<p>The easiest is probably if you have a Garmin device. For those, the native format of the software is mostly reverse engineered and and converters are written to create "garmin maps" from OpenStreetMap data. Additional information can be found in the <a href="https://wiki.openstreetmap.org/wiki/OSM_Map_On_Garmin">wiki</a>, which also contain pre-built maps that you only need to copy onto your device. Some of those maps also allow routing. Overall, Garmins are supported quite well, although some features might not fully work yet.</p>
<p>For TomTom's I don't think there currently is a way, as the format is not yet known and so it is impossible to write converters from OpenStreetMap data to the native format of TomTom's</p>
<p>Windows CE devices (which covers a large number of standard SatNav devices), again is a bit of a mixed bag. The native format in most cases is not covered and so you can't currently use OpenStreetMap data with the Software that currently runs on the device. However in most windows PNA devices, it is comparatively easy to get to the underlying windows system and start alternative programs, such as described in <a href="http://www.gpspassion.com/forumsen/topic.asp?TOPIC_ID=126303">the thread at gpspassion</a>.</p>
<p>Some of the routing software that can be run on WinCE devices is <a href="https://wiki.openstreetmap.org/wiki/Gosmore">Gosmore</a> or <a href="http://wiki.navit-project.org/index.php/Download_Navit#Navit_on_WinCE.2FWindows_Mobile">Navit</a>. You can even install a Java interpreter and run any of the <a href="https://wiki.openstreetmap.org/wiki/Software/Mobile_phones">programs for Mobile phones on it</a> like <a href="http://gpsmid.sourceforge.net/">GpsMid</a>. But non of the programs are quite as polished yet as the pre-installed software so you have to be prepared for some hackery in this case.</p>
<p>In future perhaps more devices will be supported natively but unless manufacturer's release their map formats, it is rather difficult.</p>

<p>To summarise the simplest part of the wiki for the most numerous users:</p>
<p>1: Get hold of a Garmin map file</p>
<p>2: Make sure it's named GMAPSUPP.IMG</p>
<p>3: Copy it to the Garmin folder of your Garmin's memory card (back up any existing GMAPSUPP.IMG file first!)</p>
<p>Some are routable, some are not (the download page will usually say). Some contain additional data (my UK map*, for example, also has contours), extra points of interest or formatting specific to that version. For more detail, the Garmin wiki page is: <a href="https://wiki.openstreetmap.org/wiki/OSM_Map_On_Garmin">https://wiki.openstreetmap.org/wiki/OSM_Map_On_Garmin</a></p>
<p>Try a few. ComputerTeddy's is very bright; Cloudmade's is Garmin-default style (and covers the while world, pick the bit you want: <a href="http://downloands.cloudmade.com">downloands.cloudmade.com</a>); mine is very busy. It's nice that you don't have to commit. :)</p>
<ul>
<li>Trumpet-blowing: my UK &amp; IRL map is at <a href="http://ravenfamily.org/andyg/maps">http://ravenfamily.org/andyg/maps</a></li>
</ul>

<p>If you have a Garmin you have a couple of download options, websits which will provide you with gmapsupp.img files which can be stored on your Garmin and rendered as searchable maps.</p>
<p>Pre built options ( that I use )</p>
<p><a href="http://garmin.openstreetmap.nl/">http://garmin.openstreetmap.nl/</a></p>
<p><a href="http://downloads.cloudmade.com/">http://downloads.cloudmade.com/</a> <em>EDIT: ... but be aware that those data files are <strong>outdated</strong> from end of 2011</em></p>
<p>Custom Maps are easy also. You can make them by using JOSM to download an area. Save the area as an OSM file and then use MKGMAP to create a gmpasupp.img using a command line similar to</p>
<p>java -jar mkgmap.jar --add-pois-to-areas --road-name-pois --ignore-osm-bounds name_of_file.osm</p>
<p>via the command of Windows based machines</p>

<p>I have been looking at local Police Stations to improve data such as address details, opening times etc and know of a few Police buildings that are not 'stations' so do not appear to warrant the amenity tag but not sure how to tag it correctly.</p>
<p>This example <a href="https://www.openstreetmap.org/node/311370565">here</a> is a 'tactical operations' unit (offices and garage facilities) which does not have facilities for 'where suspects and evidence are collected and processed' per the <a href="https://wiki.openstreetmap.org/wiki/Tag:amenity%3Dpolice">wiki</a> so doesn't seem to fit the amenity tag.</p>
<p>Another example is <a href="https://www.openstreetmap.org/node/312600575">here</a> which is the police headquarters and again is not a 'station' but is primarily an administrative building.</p>
<p>Should these have the amenity tag removed and have suitable name &amp; operator tags applied?</p>

<p>Good question, but the wiki description is woefully inadequate. I might want to visit a police station for my collection of mushrooms to be checked for safety, or to arrange for parking bays to be marked off for a delivery, or to report an item of lost property. I think the key property of the classic police station is that it has some public-facing aspect, but more and more are losing that aspect. Perhaps office=police, but I am not sure.</p>

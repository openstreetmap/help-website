<p>I am trying to use the JOSM import remote command to pass some osmxml to josm. I want the data to appear in a new layer to be used as visual reference areas in JOSM, rather than as data to be uploaded to OSM, so I am using upload='no'</p>
<p>My osm xml is at this url endpoint: <a href="http://tasking-manager-staging.eu-west-1.elasticbeanstalk.com/api/v1/project/1/tasks-as-osm-xml">http://tasking-manager-staging.eu-west-1.elasticbeanstalk.com/api/v1/project/1/tasks-as-osm-xml</a></p>
<p>I am using this url for the remote command:</p>
<p><a href="http://127.0.0.1:8111/import?url=http://tasking-manager-staging.eu-west-1.elasticbeanstalk.com/api/v1/project/1/tasks-as-osm-xml&amp;new_layer=true">http://127.0.0.1:8111/import?url=http://tasking-manager-staging.eu-west-1.elasticbeanstalk.com/api/v1/project/1/tasks-as-osm-xml&amp;new_layer=true</a></p>
<p>JOSM responds with a 200 OK response, but the command seems to have to have no effect in JOSM. I get a flicker of the loading bar, but no sign of the new layer or the data.</p>
<p>Can anyone offer any advice?</p>

<div class="question-body">
<p>Why does JOSM warn me that uploading my GPS traces to OSM is "considered harmful"?</p>
<p>I found a road that isn't on OSM, so I turned-on my phone's tracer and walked the length of the road, recording nodes every 5 seconds. At the end of the road, I saved the trace to a GPX file.</p>
<p>I transferred the GPX file to my computer, opened it in JOSM, right-clicked on it, and left-clicked on the "Convert to data layer" button. I then got a popup that said</p>
<blockquote>
<p>Upload of unprocessed GPS data as map data is considered harmful. If you want to upload traces, look here: <a href="https://www.openstreetmaps.org/traces">https://www.openstreetmaps.org/traces</a></p>
</blockquote>
<p>Unfortunately, the link didn't have any additional information on why this is harmful.</p>
<p>Why is it harmful for me to add my GPX data to OSM? What am I <em>supposed</em> to do to add a new road?</p>

<div class="answer-body">
<p>GPS traces often have a lot of noise in them and are <a href="https://wiki.openstreetmap.org/wiki/Accuracy_of_GNSS_data">limited in their accuracy</a>.</p>
<p>GPS traces should normally be manually redrawn to remove a lot of this "jitter". An ideal scenario would be to record multiple tracks over multiple days to try to "average out" any transient errors, but that is rarely practical as a starting point and a trace or two is normally all that is used at the start.</p>
</div>

<p>Hi.</p>
<p>Basically you can use your GPS trace has an overlay, to have an idea where the road is. You can even <a href="https://wiki.openstreetmap.org/wiki/Upload_GPS_tracks">upload it in a dedicated database</a> so that it will be available for anyone (the link you checked).</p>
<p>But to map a road, you want to create as few nodes as possible, connect with crossing roads, and so on, add relevant tags, which will not be easy if you import the GPS trace. Not impossible to clean up, but a lot of work for not much.</p>
<p>You may also want to activate a satellite imagery layer, and check if others have uploaded GPX tracks in the vicinity, it's the only way to know if your GPS unit is more or less accurate.</p>
<p>Hope this helps.</p>
<p>Regards.</p>

<p>Well, that's annoying. Did you check all the imageries available in JOSM ? Unfortunately most are from the same provider and have the same clouds... :-(</p>
<p>The best in this case would be to record GPS trace until out of the cloud, if possible. And as InsertUser said, make a few passes, in each direction.</p>
<p>My phone's GPS can easily be 100 meters off, usually in one consistent direction. Handheld GPS devices have usually better results, depending on the context (trees, buildings, etc.).</p>
<p>A lot of work for a single road you may think. Otherwise, just map the road as best as you can, with the help of your GPS trace, from memory (was there any bend here?), and tag a fixme on it describing the things to be corrected or as many others do add note=approximate. OSM is always a work in progress, when imagery will be available, you or others can improve the geometry of the road.</p>
<p>Another technique I found useful, was to map on the spot. Under the cover of trees, I mapped paths using Vespucci, stopping at every bend, trying to get my bearings, and so on. Quite long but rewarding in the end.</p>
<p>Best regards.</p>

<div class="answer-body">
<p>note that the alert says, as you quote it:</p>
<ul>
<li>Upload of <strong>unprocessed</strong> GPS data as <strong>map data</strong> is considered harmful</li>
</ul>
<p>the part which is harmful is contained in the two above highlighted sections.<br />
this is not the same as you summarized it:</p>
<ul>
<li>uploading [my] GPS traces to OSM</li>
</ul>
<p>uploading GPS traces is a good thing, if you upload them as GPS data, as traces, preferably with <a href="https://wiki.openstreetmap.org/wiki/Visibility_of_GPS_traces">visibility</a> that allows timestamps to be available.</p>
<p>uploading the same information as map data is a bad idea, because you're begging other mappers to clean up all more or less random errors contained in the points grabbed by your GPS device.</p>

<td><div class="item-right">
<div class="answer-body">
<p>The digitalglobe premium tiles (delivered via a mapbox server) in that area for zoom levels z19+ are just transparent images. In the <a href="https://github.com/osmlab/editor-layer-index/blob/gh-pages/sources/world/DigitalGlobePremiumImagery.geojson">imagery definition</a> it is set to max zoom 22 currently. I guess that is true for some other parts of the world.</p>
<p>I am not sure who could improve this situation.</p>
<ul>
<li>iD could discard such invalid transparent-only images and interpolate the lower zoom level imagery. Back in 2013 two issues (<a href="https://github.com/openstreetmap/iD/issues/1499">1</a>, <a href="https://github.com/openstreetmap/iD/issues/1983">2</a>) were not anymore worked on.</li>
<li>iD could give the user an option to use and interpolate a tile set from a specific zoom level.</li>
<li>Mapbox could just not serve an image for invalid zoom levels.</li>
<li>Mapbox could interpolate images for not-existent zoom levels in the source imagery and deliver those interpolations.</li>
</ul>
<p>The other OSM editor <a href="https://wiki.openstreetmap.org/wiki/JOSM">JOSM</a> also gets the transparent image as response from the mapbox server, but somehow notices and continues to display z18 tiles (and a message "no tiles at this zoom level").</p>
<p>My suggestion is to use other imagery in iD, use JOSM, or open a new issue for iD (note that this needs a third-party user account; please include a link to this discussion for reference).</p>
</div>

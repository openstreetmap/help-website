<div class="answer-body">
<p>Wisepilot isn't part of OpenStreetMap; it's a separate software application that just happens to use OSM data. If you want software support for it you'll have to <a href="http://www.appello.com/support/contact_us/">ask them</a>.<br />
</p>
<p>Incidentally, their website now has an <a href="http://www.appello.com/osm/">OpenStreetMap</a> page which contains a "How to become an editor" section. Errors reported in WisePilot do create anonymous notes in OpenStreetMap, so do take care when reporting issues that they're reported in the right place and apply to current OpenStreetMap data (rather than what might have loaded in the app). <a href="https://lists.openstreetmap.org/pipermail/talk/2014-August/070485.html">This mailing list</a> thread has some more information about the "notes" issue.</p>
</div>

<p>To get Navteq updates for Wisepilot you have to ask the manufacturer of <span>Wisepilot</span>. Write them, call them if you cannot find anything in their FAQ/Support site.</p>
<p>Adding OSM-based maps yourself to Wisepilot would only be possible for yourself if the used map format is known and not secret. I could not find info on our wiki page about this.</p>
<p>In case you did not know: OSM is just about the raw data – not really about finished maps. Likely the maps which are used with Wisepilot are made by the Wisepilot manufacturer – based on the OSM data.</p>

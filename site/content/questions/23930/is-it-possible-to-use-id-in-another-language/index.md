<div class="answer-body">
<p>iD (on osm.org) uses the language (if a translation is available) which you have <em>set in your osm.org user preferences</em> (go to <a href="http://www.openstreetmap.org/">http://www.openstreetmap.org/</a> , login if needed, click on your name in the top row, click on the settings link on the right, change the priority list of language codes so that your preferred language is first). <a href="https://www.transifex.com/projects/p/id-editor/">Those are the available language codes</a> (how over a language name to see the code in a small popup; <a href="https://github.com/systemed/iD/tree/master/dist/locales">those are already included/transferred into the latest iD code</a>)</p>
<p>The translations (<a href="https://github.com/systemed/iD/blob/master/CONTRIBUTING.md#translating">how to translate</a>) <a href="https://github.com/systemed/iD/blob/master/CONTRIBUTING.md#adding-new-strings-for-translation">apparently</a> <a href="https://github.com/systemed/iD/commits/master/dist/locales">get updated manually</a> every now and then (few weeks). That is the <em>latest</em> code. I guess that it takes some time more (maybe even some more weeks until a new <em>release</em> version is produced and pulled to the osm.org homepage) until those translations arrive on the osm.org homepage then (since osm.org's iD seems to load the translations from <a href="http://www.openstreetmap.org/assets/iD/locales/…">http://www.openstreetmap.org/assets/iD/locales/…</a> ).</p>

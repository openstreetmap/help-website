<p>Hey</p>
<p>I'm wondering how to tag a DSLAM outdoor cabinet like this one: see right side in the photo below. They're pretty common in Germany.</p>
<p><img src="//upload.wikimedia.org/wikipedia/commons/thumb/b/b3/Outdoor_DSLAM_Endstufe.JPG/640px-Outdoor_DSLAM_Endstufe.JPG" alt="Photo of an outdoor dslam" /><br />
<span class="small"><span>Photo by FEXX, PD</span></span></p>
<p>Some possible Tags I found:</p>
<ul>
<li><strong><code>man_made=MDF</code></strong> for Main Distribution Frame, but this one can't distinguish between outdoor/indoor, <span>Wiki</span></li>
<li><p><strong><code>man_made=street_cabinet</code></strong> + <strong><code>street_cabinet=telecom</code></strong> <em>or</em> <strong><code>street_cabinet=outdoor_dslam</code></strong> there was Proposal for a generic tag for street cabinets, but there isn't a straigt definition for DSLAMs , see <span>Wiki</span></p></li>
<li><p><strong><code>power=cable_distribution_cabinet</code></strong> + <strong><code>communication=cable_distribution_cabinet</code></strong> <em>or</em> <strong><code>communication=outdoor_dslam</code></strong> pretty often used, see <span>Wiki</span></p></li>
</ul>
<p>I don't know the technically differences that might be in the tags above, so it would be great to hear some recommendations or even other tags! At the moment the new street_cabinet tagging is most appealing to me.</p>
<p>Thanks in advance!</p>

<p>Sadly there is no kind of unified schema for telecommunication equipment as we have already for power=* stuff (compare for example <a href="https://wiki.openstreetmap.org/wiki/User:!i!/telecommunication).">https://wiki.openstreetmap.org/wiki/User:!i!/telecommunication).</a></p>
<p>There is a german redirect for "DSL Hauptverteiler", so I guess currently man_made=MDF is the right choice as there was for example <a href="https://wiki.openstreetmap.org/wiki/DE:Tag:man_made%3DMDF">an import</a> with this tag. Would be great, if one day we can bring all things together ... but tagging discussions can take a lot of time ...</p>

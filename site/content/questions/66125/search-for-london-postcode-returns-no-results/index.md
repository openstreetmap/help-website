<div class="answer-body">
<p>Postcode data in OSM is sparse, at best. This is because most OSM data is gathered by people either looking at aerial imagery or going out in the field looking at reality, and postcodes simply do not lend themselves to being mapped this way.</p>
<p>Commercial operators, like Google, can buy data sets from those who maintain them, such as the Royal Mail in the UK. OSM does not have the budget for this and, more importantly, one of the points (some would say <em>the</em> point) of OSM is to be freely distributable, and the licensing agreements for bought postcode data would almost certainly not allow that.</p>

<p>Well, it appears that no one has ever bothered to input any data for it into OpenStreetMap. See for yourself: <a href="http://overpass-turbo.eu/s/CrU">Database query for E12 5PG</a>. In fact very few postcodes have tagged at all in your area of interest: <a href="http://overpass-turbo.eu/s/CrZ">Database query for any postcode</a>. <p>Additionally, I believe that the search engine on openstreetmap.org only returns postcode areas (if they are mapped). It does not identify objects just based on postcode.</p>
<p>So it seems you would need to map any features with this postcode in OpenStreetMap yourself. That's how the map grows.</p>

<div class="answer-body">
<p>It is odd that this postcode does not return a result. Nominatim does use a complete dataset of postcodes for Great Britain which are <a href="https://github.com/openstreetmap/Nominatim/blob/9b7f0627eabba2aaeaf5d31b8a36a7dc2ff8d45a/data/gb_postcode_table.sql">stored separately</a> from OSM data. I believe the source is either ONS postcodes of Ordnance Suryey Code Point Open. Both data sets are open data, although Code Point Open had certain restrictions associated with it.</p>
<p>If a postcode is not present in the OSM data (and at present we have perhaps 150k or a little under 10% of the total), Nominatim used to display something like E12 5## and the viewport will be centred on the postcode centroid with a medium zoom level; now it shows the postcode at a higher zoom level (e.g. <a href="http://oscompare.raggedred.net/?zoom=18&amp;lat=51.55099&amp;lon=0.05994&amp;layers=BTFFTF">E12 5PQ</a>).</p>
<p>As the <a href="http://oscompare.raggedred.net/?zoom=18&amp;lat=51.55099&amp;lon=0.05994&amp;layers=BTFFTF">specific postcode is present</a> in the ONS dataset, I can only assume that this specific postcode was omitted from the imported data. I don't believe this list is regularly updated. I would suggest filing a specific bug report on github.</p>
<p><strong>Summary</strong> (if above is tldr): the postcode should be detected by Nominatim even though OSM coverage is sparse, because it holds a table of British postcodes from an external source.</p>
<p><a href="https://github.com/openstreetmap/Nominatim/blob/9b7f0627eabba2aaeaf5d31b8a36a7dc2ff8d45a/data/gb_postcode_table.sql">1</a>: <a href="https://www.openstreetmap.org/search?query=E12">https://www.openstreetmap.org/search?query=E12</a> 5PQ</p>

<div class="answer-body">
<p>Are you trying to do this as a user, or make a website which calculates this for users?</p>
<p>Is this what you're trying to get? <a href="https://graphhopper.com/maps/?point=39.582936%2C2.671051&amp;point=39.760519%2C2.868805&amp;locale=cs-CZ&amp;vehicle=bike&amp;weighting=fastest&amp;elevation=true&amp;layer=Omniscale">https://graphhopper.com/maps/?point=39.582936%2C2.671051&amp;point=39.760519%2C2.868805&amp;locale=cs-CZ&amp;vehicle=bike&amp;weighting=fastest&amp;elevation=true&amp;layer=Omniscale</a></p>
<p>If doing this as a developer: elevation is not really a part of the OSM data. However, there are projects combining various free sources (OSM for maps and SRTM for elevation). Some of them are specifically geared towards cycle routing, such as <a href="https://www.cyclestreets.net/">https://www.cyclestreets.net/</a> (code is at <a href="https://github.com/cyclestreets">https://github.com/cyclestreets</a> )</p>
<p>Apparently Graphhopper also makes this possible: <a href="https://graphhopper.com/blog/2015/04/21/elevation-data-and-openstreetmap/">https://graphhopper.com/blog/2015/04/21/elevation-data-and-openstreetmap/</a></p>

<div class="answer-body">
<p>On which map? There are many <span>OSM-based maps and services</span>. For example on osm.fr's humanitarian map you can see names of post offices. Example post offices: <a href="http://www.openstreetmap.org/node/293199375#map=19/49.90587/8.58514&amp;layers=H">hum1</a> vs. <a href="http://www.openstreetmap.org/node/293199375#map=19/49.90587/8.58514">mapnik1</a>, <a href="http://www.openstreetmap.org/node/300088477#map=19/50.05668/8.29521&amp;layers=H">hum2</a> vs. <a href="http://www.openstreetmap.org/node/300088477#map=19/50.05668/8.29521">mapnik2</a>.</p>
<p>So, if you were speaking of osm.org's standard map (often called "mapnik"), then it seems that names for post offices are not shown, yes. I am not sure if intentional or not (only the developers might know) – but you could suggest (see <a href="/questions/6241/change-request-adding-a-feature-to-the-default-osm-map-style">1</a> or <a href="/questions/1107/can-i-influence-what-gets-rendered-in-the-main-mapnik-layer">2</a>) to <a href="/questions/6521/missing-amenity-name-in-mapnik-bug-or-feature">try</a> (if space permits) to place a name label for post offices on mapnik, too.</p>

<div class="answer-body">
<p>The correct approach to private roads is normally to map them and add the relevant <a href="https://wiki.openstreetmap.org/wiki/Key:access">access tag</a> indicate the public aren't allowed on them. An exception to this may be things that are so well hidden they cease to be verifiable.</p>
<p>There is now a web based <a href="://wiki.openstreetmap.org/wiki/Revert_UI">Revert UI</a> for undoing changesets, but my recommendation would still be to download JOSM and use its <a href="https://wiki.openstreetmap.org/wiki/JOSM/Plugins/Reverter">Reverter plugin</a> as that lets you review the changes before upload.</p>

<p>While I'd agree that normally the best way to handle roads and tracks that have been mapped on private property is to ensure that they are mapped as "access=private" rather than just deleting them (as suggested by "insertuser"), I'd actually try and have a discussion with the person who deleted the data first before just reverting their deletion.</p>
<p>Probably the best way to do that is via a changeset discussion - type something in the "discussion" box <a href="https://www.openstreetmap.org/changeset/118159013">here</a> and click "comment" (however that translates into Italian). The mapper concerned <a href="http://resultmaps.neis-one.org/osm-discussion-comments?uid=3151555">does tend to reply</a> to these.</p>

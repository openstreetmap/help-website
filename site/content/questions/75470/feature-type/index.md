<div class="answer-body">
<p>Administrative regions are mapped with the <a href="https://wiki.openstreetmap.org/wiki/Tag:boundary%3Dadministrative"><code>boundary=administrative</code></a> tag on <a href="https://wiki.openstreetmap.org/wiki/Relation:boundary">boundary relations</a> that reference <a href="https://wiki.openstreetmap.org/wiki/Way">ways</a> that mark all or part of a boundary between two neighbouring areas. To allow consistency of tagging worldwide the administrative boundary type is indicated by a numeric <code>admin_level</code> whose national 'translations' are indicated on the page for <code>boundary=administrative</code>.</p>
<p>The use of relations means that there need only be one way per border rather than multiple overlapping ways, and avoids some issues that occur with really long ways. For editing relations of this sort use of one of the more advanced pieces of <a href="https://wiki.openstreetmap.org/wiki/Editors">editing software</a> is highly recommended (e.g. JOSM).</p>
<p>As with any addition please make sure that the source you use is <a href="https://wiki.openstreetmap.org/wiki/Copyright">suitable for use in OSM</a>.</p>

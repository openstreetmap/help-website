<div class="answer-body">
<p>There is a problem where OSM's "map what is on the ground" philosophy conflicts with the land owner/manager's rights. Specifically, if someone trespasses and then uploads a GPS track to OSM there are mappers who will add it to the map. That is then picked up by outdoor apps which then attracts more trespassers. The path becomes a self-fulfilling prophecy.</p>
<p>There is a way to break the cycle: Instead of deleting the path, change its tags. Change the highway=path to something like not:highway=path. Add access=no or access=private. Add an informal=yes (indicates non-authorized trail). Add a note=* saying what the situation is. By doing this you will reduce the possibility that a mapper using GPS tracks will add it back in. And with that tagging there shouldn't be any outdoor exercise/mapping apps that will show it.</p>
</div>

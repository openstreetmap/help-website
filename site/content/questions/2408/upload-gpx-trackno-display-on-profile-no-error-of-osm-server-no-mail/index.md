<p>[SOLVED]</p>
<p>1) I use a Holux GR-245</p>
<p>2) I use <a href="http://BT747.org">BT747.org</a> to retrieve my traces</p>
<p>3) It transform in GPX track (standard, not the OSM specific GPX format)</p>
<p>4) I use GPX Editor, a freeware to cut no-logical traces (poor signal)</p>
<p>the think is that the specific "head" of GPX is too long (title #00001# 010407-13:34 for example...BEFORE the NMEA tracks).</p>
<p>perhaps a <a href="http://BT747.org">BT747.org</a> feature but OSM server reject this with no indicator...to debug the GPX track.</p>
<p>i have uploaded 4 GPX track and it's ok, now...</p>

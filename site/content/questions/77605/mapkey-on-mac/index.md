<p>This is a known issue with Safari. Unfortunately Leaflet (the map-loading library used by OSM) is currently broken in Safari and there is no timescale for resolution.</p>
<p>See <a href="https://github.com/Leaflet/Leaflet/issues/7255">https://github.com/Leaflet/Leaflet/issues/7255</a> , <a href="https://github.com/openstreetmap/openstreetmap-website/issues/2811">https://github.com/openstreetmap/openstreetmap-website/issues/2811</a></p>

<div class="answer-body">
<p>The inability to load the map key sounds like a bug that should be reported on the <a href="https://github.com/openstreetmap/openstreetmap-website">Github</a>.</p>
<p>Having said that, the default render doesn't show road surface. I think the HOT layer does distinguish between paved and unpaved, but it's quite subtle.</p>
<p>Your best bet is probably to look up the specific surface keys you are interested in <a href="https://wiki.openstreetmap.org/wiki/Key:surface">on the wiki</a> and devise an <a href="https://wiki.openstreetmap.org/wiki/Overpass_turbo">overpass turbo</a> query to retrieve and show them.</p>
</div>

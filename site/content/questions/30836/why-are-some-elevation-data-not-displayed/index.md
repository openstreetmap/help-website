<p>In Potlatch, I added 4 peaks with these tags:<br />
</p>
<p>natural = peak<br />
name = the name<br />
ele = a number<br />
</p>
<p>After saving and closing, I checked the map with www.openstreetmap.org - 3 of the peaks displayed as expected:<br />
</p>
<p>a triangle<br />
the name<br />
a number<br />
</p>
<p>but one of the peaks displayed only<br />
</p>
<p>a triangle<br />
the name<br />
- the elevation number is missing.<br />
</p>
<p>You can see the 4 peak here:<br />
</p>
<p><a href="https://www.openstreetmap.org/#map=14/55.1656/9.9929">https://www.openstreetmap.org/#map=14/55.1656/9.9929</a><br />
</p>
<p>the 3 upper peaks with elevation displayed - the 1 lower peak ('Toppen') without elevation. Other zoom-levels have the same problem.<br />
</p>
<p>I have double-checked the entered data - there is no difference between the tags in 4 peaks.<br />
</p>
<p>In other locations on the map, I have found many other examples of peaks with elevation data that are not displayed for no apparent reason.<br />
</p>
<p>Why this missing display of elevation?</p>

<p><a href="https://github.com/gravitystorm/openstreetmap-carto/issues/342">There was a bug</a> in the <a href="https://github.com/gravitystorm/openstreetmap-carto">openstreetmap-carto</a> stylesheets, which meant that elevation clashed with names with long descenders (e.g. the letter p). <a href="https://github.com/gravitystorm/openstreetmap-carto/pull/343">It has been fixed</a>, and the map will update after the next release of the stylesheets.</p>

<p>Could be a bug in the renderer or its database. Or the renderer has some reason, unknown to us, for not displaying it. You can try to <a href="https://github.com/gravitystorm/openstreetmap-carto/issues">create a new issue at the openstreetmap-carto github project</a> but it will require a separate GitHub account and probably is not worth the effort (and possibly even the wrong component).</p>
<p>The important information is: The data you entered is fine. And the <a href="https://www.openstreetmap.org/#map=14/55.1656/9.9822&amp;layers=H">humanitarian layer</a> for example displays the elevation of all four peaks (you can switch the layer via the sidebar on the right).</p>

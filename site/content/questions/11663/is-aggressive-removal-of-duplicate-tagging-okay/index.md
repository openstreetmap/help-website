<div class="question-body">
<p>As more information comes in, one more and more comes across cases where the same information has been either spread over several different data points (e.g. a church building is mapped, and a separate node bears the <code>amenity</code> and <code>name</code> tags). I have in fact come across several cases where the same, or some information is spread over <strong>three</strong> pieces of data: an area, a building, and a node (usually bearing all the same <code>name</code>, but an area and node bearing duplicate <code>amenity</code> tags is also common).</p>
<p>I have been removing such duplication with extreme prejudice whenever I run across it, but I'm vaguely wondering if it is a sound approach. Any thoughts?</p>

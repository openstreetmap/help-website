<td><div class="item-right">
<div class="answer-body">
<p>Add each individual lean-to as a node and tag it with <a href="https://wiki.openstreetmap.org/wiki/Tag:amenity%3Dshelter">amenity=shelter</a> + <a href="https://wiki.openstreetmap.org/wiki/Tag:shelter_type%3Dlean_to">shelter_type=lean_to</a>.</p>
<p>Explicitly searching for camp sites with this type of shelter will be a challenge that has to be solved by the end-user application. This is how OSM's data model currently works.</p>

<p>If campsites are mapped as ways and the shelters are inside their perimeters, it should be easy enough. If campsites are mapped as nodes, a little harder.</p>
<p>Here's a quick overpass turbo query that searches for campsites that have a shelter within 25 meters:</p>
<p><a href="http://overpass-turbo.eu/s/z2A">http://overpass-turbo.eu/s/z2A</a></p>
<p>Looks like most of them are ways. I didn't restrict shelter_type.</p>
<p>A lot of these camp sites are tagged with a shelter= (or shelters=) giving a count of shelters. But this (shelter as a key) isn't documented in the wiki except as part of a bus stop.</p>

<div class="question-body">
<p>In general, how can I find a mapping between the <a href="https://osmand.net/docs/user/map/point-layers-on-map/#poi-types">POI Types</a> (Categories) in OsmAnd and the <a href="https://wiki.openstreetmap.org/wiki/Tags">OSM tags</a> that it's actually translating-to in the OSM dataset?</p>
<p>I'm trying to buy a zipper. On OsmAnd, the closest thing to "a store that sells zippers" that I can find in the "Categories" section of POIs is "Drapery Store". But I'm not sure if a store that sells curtains is going to sell zippers.</p>
<p>If I check the OSM wiki, then I see that there's <a href="https://wiki.openstreetmap.org/wiki/Tag:shop%3Dsewing">a sewing shop that could be tagged</a> in a number of ways:</p>
<ol>
<li>shop=sewing</li>
<li>shop=fabric</li>
<li>shop=wool</li>
<li>shop=craft</li>
<li>shop=haberdashery</li>
<li>craft=sewing</li>
</ol>
<p>I'm not sure if searching for a "Drapery store" in OsmAnd will return all of the above or not. How can I know?</p>
<p>But I'm not asking specifically about my query with the drapery store and the finite list of tags enumerated above. I'm asking a more general question:</p>
<p>How can I translate the "POI Type" (or "Categories") in OsmAnd to their respective, actual tags in the OSM dataset?</p>

<div class="answer-body">
<p>If you've enabled the <a href="https://osmand.net/docs/user/plugins/osm-editing">"OpenStreetMap editing" plugin</a> then you can attempt to <a href="https://osmand.net/docs/user/map/map-context-menu/#-create--modify-poi">create a new POI</a> then you can see what tags are used by OsmAnd for a given POI Type (Category).</p>
<ol>
<li><p>First, choose some point on the map and then click "Create POI"</p></li>
<li><p>Next, enter something that you'd like to "translate" to tags in the "POI Type" field (eg "Drapery store")</p></li>
<li><p>Finally, click the "Advanced" tab.</p></li>
</ol>
<p>If, for example, you had entered "Drapery store" into the "POI Type" field, then you should see one name=value tag set</p>
<blockquote>
<p>shop=fabric</p>
</blockquote>
<p>Therefore, <strong>it appears that <code>Drapery store</code> in OsmAnd translates to <code>shop=fabric</code> tags</strong> in the OSM dataset.</p>
<p>I have no idea how to do this without the OpenStreetMap editing plugin.</p>

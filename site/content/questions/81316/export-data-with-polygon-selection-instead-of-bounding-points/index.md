<div class="answer-body">
<p>For relatively small scale usage I would use overpass or overpass-turbo, you can describe a <a href="https://wiki.openstreetmap.org/wiki/Overpass_API/Overpass_QL#By_polygon_.28poly.29">bounding polygon</a> as a series of co-ordinate pairs, so probably most useful if you are using the same poly again &amp; again or it has a relatively small number of nodes. If the area is already defined in OSM then this can be used directly in overpass queries (see the documentation for examples).</p>
<p>I normally don't recommend Osmium directly because on some platforms there is quite a long dependency chain for compilation, and therefore it's not necessarily the tool to fix a little problem if not already in use. Most OSM ETL-style tools (osmconvert, osmosis, osmium-tool) will take a <a href="https://wiki.openstreetmap.org/wiki/Osmosis/Polygon_Filter_File_Format">polygon file</a> and allow you to clip a larger dataset by that polygon file. Both QGIS &amp; JOSM allow the creation of OSM style polygon files directly. Persnally I find <a href="https://wiki.openstreetmap.org/wiki/Osmconvert">osmconvert</a> satisfactory for this type of task.</p>

<div class="answer-body">
<p>It is certainly possible to use OpenHistoricalMap <strong>BUT</strong> it is would be a <strong>major</strong> project to create a streetmap of the UK for 1911. There are short-cuts, see my post on <em>City Stripping</em>: <a href="http://sk53-osm.blogspot.co.uk/2014/12/city-stripping-building-historical-road.html.">http://sk53-osm.blogspot.co.uk/2014/12/city-stripping-building-historical-road.html.</a></p>
<p>A second problem is that ideally you need a suitable geocoding engine to work from the census data to relate it to the map. For some places, notably central Edinbugh, existing OSM data is entirely suitable for geocoding historical data (see the MESH project), but this is unlikely to be true for the country as a whole.</p>
<p>The way I would approach this is to select a single current local authority and see how much work is involved in taking current OSM data and adapting it to the situation in 1911. Fairly recently I did this for Helsinki in 1907, which I think took me 3-4 hours altogether. Once one has this kind of baseline one can estimate a) how much work is required, and b) whether the approach as a whole will work.</p>
<p>Note you will need a geolocated street-level gazetteer for 1911 anyway: OHM is one way of creating such data. Because OHM uses the OSM toolset productivity can be higher than with other approaches; and much higher where existing OSM data can be used as the basis.</p>

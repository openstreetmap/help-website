<p><a href="https://wiki.openstreetmap.org/wiki/Node">Nodes</a> are only one of <a href="https://wiki.openstreetmap.org/wiki/Elements">three types</a> of OpenStreetMap data, and rarely if ever the data type that has the most visual impact on a map. The <a href="https://wiki.openstreetmap.org/wiki/Key:highway">highway key</a> has some uses on nodes but it is mostly used on other elements.</p>
<p>If you edit your post (or comment) to add more information about your process and software you are trying to use to create a map we may be able to offer better help, otherwise we can only really provide <a href="https://wiki.openstreetmap.org/wiki/Using_OpenStreetMap">generic links</a> to more information.</p>

<p>There is software specifically aimed at map creation that will certainly do a prettier job than Matlab (although the preview on <a href="https://www.mathworks.com/matlabcentral/fileexchange/35819-openstreetmap-functions">this package</a> looks better than I'd have expected). If you're not trying to incorporate it into something else Matlab based you may want to look at the software listed <a href="https://wiki.openstreetmap.org/wiki/Rendering">here</a>. There are also convertors/plugins for more mainstream <a href="https://wiki.openstreetmap.org/wiki/GIS_software">GIS</a> packages (including open source ones) that (in theory) can be made to do almost anything map related you can think of. Finally if you would just like a method to take a map and delete out anything that isn't a road, some extracts services like <a href="https://extract.bbbike.org/?lang=en">bbbike</a> will give you an SVG file for your area of interest that can be stripped down without leaving behind ugly artifacts.</p>

<td><div class="item-right">
<div class="answer-body">
<p>Personally I use the OpenJDK version of Java (installed using the "Chocolatey" package manager) and have had no problem using JOSM with that.</p>
<p>Issues that have been observed with JOSM that are related to issues in Java (various platforms and versions) can be found <a href="https://josm.openstreetmap.de/wiki/JavaBugs">here</a>.</p>
<p>OpenJDK is also recommended by <a href="https://lists.openstreetmap.org/pipermail/talk/2021-April/086395.html">this</a> mailing list thread, although as noted <a href="https://lists.openstreetmap.org/pipermail/talk/2021-April/086436.html">here</a> it's actually still an Oracle build; just one with a licence less likely to bite you in the posterior. Much of the rest of that thread is advanced muppettry and tinfoilhattery and can be ignored, but Clifford's recommendation is entirey sensible.</p>
<p>Non-Oracle builds are available. <a href="https://bell-sw.com/pages/unified-java-runtime/">Bell-sw's</a> is one, <a href="https://www.microsoft.com/openjdk">Microsoft's</a> is another. Azul is another, but they want their site visitors to "<a href="https://www.azul.com/products/core/">contact sales</a>", so probably isn't an option here.</p>
<p>In each case the software you're getting is basically the same; just compiled on someone else's computer. The licence terms and the associated support will vary.</p>

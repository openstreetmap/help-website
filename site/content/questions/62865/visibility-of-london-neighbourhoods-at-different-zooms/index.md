<div class="answer-body">
<p>I suspect that what is happening is that <a href="https://www.openstreetmap.org/node/201212">Muswell Hill</a> is not appearing at zoom 13 because it's a place=suburb, and there isn't room for it after the place=suburb for <a href="https://www.openstreetmap.org/node/3904886817">Fortis Green</a> has been displayed just to the west. The software that decides where to place text is ultimately <a href="https://wiki.openstreetmap.org/wiki/Mapnik">Mapnik</a>, though the map style describes how things should be displayed (if there's room).</p>
<p>So to answer the question - yes, there are various settings that determine what gets displayed (size of text, that sort of thing) but there's nother that chooses between (in this case) Fortis Green and Muswell Hill - that's just random luck which happens to get shown first. Wood Green seems to exist only as a <a href="https://www.openstreetmap.org/node/1825273060#map=15/51.6013/-0.1139">station</a> not as a suburb.</p>
<p>There isn't really any solution to the problem, since you can't have "every suburb displaying more prominently than anything else". If there's not room for two near each other, one will lose out.</p>

<p>Indeed it's not clear. You can test it by <a href="https://wiki.openstreetmap.org/wiki/Standard_tile_layer#Contributing_and_installing">installing testing environment for osm-carto</a> with a data from this area and remove some layers to see when it might start showing, but it's quite nitpicking activity and it's also possible that it's excluded by the rule that it'd be just too close to the Crouch End label (which you can still test by downloading data in .osm format and just remove/edit things in the text/XML editor before importing to a database).</p>
<p>There are no limits in what you can add to the OSM database as long as it's something following our rules (see <a href="https://wiki.openstreetmap.org/wiki/Good_practice">https://wiki.openstreetmap.org/wiki/Good_practice</a> ). Suburbs are surely something we welcome to be added and edited.</p>

<div class="question-body">
<p>I primarily use JOSM as my editor. It's a great bit of software.</p>
<p>I started out on OSM using iD, as do many people. One of the very few things I miss from iD is how the similar menu item (called "square", I think) allows for a few different angles other than 90° (like 45°). This makes it easy to get the shape of buildings with a combination of 90° and 45° corners right.</p>
<p>I know the <code>draw nodes</code> tool has a fixed angle feature when drawing, but this doesn't work as well, because any error in the original couple of nodes is continued throughout the trace, while the orthogonalize menu item "averages out" all the errors, to make the angles 90°, so provides better results.</p>
<p>Is there any way to replicate this behaviour with JOSM? A plug-in maybe?</p>

<p>Hi Keith, youll be familiar with the Q key in JOSM ? It bends all corners in the object up to squares. And you have to cut all other non 90 or 100 degree corners and after Q connect them again, but JOSM will correct you if so when you did not. If its a trapezium like shape make it square disconnect one side only and move that piece to the right position and connect again.</p>

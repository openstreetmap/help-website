<p>From memory the OpenStreetMap layer is built into MapInfo (newer versions only), but you can add it yourself with these two files:</p>
<p>OpenStreetMap.tab:</p>
<pre><code>!table
!version 1050
!charset WindowsLatin1
&#10;
Definition Table
  File &quot;OpenStreetMap.xml&quot;
  Type &quot;TILESERVER&quot;
  CoordSys Earth Projection 10, 157, &quot;m&quot;, 0 Bounds (-20037508.342789244, -20037508.343038857) 
       (20037508.342789244, 20037508.343038857)
ReadOnly</code></pre>
<p>OpenStreetMap.xml:</p>
<pre><code>&lt;?xml version=&quot;1.0&quot; encoding=&quot;utf-8&quot;?&gt;
&lt;TileServerInfo Type=&quot;LevelRowColumn&quot;&gt;
    &lt;Url&gt;https://a.tile.openstreetmap.org/{LEVEL}/{ROW}/{COL}.png&lt;/Url&gt;
    &lt;MinLevel&gt;0&lt;/MinLevel&gt;
    &lt;MaxLevel&gt;19&lt;/MaxLevel&gt;
    &lt;TileSize Height=&quot;256&quot; Width=&quot;256&quot; /&gt;
    &lt;AttributionText Font=&quot;Font (&quot;Tahoma&quot;,257,8,16777215,0)&quot;&gt;© OpenStreetMap contributors&lt;/AttributionText&gt;   
&lt;/TileServerInfo&gt;</code></pre>

<p>Thank you! It works. I had to fiddle with the XML a bit, as I was getting error messages when trying to open the TAB file. The error message stated that it was missing whitespace where you see "Tehoma" in line 7. After some trial and error, I found it didn’t like the quotation marks around Tehoma. Once those were removed, that error message no longer showed up. Instead, it choked on the copyright symbol on that same line. I removed that, and now it’s working fine.</p>
<p>That line now looks like this in my copy:</p>
<p>&lt;attributiontext font="Font (Tehoma,257,8,16777215,0)"&gt; OpenStreetMap contributors&lt;/attributiontext&gt;</p>
<p>This is running under Windows 10, using MapInfo 12.0.0 Build 16.</p>

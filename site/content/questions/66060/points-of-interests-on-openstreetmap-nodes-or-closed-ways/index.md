<div class="question-body">
<p>Most of the places (cafés, restaurants and the like) I am working with on OpenStreetMap are <a href="https://wiki.openstreetmap.org/wiki/Node">nodes</a> (like <a href="https://www.openstreetmap.org/node/5919670289">this one</a>).</p>
<p>However I've recently seen cafés (like <a href="https://www.openstreetmap.org/way/492846792">this one</a>) being mapped as <a href="https://wiki.openstreetmap.org/wiki/Way#Closed_way">closed ways</a> - more specifically as <a href="https://wiki.openstreetmap.org/wiki/Area">areas</a>.</p>
<p>Areas seem to make sense: a café or restaurant is never a single point in space but a certain geographical area.</p>
<p>However, the <a href="https://wiki.openstreetmap.org/wiki/ID">OpenStreetMap default browser editor</a> recommends:</p>
<ul>
<li>a <strong>point (node)</strong> to "add restaurants, monuments, postboxes or other points to the map"</li>
<li>a <strong>line (way)</strong> to "add roads, streets, ways, canals or other lines to the map"</li>
</ul>
<p>When I contribute new places to OSM, should I add a node or a way?</p>
<p>This is key when using the <a href="https://wiki.openstreetmap.org/wiki/Overpass_API">Overpass API</a>, where nodes and ways are treated very differently.</p>

<div class="answer-body">
<p>It's really up to you. If you can discern the building's outline from the satellite imagery and you are willing to draw it as a closed way, feel free to use that method.</p>
<p>The only time this isn't a good idea is if the building contains other shops or cafes. Then the node approach is better because you can add nodes for the other POIs inside the building. As long as you're communicating the existence of the POI, be it a monument, bar, supermarket, whether one uses a node or way to show it isn't crucially important.</p>
<p>Some objects, like the postbox example you gave, are probably always better mapped as nodes as most of them occupy such a small space they cannot be drawn as ways.</p>

<p>Dave</p>

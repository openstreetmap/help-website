<td><div class="item-right">
<div class="answer-body">
<p>First of all, OpenStreetMap's "API" is probably not what you are thinking of. Our API sends vector map data back for <em>editing purposes only</em> in XML format. If you want an API for displaying map tiles within a web view, like that for Google Maps, you should look at <a href="http://leaflet.cloudmade.com/">Leaflet</a>, <a href="http://openlayers.org/">OpenLayers</a> or <a href="http://mapstraction.com/">Mapstraction</a>. If you want an API for incorporating into an Android app, see <a href="https://labs.ericsson.com/developer-community/blog/openstreetmap-android">Ericsson's OSM API</a>, <a href="http://wiki.openstreetmap.org/wiki/Mapsforge">Mapsforge</a>, or <a href="http://www.nutiteq.com/android-mapping-api-sdk">Nutiteq</a>. I'm guessing that Mapsforge might be the most useful app for your offline city maps.</p>
<p>Secondly, OpenStreetMap does not provide free server resources for your app. We provide the data, but with the intention that other people will redistribute it. Your app must not access our servers except for very very limited use. Rather, you should find a third party that redistributes OSM data, whether as tiles (such as <a href="http://developer.mapquest.com/web/products/open/map">MapQuest Open</a>) or vector data. Better still, you can download the data yourself (from <a href="http://planet.openstreetmap.org/">planet.openstreetmap.org</a> or a mirror site) and host it on your own servers. (To find out the restrictions on data and tiles served from OSM's servers, see the <a href="http://wiki.openstreetmap.org/wiki/Tile_usage_policy">Tile Usage Policy</a> and <a href="http://wiki.openstreetmap.org/wiki/API_usage_policy">API Usage Policy</a>.)</p>
<p>Finally, for the copyright requirements (attribution and 'share alike'), see our clear <a href="http://www.openstreetmap.org/copyright">copyright page</a> and <a href="http://wiki.openstreetmap.org/wiki/Legal_FAQ">Legal FAQ</a>.</p>

<div class="answer-body">
<p>One very important thing is to <em>not</em> download the tiles from <a href="http://osm.org">osm.org</a> onto the android phones for offline use in an app. This has caused a lot of trouble for OSM in the past. If you create a good app it will cause a huge load on the OSM servers, which are primarily there for <em>editing</em> the map data. The policy is here: <a href="http://wiki.openstreetmap.org/wiki/Tile_usage_policy">http://wiki.openstreetmap.org/wiki/Tile_usage_policy</a></p>
<p>If you want the map tiles you should use the data and render your own tiles hosted on your own server, or find a commercial source for these tiles. You can provide an app without map tiles, using the data to render a map on the phone.</p>
<p>It is very good that you ask before you write your app.</p>

<p>Yes, you can use OSM data for commercial purposes, as long as you retain attribution (in other words, your app should clearly state that the data comes from OSM). These are <a href="http://wiki.openstreetmap.org/wiki/Legal_FAQ">FAQs</a>.</p>
<p>While you're at it, if you do make money from OSM, consider <a href="http://wiki.openstreetmap.org/wiki/Donate">donating</a> a percentage back to OSM. This is by no mean an obligation, but should be seen as a win-win for commercial users.</p>

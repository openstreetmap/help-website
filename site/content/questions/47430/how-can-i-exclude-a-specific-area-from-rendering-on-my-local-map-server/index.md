<div class="answer-body">
<p>Assuming you are using the standing osm2pgsql/postgis/mod_tile approach, one simple solution is to just delete the data from your database. You can write a postgis query to delete everything a specified polygon, in essense, just cut it out.</p>
<p>If you are doing data updates (with osmosis) this might cause problems. If you want to keep the data up to date, another approach could be to save the area you don't want (Maharashtra) as a shapefile, and then configure the mapnik style to draw a white (or whatever other colour) polygon in that area. The resultant image will have that are "blocked out". Regular data updates should work in that case.</p>

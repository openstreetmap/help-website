<p>Whilst it's perhaps overkill for what you want, it's perhaps worth mentioning that the resources on the <a href="http://switch2osm.org/">switch2osm</a> site should work for you too. You could <a href="http://switch2osm.org/serving-tiles/">set up a tile server</a>, but instead of loading data from OSM load the data that you've edited. You could then use one of the <a href="http://switch2osm.org/using-tiles/">using tiles</a> examples to create a slippy map on your website pointing at your tile server.</p>

<p>A tile server is a computer where you can access map tiles (little images, when viewed together that form a map). Then, on an other computer, you can have a website that stitches those tiles together to a draggable map.</p>
<p>One tile looks like this: <a href="http://a.tile.openstreetmap.org/15/16663/10976.png">http://a.tile.openstreetmap.org/15/16663/10976.png</a></p>
<p>If you render your own tiles, you don't need to make it publicly accessible.</p>

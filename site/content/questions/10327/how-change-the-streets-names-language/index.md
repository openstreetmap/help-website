<p>Most straightforward option is to use <a href="http://toolserver.org/~osm/locale/en.html">toolserver's locale-specific maps</a> which shows the english labels if available (just edit the link if you want a different language). Of course this only works if the object has a "name:en" tag; otherwise the local-language "name" tag will be used.</p>

<p>I always put and encourage the English names to be used in "name", if a tag is in a non English speaking country, add the local name with the local language tag. This is tricky though, some English names aren't appropriate if a place has no English name but Welsh locations should be in English for example.</p>
<p>eg SHEFFIELD:</p>
<p>name=Sheffield</p>
<p>name:fa=شفیلد</p>
<p>name:ja=シェフィールド</p>
<p>Amongst many others. That means the default English interface OSM always shows in English but with <a href="http://toolserver.org/~osm/locale/en.html">http://toolserver.org/~osm/locale/en.html</a> can show the local name when the local language is selected.</p>
<p>I wouldn't advise changing the name of Lyon which in American is spelt with an S at the end.</p>

<div class="comment-text">
<p>Bad suggestion. The "name" tag is only for the local language (<a href="https://wiki.openstreetmap.org/wiki/Names#Localization).">https://wiki.openstreetmap.org/wiki/Names#Localization).</a> This may become tricky for places which have multiple local languages, but this is an issue solved on a case-by-case basis by the local mapper. "name:xx" tags are easy in comparison: just add as many as you can.</p>
<p>English does not have a special status in OSM. The default OSM rendering is not (for better or worse) an "English" rendering but a local rendering. Defaulting to english in the name tag would be, in a way, tagging for the renderer.</p>
</div>

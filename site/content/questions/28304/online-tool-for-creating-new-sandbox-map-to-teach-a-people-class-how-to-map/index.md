<div class="answer-body">
<p>One possibility might be to use the development server at <a href="http://api06.dev.openstreetmap.org/">http://api06.dev.openstreetmap.org/</a> . It will allow people to get familiar with editors etc., but they won't see the results of their editing on any rendered map anywhere (unless there's a map already rendered from that data somewhere that I'm not aware of).</p>
<p>I've used it a few times when setting up test data for questions and answers on this site, and I've also used it to have a play with the new editors away from live data.</p>
<p>(thinking aloud here...)</p>
<p>If you wanted to see the results of people's editing on an OSM-style slippy map someone could perhaps use a <a href="https://wiki.openstreetmap.org/wiki/API_v0.6#Retrieving_map_data_by_bounding_box:_GET_.2Fapi.2F0.6.2Fmap">map call</a> against the dev API to extract the data that you're interested in, and feed it into a <a href="https://switch2osm.org/manually-building-a-tile-server-16-04-2-lts/">switch2osm</a>-style rendering database. Setting that up wouldn't be straightforward if you've not done it before, but it shouldn't be that complicated.</p>
<p>(editing this answer to add a bit more detail since this question has popped up again)</p>
<p>One way of doing it is to set JOSM to download from the dev API - set the API in there to <a href="http://api06.dev.openstreetmap.org/api">http://api06.dev.openstreetmap.org/api</a> . Try and download the biggest area that the API will let you (it's the dev API, so there won't actually be that much data there). JOSM tells you when a bounding box is "likely too large" when you're specifying it - a "not quite too large" one would have the scale at the top left displaying about 5km. Save the resulting .osm file and load that.</p>

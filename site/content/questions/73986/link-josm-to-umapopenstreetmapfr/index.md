<p>met, it is totally acceptable and feasible to use JOSM to edit features, save the result locally into an .osm file and load that file into uMap. I do that too.</p>
<p>I have for example a uMap where I want to highlight certain rivers in an area. So in JOSM I filter out all the rivers, remove all node tags (e.g. weirs) and combine the river segments into one long line per river. Then I add tags I want to show in uMap, save the .osm file and load that into a layer in uMap.</p>
<p>I don't have a solution for the OPs ask for a more connected way of working, though.</p>

<div class="answer-body">
<p>I just split the <a href="http://osm.org/go/0EH5TuVRr">building in two</a>, though I know address mappings vary. I know others add entrance nodes to the building outlines and add the address to these. I don't feel adding separate nodes within the outline is right based on the <a href="http://wiki.openstreetmap.org/wiki/One_feature,_one_OSM_element">one feature one osm element</a> guideline</p>

<p>Note that there are plenty of schools of thought about address placement : Should the node be on the building entrance or on the letterbox ? On the building itself or on the whole grounds ?</p>
<p>All these nuances work equaly well from a routing POV. They give different renderings, but which is better is highly subjective. The semantics are different, but I've never seen the intended semantic tagged, so it seems unusable.</p>

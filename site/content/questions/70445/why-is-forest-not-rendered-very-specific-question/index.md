<p>The multipolygon defining the forest (<a href="https://www.openstreetmap.org/relation/9678248">relation 9678248</a>) seems to be broken. I cannot check in detail now but by just looking at it I see that for example the <a href="https://www.openstreetmap.org/way/222762787">wetland between Stora and Lilla Jolpan</a> is supposed to be an inner within the outer <a href="https://www.openstreetmap.org/way/24743583">Stora Jolpan way</a>. But in fact both intersect.</p>
<p>I would try to clean up the multipolygon so that it only has true inners. One of the quality tools out there might find more problems with the relation.</p>
<p>The mentioned wetland is half floating on the ocean. I guess that needs to be mapped differently anyway.</p>

<p>That looks like a bit of a mess.</p>
<p><a href="http://tools.geofabrik.de/osmi/?view=areas&amp;lon=18.81562&amp;lat=59.47894&amp;zoom=15&amp;opacity=1.00">http://tools.geofabrik.de/osmi/?view=areas&amp;lon=18.81562&amp;lat=59.47894&amp;zoom=15&amp;opacity=1.00</a></p>
<p>Tip: Any 'inner' which touches an 'outer' shouldn't be part of the MP relation. So all wetlands &amp; beaches plus a couple of grassed areas which reach to the coast. They should be mapped as ways so remove the inner relation ref. The 'outer' should be drawn around them.</p>

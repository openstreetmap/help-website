<p>Hi,</p>
<p>For the las few days we are having trouble loading map images from Firefox 12. When trying to load from {a|b|c}.tile.openstreetmap.org, the response is 403 Forbidenn.</p>
<p>Trying to load the same page with current version of Firefox, Chrome or Edge works fine.</p>
<p>Also, if we try to load a single Image (a.tile.openstreetmap.org/11/623/1225.png) whe have an Access Denied error page.</p>
<p>We have tried from client computers in Spain and Chile with same results.</p>
<p>We think the problem is related to User-Agent, wich is in Firefox 12: Mozilla/5.0 (Windows NT 5.1; WOW64; rv:12.0) Gecko/20100101 Firefox/12.0</p>

<div class="answer-body">
<p>Yes, you're right. I tried Firefox 12 and also got HTTP 403 when using the following user agent: <code>Mozilla/5.0 (X11; Linux x86_64; rv:12.0) Gecko/20100101 Firefox/12.0</code>.</p>
<p>It looks like those old user agents get used quite a bit by some abusive OSM Tile scrapers, and are therefore banned.</p>
<p>If updating to a more recent Firefox version (think security!!) is not an option for you, a Firefox extension like User Agent Switcher might be a workaround for the time being. In any case I strongly recommend to update.</p>
<p><a href="https://github.com/openstreetmap/chef/blob/f6ba704c4a2c12c7eb71eda60ab88cfef12ac73b/cookbooks/tilecache/templates/default/squid.conf.erb">(actual config file)</a></p>

<table style="width:100%;"><colgroup><col style="width: 50%" /><col style="width: 50%" /></colgroup><tbody><tr class="odd"><td style="width: 30px; vertical-align: top"><div class="vote-buttons"><div id="post-24279-score" class="post-score" title="current number of votes">2</div></div></td><td><div class="item-right"><div class="answer-body"><p><del>That method (-o wlan.wep_keyxxx) does <strong>not</strong> work with Wireshark 1.11.0, as the management of the wireless keys has changes some releases ago (I believe 1.8.0).</del></p><del></del><p>So, this are your options:</p><p><strong>Downgrade to 1.6.x</strong></p><p>You will be able to use the option <strong>-o wlan.wep_key1</strong></p><blockquote><p>tshark -nr input.pcap -o wlan.enable_decryption:TRUE -o wlan.wep_key1:wpa-psk:passphrase</p></blockquote><p><strong>Wireshark 1.8.x - 1.11.x</strong></p><p>Create a file called <strong>80211_keys</strong> in the profile directory: %APPDATA%\Wireshark\profiles\xyz, while xyz is the name of the profile.</p><p>Content of <strong>80211_keys</strong></p><pre><code># This file is automatically generated, DO NOT MODIFY.
&quot;wpa-pwd&quot;,&quot;Passphrase:SSID&quot;</code></pre><p>Then run tshark:</p><blockquote><p>tshark -nr input.pcap -o wlan.enable_decryption:TRUE</p></blockquote></s><p><del>BTW: With tshark option <strong>-C</strong> you can specify a certain profile (see man page).</del></p><p><strong>UPDATE</strong> Based on the answer of @Pascal Quantin in the following question, I realized, that the WEP/WPA keys <strong>can be added</strong> through tshark as a UAT entry!!</p><blockquote><p><a href="http://ask.wireshark.org/questions/24474/user_dlt-option-in-tshark">http://ask.wireshark.org/questions/24474/user_dlt-option-in-tshark</a></p></blockquote><p>So, here we go, without the need to modify any files manually</p><blockquote><p>tshark -nr input.pcap -o wlan.enable_decryption:TRUE <strong>-o "uat:80211_keys:\"wpa-pwd\",\" Passphrase:SSID\""</strong> -R "http"</p></blockquote><p>Example for this <a href="http://wiki.wireshark.org/SampleCaptures?action=AttachFile&amp;do=get&amp;target=wpa-Induction.pcap">sample capture file</a>:</p><blockquote><p>tshark -nr input.pcap -o wlan.enable_decryption:TRUE <strong>-o "uat:80211_keys:\"wpa-pwd\",\" Induction:Coherer\""</strong> -R "http"</p></blockquote><p><strong>Hint:</strong> The quotes and backslashes need to be exactly as shown above, otherwise the UAT entry will not be recognized.</p><p>Nice feature. Again something new I learned about tshark/Wireshark :-))</p><p>PS: Some samples for the UAT 'option' should probably be added to the docs (man page) of tshark.</p><p>Regards<br />

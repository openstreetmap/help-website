<div class="answer-body">
<p>This does not seem to be OSM data. The Kosovo object has been populated, somewhat inaccurately, with 2- and 3-letter ISO country codes (XK &amp; XKR) since March 2017. I cannot <a href="https://osmlab.github.io/osm-deep-history/#/relation/2088990">find any time</a> when any tag on that relation shows CS-KM. Furthermore there are <strong>NO</strong> objects in the OSM database with <a href="https://taginfo.openstreetmap.org/search?q=CS-KM#values">that value</a>.</p>
<p>CS ISO codes ceased to be used back in 2006 according to <a href="https://en.wikipedia.org/wiki/ISO_3166-2:CS">wikipedia</a>.</p>
<p>You need to explain which website, application etc that you are using. It may be a created by a firm who merge OSM data with other sources, in which case it is likely you need to direct your query to them.</p>

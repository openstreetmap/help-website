<div class="answer-body">
<p>When creating new tags don't use something cryptic that only your application can understand; OSM data is designed to be used by lots of people and applications. If you find the need to store private data you'll need to do it in a private database.</p>
<p>You can have a look at see what people have used previously for by searching taginfo - for example for "<a href="http://taginfo.openstreetmap.org/search?q=skate">skate</a>".<br />
</p>
<p>There's not a whole lot of information in there for skateboarding, so you might end up tagging things that haven't been tagged before. It might be helpful to borrow from other related sports - there are already a few <a href="http://taginfo.openstreetmap.org/search?q=halfpipe#values">halfpipes</a> (I'm presuming for snowboarders) in OSM.</p>

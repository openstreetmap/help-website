<p>I have got a few thousand cleaned GPX files of motorbikes. These trace files majorly represent missing roads in OSM (which I have derived by using months of bikers data and identifying trace points which are not near any of the road segments in OSM). I want to contribute to OSM on a regular basis and want to take an opinion on my workflow for submission of these trace files to OSM.</p>
<p>Workflow:</p>
<ol>
<li>Upload all GPX files in bulk using <a href="https://wiki.openstreetmap.org/wiki/Upload_GPS_tracks">OSM UI</a> or using <a href="https://wiki.openstreetmap.org/wiki/API_v0.6">API</a>.</li>
<li>Pass some GPS-tags, e.g. name, id, status, etc. So, that I can query data that I have uploaded.</li>
<li>Have a team of 2-3 people who can then use JOSM or other editors to make ways/roads from this GPX data to OSM</li>
</ol>
<p>Questions -</p>
<ol>
<li>Is this a correct workflow for uploading and making ways/roads from GPX files?</li>
<li>Can I use GPS-tags to track of how many gpx files have been converted into ways/roads? Also, Is there any alternative to have this capability?</li>
</ol>

<p>We are considering using OSM instead of Google Maps for a new web application. The map requirements for this application are:</p>
<ol>
<li>A map view showing start and stop addresses, and the route taken to get there. We need to be able to configure the size of this map to fit the display of our app.</li>
<li>Provides distance to travel between 2 points</li>
<li>Provides time to travel between 2points</li>
<li>Provides auto-complete suggestions when the user types in an address.</li>
</ol>
<p>Can OSM do all this? If not, what can it do and what can't it do?</p>

<p>First of all: great that you are considering to use OSM!</p>
<p>Answer:</p>
<ul>
<li>OSM itself? – No (the core OSM is just a massive amount of geodata which you can use <span>nearly</span> as you like).</li>
<li><span>Services</span> built on OSM data (maybe run and/or coded/customized by yourself)? – Yes (and even much more).</li>
</ul>
<p>Your question seems to be about routing and a web frontend (I tried to make this clear in the question title). See <a href="https://wiki.openstreetmap.org/wiki/Routing">routing</a> in our wiki. You may also be interested in <span>Commercial OSM Software and Services</span> if you do not want to to development work on your own.</p>

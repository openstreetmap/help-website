<p>This could be either a mapping error or a technical issue with the OCM style. If it is the former it is likely already fixed as the other layers don't show the issue, OCM is simply not updated as fast as the other layers.</p>
<p>However in both situations we can't really help any more here as the OCM map is a product of Andy Allen / <a href="http://www.thunderforest.com/,">http://www.thunderforest.com/,</a> if the issues doesn't go away on its own accord, I would suggest contacting Andy.</p>

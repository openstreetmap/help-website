<div class="answer-body">
<p>The other answers give helpful editor-specific information. In case it's helpful to see the actual tags for additional attributes for golf holes, here they are:</p>
<ol>
<li>for hole number, use <a href="https://wiki.openstreetmap.org/wiki/Key:ref">ref=*</a></li>
<li>for par, use <a href="https://wiki.openstreetmap.org/wiki/Key:par">par=*</a>. If one set of tees has a different par, you can denote that after a colon, such as par:red=*</li>
<li>for handicap, use handicap=*. For holes that have different handicaps, I think it makes most sense to use the same format with a colon mentioned for par above, although some people seem to use an underscore instead.</li>
<li>(bonus) you can also add the distance. In this case, dist:color=* seems to be used. (maybe because <a href="https://wiki.openstreetmap.org/wiki/Key:distance">distance=*</a> is used in a different contexts along railways and highways?) It is not documented, so I'm not sure if it's assumed to be feet or meters or just whatever the local units are.</li>
</ol>

<div class="comment-text">
<p>Please don't make up text to put into the name/title just because it shows on a particular map. By putting the information into the specific tags, a given map rendering can easily choose whether to show them or not. Putting the information in the name so that it is shown on one particular map breaks this.</p>
<p>This is a frequent discussion for OSM: <a href="https://wiki.openstreetmap.org/wiki/Tagging_for_the_renderer">https://wiki.openstreetmap.org/wiki/Tagging_for_the_renderer</a></p>

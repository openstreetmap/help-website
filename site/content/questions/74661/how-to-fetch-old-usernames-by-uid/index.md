<div class="answer-body">
<p>If you want to find usernames that a particular account used to <em>edit</em> with then "Who's That" will help - like this: <a href="http://whosthat.osmz.ru/?q=SomeoneElse2">http://whosthat.osmz.ru/?q=SomeoneElse2</a>. However, that won't necessarily match all names that that account has been called (often users who have been blocked change the account name afterwards). The userid will be constant though, so you can use it e.g. at <a href="http://resultmaps.neis-one.org/osm-discussion-comments?uid=490556">http://resultmaps.neis-one.org/osm-discussion-comments?uid=490556</a>.</p>
<p>However, often "vandal" users (of which there are thankfully not many) will create multiple accounts, so you sometimes need to look for "edits to the same sort of objects in a similar style".</p>
<p>If you're aware of a vandal account please of course email the Data Working Group at data@osmfoundation.org !</p>

<div class="answer-body">
<p>Another option that leads to the same results, but does not depend on a third-party web site, is this:</p>
<ul>
<li>get a full history planet file (.osh.pbf)</li>
<li>install the <code>osmium</code> command line utility</li>
<li>convert the full planet file into easily parsable "opl" format and check for all lines that contain " i&lt;userid&gt;"</li>
<li>those lines will also have " u&lt;username&gt;" which you can use to list the user name.</li>
</ul>
<p>Here's a Linux one-liner (fsvo) that will do it:</p>
<pre><code>curl https://planet.openstreetmap.org/pbf/full-history/history-latest.osm.pbf |
   osmium cat -Fosh.pbf -fopl |
   grep &quot; i490556 &quot; |
   sed -e &quot;s/.* u//&quot; |
   cut -d\  -f1 |
   sort -u</code></pre>
<p>If you are only looking for actions in a particular country or region, you can also download a regional history file instead which speeds up the process.</p>
<p>Note that neither whosthat.osmz.ru nor this process will return user names that the user may have had, and may have used to write messages, but has not used for editing OSM.</p>

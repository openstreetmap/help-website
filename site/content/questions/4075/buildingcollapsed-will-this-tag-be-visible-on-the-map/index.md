<div class="answer-body">
<p>The standard 'Mapnik' render is not designed as a crisis mapping platform. Just as it is also not the best map for planning cycle tours, MTB biking, white-water excursions or a host of other activities which have specialist dedicated maps produced from OSM data. Please remember OpenStreetMap is primarily about creating geographical data: rendered maps are just one of the ways which this data is consumed.</p>
<p>Suitable visualisations of crisis related data are available: for instance those made by <a href="http://resultmap.neis-one.org/japan.html">Pascal Neis</a>, or the <a href="http://www.tappenbeck.net/osm/maps/deu/index.php?id=2000">POI map</a> or the damage overlay capabilities of All-in-One map for Garmin devices.</p>

<div class="answer-body">
<p>Yes, mapnik treats all building values that are not "no" as buildings and will render them. Please note that the wiki defines building=&lt;building typology=""&gt; but <em>"collapsed"</em> is a building <em>state</em>, not a <em>typology</em>, so I'd consider this "tagging for the renderer" (because although "semantically wrong" it will be displayed "OK").</p>
<p>It might be an alternative to use the well established attribute ruins=yes.</p>

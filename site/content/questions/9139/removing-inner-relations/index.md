<div class="answer-body">
<p>Not sure what you mean here. Relation members are easy enough to edit in the relations screen. Do you want to delete a relation, but transfer its tags back to the relation members ?</p>
<p>If that's what you want (are you sure ? relations are a good thing, think twice before deleting one), here's an option :</p>
<ul>
<li>Select a relation, copy it (Ctrl+C)</li>
<li>Select a member, and paste tags (Ctrl_Shift+V)</li>
<li>Delete the relation</li>
<li>Edit/remove tags that have nothing to do in the relation member, like "type=route" for example</li>
<li>Copy the member (Ctrl+C), select the other members, and paste tags (Ctrl+Shif+V)</li>
</ul>
<p>You may want to use a temporary object to edit the tags to be pasted, so that only the tags you want to paste are included.</p>

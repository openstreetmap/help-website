<div class="answer-body">
<p>That is likely related to the reverse geocoding feature of <span>Nominatim</span>, which seems to be used here to add full location information in the search results. You will also note that "Windig" is added to the location description if you centre the map on Fribourg and click "where am i" in the left menu panel.</p>
<p>That seems to be because there is <a href="http://www.openstreetmap.org/browse/node/859250421">Windig</a> (<code>place = suburb</code>) defined as a node (just one location point) and the centre has no suburb. Nominatim searches to which suburb your location belongs and only finds Windig as most <a href="http://overpass-turbo.eu/s/1j6">nearest suburb</a>. This could be improved by marking Windig suburb as an area instead of just a point - to help Nominatim with guessing how big this suburb is. See also <a href="/questions/16357/">nominatim-town-suburb-village-hierarchy-confusion-in-results</a>.</p>
<p>All in all: you have done your entry correctly. I hope you continue to improve OpenStreetMap when you have a bit of free time! :-)</p>

Search for your feature on <a href="http://nominatim.openstreetmap.org">http://nominatim.openstreetmap.org</a> and click on <em>(details)</em> at the relevant result. There you can see the <a href="http://nominatim.openstreetmap.org/details.php?place_id=9146353815">address hierarchy for La Petite Cantine</a>, including the suburb node for Windig.</p>

<p><a href="https://forum.openstreetmap.org/viewtopic.php?pid=623062#p623062">This forum thread</a> has some background for this question.</p>
<p>As I mentioned in that thread, you will have to split the osm file into smaller parts if you want to use OSM2World – a 500 MB input file is too much to render at once, even with more hardware. I've been using <a href="https://github.com/PedaB/mapsplit">mapsplit</a> for this purpose when I set up my <a href="http://maps.osm2world.org/">online map</a>, but you might prefer other tools. In any case, you also need to think about recombining the result and removing duplicated buildings in the process, which is not trivial.</p>

<p>I am trying to find how to tag changing places in OSM (<a href="https://changingplaces.org.au">details here</a>).</p>
<p>The existing <a href="https://wiki.openstreetmap.org/wiki/Tag:amenity%3Dtoilets">amenity=toilets</a> schema offers some useful tags that incorporate individual elements of a changing place. However, I am wondering if this would be the best approach, as some tags could be ambiguous.</p>
<p>For example, one facility in a changing place is a height-adjustable change table. While <code>changing_table=yes/no/limited</code> does exist, it is not clear in the tag whether it is intended for adults or babies, nor if it is height-adjustable.</p>
<p>Any guidance on this would be greatly appreciated.</p>

<div class="answer-body">
<p>I don't believe there is an easy answer.</p>
<p>If this is mainly a property of toilets (aka you can assume that all of these facilities have a toilet and the toilet is the main facility you would be looking for) extending the toilet tagging scheme would seem to be best.</p>
<p>If on the other hand these are mainly stand-alone facilities that may or may not have a toilet then a separate tagging scheme might make more sense.</p>
<p>Potentially we will need both to map the real world.</p>
<p>In any case a better place for discussion and further input is the tagging mailing list: <a href="https://lists.openstreetmap.org/listinfo/tagging">https://lists.openstreetmap.org/listinfo/tagging</a></p>

<p>I believe the <a href="https://wiki.openstreetmap.org/wiki/Key:diaper">diaper tagging scheme</a> was designed for exactly this purpose. Check out if it already covers what you are looking for or how it needed be expended. I'm not sure how sufficiently it covers the adult changing places aspect. There is also a specialized map "<a href="https://babykarte.github.io/#">Babykarte</a>" which shows changing tables on a map. <a href="https://wiki.openstreetmap.org/wiki/Babykarte#Wickelpl.C3.A4tze">Its description</a> is only available in German unfortunately. If you want to expand the scheme the tagging mailing list is probably the best location to start the discussion.</p>

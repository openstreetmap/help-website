<div class="answer-body">
<p>If you need special maps like you mentioned you should start investigating in the OSM wiki</p>
<p>(or even on this FAQ site by searching for keywords like "render own map" or own style" or similar.</p>
<p>Because there is ONE OpenstreetMap database, but many many possibilities how to convert the raw OSM data to bitmaps or PDF or SVG files.</p>
<p>So you have to find a webservice that might offer maps rendered near your aims, and then you have to find a way to get graphic output like you need.</p>
<p>Besides commercial companies you can do it all on your own, you can download the raw data, do some filtering, and then render any graphic in a format of your choice.</p>
<p>So start reading in the OSM wiki about Rendering, osmfilter, SVG, commercial, List_of_OSM_based_Services</p>

<p>The are two questions here, but <a href="https://help.openstreetmap.org/questions/20714/how-to-getuse-transport-layer-of-osm">the answer to this previous question</a> may actually help with both. There's some more information about the OSM transport layer (including the terms of use) <a href="http://www.thunderforest.com/">here</a>.</p>
<p>If you haven't already seen it, <a href="http://wiki.openstreetmap.org/wiki/Beginners_Guide_1.3">this page of the beginners' guide</a> might be useful. It describes the sort of data that is contained within OSM. What you won't see there are separate layers for e.g. hydrology and infrastructure. If you need to extract certain types of data you'll need to process a <a href="http://wiki.openstreetmap.org/wiki/Extract">data extract</a> with something like <a href="http://wiki.openstreetmap.org/wiki/Osmfilter">osmfilter</a>, or query data with something like <a href="http://wiki.openstreetmap.org/wiki/Overpass_API">Overpass</a> (there's also a <a href="http://wiki.openstreetmap.org/wiki/Overpass_turbo">graphical front-end for that</a>).</p>

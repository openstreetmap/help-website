<p>I guess that may be because there is some coastline mapping error (not closed or something).</p>
<p>The problem <em>seems</em> to be fixed already, just those tiles were not <span>updated</span> yet. For testing, I have applied /dirty to one tile in each affected zoom level there (e.g. <a href="https://b.tile.openstreetmap.org/14/8150/5396.png">https://b.tile.openstreetmap.org/14/8150/5396.png</a> → <a href="https://b.tile.openstreetmap.org/14/8150/5396.png/dirty">https://b.tile.openstreetmap.org/14/8150/5396.png/dirty</a> ), reloaded without cache with ctrl+f5 (Firefox), no more blue. Opened again, now some tiles are blue again. It may be an issue of the tile caching servers. On the <a href="https://www.openstreetmap.org/#map=13/52.1761/-0.9204&amp;layers=H">"Humanitarian" tiles</a> I could not find "blue" (those tiles are also usually updated in less than a day). So I would assume this to be fixed. Just wait.</p>

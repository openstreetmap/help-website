<p>I went for …</p>
<ul>
<li><a href="http://taginfo.openstreetmap.org/tags/barrier=tree_stump"><code>barrier=tree_stump</code></a></li>
<li><code>height=0.42</code></li>
<li><code>material=wood</code></li>
<li><code>width</code> might be a good addition, especially if it is very wide</li>
</ul>
<p>… when I tagged such an unusual barrier the last time. With "<span>tree stump</span>" (instead of "stub") it is likely more clear that it is a part of a tree than if you use "stub".</p>

<p>… If you would use it as a node on a normal way I think tags which show who/what can pass would be good. Maybe this as an default override (if there are really no legal restrictions): <code>access=yes</code> <code>maxwidth=1.5</code> see <span>barrier documentation</span>. In principle I think those legal access tags are often not good for barriers (because those are not legal restrictions) and physical (e.g. <code>maxwidth:physical</code>) ones should be used instead, but the <code>:physical</code> ones are used very seldom and not (yet) documented on that barrier page.</p>

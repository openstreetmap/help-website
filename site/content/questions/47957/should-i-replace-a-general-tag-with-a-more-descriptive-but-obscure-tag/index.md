<div class="question-body">
<p>I have a cycle path in my area that was originally tagged with <code>surface=gravel</code>. Technically, it is not a gravel path, it is a limestone path. The difference is minor, but limestone and gravel are quite different for cycling purposes. (Limestone is generally easier to cycle on). The only issue is, the <code>surface=limestone</code> tag is barely used at all according to taginfo.</p>
<p>Should I replace the widely used, surface:gravel tag with the more accurate, but obscure, surface:limestone tag?</p>

<div class="answer-body">
<p>Well, all of those surface values are somewhat categories, we need to stop somewhere in increasing the detail level of those categories. Otherwise nobody will understand that more detailed value. Is the difference really that big? So, I would stick to the values listed on <a href="https://wiki.openstreetmap.org/wiki/Key:surface">https://wiki.openstreetmap.org/wiki/Key:surface</a> (and the widely used ones – see taginfo).</p>
<p>Instead sub-categories could be used (<a href="https://wiki.openstreetmap.org/wiki/Any_tags_you_like#Syntactic_conventions_for_new_tags">iterative refinement</a> is often used in tagging). However, I did not yet see something like <code>surface:gravel=limestone</code> or <code>gravel=limestone</code> (depending on which style you want to use). I had a look: on 112 objects in one area in the US <a href="https://taginfo.openstreetmap.org/tags/gravel=crushed_limestone"><code>gravel=crushed_limestone</code></a> is used as a refinement of <code>surface=gravel</code>. Maybe from one mapper.</p>
<p>Maybe you want to add other describing tags instead (e.g. <code>smoothness</code>)?</p>

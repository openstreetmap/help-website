<p>You can download map regions, see <a href="http://docs.kde.org/development/en/kdeedu/marble/download-region.html">this documentation</a>. But please be gentle to the servers. If you download an area that's too big, you will be blocked.</p>
<p>If you want more, you can also set up your own rendering, and use it with Marble. But that would require a quite heavy computer, and it's not documented how to achieve it in Marble (because they are afraid people will use it to display proprietary maps).</p>

<p>I'm not at all familiar with Marble but a quick poke around suggests that you might want to start reading <a href="http://blogs.kde.org/2008/02/09/marbles-secrets-part-i-behind-scenes-marble">here</a> for an introduction to how Marble themes are set up.</p>
<p>If you have Marble installed you'll have an "<code>/usr/share/kde4/apps/marble/data/maps/earth/openstreetmap</code>" directory with an "<code>openstreetmap.dgml</code>" file in it. In there you can see how the tile servers are configured.</p>
<p>If you never wanted to get blocked from a tile server, you could try <a href="http://switch2osm.org/serving-tiles/">serving your own tiles</a>.</p>

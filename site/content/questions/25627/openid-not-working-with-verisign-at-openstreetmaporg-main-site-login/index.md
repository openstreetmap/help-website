<p>You are mistaken; "&lt;user&gt;.pip.verisignlabs.com" is <em>not</em> a URL; every URL must start with a scheme name, a colon, and two slashes. See <a href="http://en.wikipedia.org/wiki/URL">Wikipedia</a> for details.</p>
<p>The best way to suggest a change to the OpenStreetMap web site code is through the <a href="https://github.com/openstreetmap/openstreetmap-website/issues">GitHub issue tracker.</a></p>

<p>The OpenID spec actually specifies that the end user may leave off the http part ( <a href="https://openid.net/specs/openid-authentication-1_1.html#anchor6">https://openid.net/specs/openid-authentication-1_1.html#anchor6</a> ). The problem is that the rails form requires contrary to the spec a valid full URL. I think this happened in one of the redesigns.</p>
<p>However, there is an additional issue with verisign and http vs https. If you use http with your URL, login fails with "Sorry, could not log in with those details." It only works with https. I am not sure though if that is a bug with verisign or with the OpenID plugin of rails.</p>

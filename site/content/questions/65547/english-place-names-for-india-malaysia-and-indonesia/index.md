<div class="answer-body">
<p>The Indian OSM community has decided to use the English form in the primary <code>name</code> tag, with the name in other languages specified in other tags.</p>
<p><a href="https://wiki.openstreetmap.org/wiki/WikiProject_India#Naming_in_different_scripts_and_languages">WikiProject India: Naming in different scripts and languages</a></p>
<p>The Standard map on the main openstreetmap.org site uses the <code>name</code> tag and therefore the name in English for these countries, but other maps can choose to use the other languages instead.</p>

<div class="answer-body">
<p>Disclaimer: I'm from Malaysia.</p>
<p>Some places in the country actually use "English" names e.g. luxury residential areas, shopping centres, bars and the likes (mostly private properties and spaces) -- in highly urbanised areas.</p>
<p>Other amenities and official government-related infrastructures will definitely use the national language; namely the Malay language. But some others, for example, Taoist temples; might be in Chinese only.</p>
<p>There's already <a href="https://wiki.openstreetmap.org/wiki/WikiProject_Malaysia/Other_tagging_aspects#Multilingual_names">a guide on the Wiki</a> on <code>name=*</code>s, in the spirit of <em>map what you really see</em>.</p>
<p>Moving back to the topic. This is my personal opinion: probably something to do with the editor interface.</p>
<p>I've seen new editors, and their language locales (per their browser), are usually <code>en-US</code> or <code>en-XX</code>. So the iD editor will be displayed in English, definitely. This might somehow, indirectly, hinted them to translate <em>almost</em> everything to English (when it's <strong>not</strong> necessary at all!).</p>
<p>However, from what I've seen, the default localisation for OSes are in English; so any installed browser will be in English too. And there you have it: an almost Catch-22 situation.</p>
<p>P.S. I've tried my best to translate the iD editor to the local language as well. It's not 100% complete but I've only seen one people using it so far.</p>

<p>This is the decision of local communities I guess.</p>
<p>I know only India case - there's special, localized map made for displaying in different languages of this big country, and English might be chosen as more neutral than any of 22 languages spoken there:</p>
<p><a href="https://www.openstreetmap.org/user/PlaneMad/diary/38176">https://www.openstreetmap.org/user/PlaneMad/diary/38176</a></p>
<p>In general we have currently no solution for displaying any language in OSM, but there are maps available in any languages with Wikimedia style (light one):</p>
<p><a href="https://blog.wikimedia.org/2018/06/28/interactive-maps-now-in-your-language/">https://blog.wikimedia.org/2018/06/28/interactive-maps-now-in-your-language/</a></p>
<p>For example:</p>
<p><a href="https://maps.wikimedia.org/?lang=zsm#7/3.519/-246.583">https://maps.wikimedia.org/?lang=zsm#6/3.722/-251.840</a></p>
<p><a href="https://maps.wikimedia.org/?lang=ind#5/-4.149/-250.554">https://maps.wikimedia.org/?lang=ind#5/-4.149/-250.554</a></p>
<p>I hope one day OSM as a project will be able to render vector maps, which helps to change languages for example. It's not clear if this will be change in the current map style or something completely new, but it's something everybody wants, but no one is engineering right now...</p>
<p><a href="https://wiki.openstreetmap.org/wiki/Top_Ten_Tasks#Localized_map_rendering">https://wiki.openstreetmap.org/wiki/Top_Ten_Tasks#Localized_map_rendering</a></p>
<p><a href="https://github.com/gravitystorm/openstreetmap-carto/issues/3201">https://github.com/gravitystorm/openstreetmap-carto/issues/3201</a></p>
<p><a href="https://github.com/openstreetmap/operations/issues/214#issuecomment-408816591">https://github.com/openstreetmap/operations/issues/214#issuecomment-408816591</a></p>

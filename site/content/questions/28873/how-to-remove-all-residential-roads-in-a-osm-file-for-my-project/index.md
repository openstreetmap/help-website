<p>The simplest way to remove given classes of objects from an OSM <strong>FILE</strong> (i.e., not the database) is to use <a href="https://wiki.openstreetmap.org/wiki/Osmfilter">osmfilter</a>. A bit of experimentation is required to find the exact set of filters which meet ones requirements. I recommend using a suitable small dataset before you embark on doing it on your main data (I think Liechtenstein and Luxembourg are favourites as suitable datasets).</p>

<p>To delete a special type of road or to delete all trees from your .osm file: just use our offline editor <span>JOSM</span>. It has a find function which can search for tags (e.g. natural=tree). Maybe it may even be possible to use the search to select dead ends. It has a quite powerful <span>syntax</span>. Note to use the <a href="http://josm.openstreetmap.de/wiki/Help/Action/Purge">purge</a> action instead of "delete" (which only marks a object as to be deleted in the .osm file). When finished save into the .osm file again.</p>
<p>Take care not to upload such changes to the OSM servers with JOSM.</p>
<p>You also could use an XML or even text editor to manipulate the downloaded <span>OSM data</span>.</p>

<div class="comment-text">
<p>If you are using JOSM to remove objects from an OSM file, you do not want to use delete, you want to use purge.</p>
<p>As a general rule, filtering the .osm file to remove data is not the optimal place, you're normally better off removing data as part of the process that interprets the .osm file.</p>
<p>Filtering out to only accept specific data is generally reasonable.</p>
<p>Of course, without details of the software that will consume the .osm file, it's impossible to give more specific advice.</p>

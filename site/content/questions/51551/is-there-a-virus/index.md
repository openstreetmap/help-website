<div class="answer-body">
<p>No, I doubt that opentopomap is malicious. Especially not the PNG map images (that would need a big security hole in your browser/OS graphics library to be possible) which are loaded on your umap. Yes, so, I think this is a mistake of ESET.</p>
<p>Details:<br />
Yes, ESET currently reports this site as "Malware site" - see <a href="https://www.virustotal.com/de/url/b61c54f47be2087dce1bd1659047e5506af757eb9c2b93af850e66e39bdc62b3/analysis/1471636694/">https://www.virustotal.com/de/url/b61c54f47be2087dce1bd1659047e5506af757eb9c2b93af850e66e39bdc62b3/analysis/1471636694/</a> . No other of the 68 scanners at this scanner comparison site does so. In fact ESET categorizes the whole opentopomap site as "Malware site" - see <a href="https://www.virustotal.com/de/url/b71976f85ec657d2859ed12a2bd2ebb045b0339954c78b6c3e38d5d7fdefa19f/analysis/1471636840/">https://www.virustotal.com/de/url/b71976f85ec657d2859ed12a2bd2ebb045b0339954c78b6c3e38d5d7fdefa19f/analysis/1471636840/</a> .</p>
<p>I could not see calls to third party resources by the opentopomap website (which often happens for infected sites).</p>
<p>I have written the opentopomap developer derstefan an osm message, asking if he is aware of any recent issues.</p>
<p><em>Update:</em> now ESET changed its assessment of <a href="http://opentopomap.org/">http://opentopomap.org/</a> into "clean" - see <a href="https://www.virustotal.com/de/url/b71976f85ec657d2859ed12a2bd2ebb045b0339954c78b6c3e38d5d7fdefa19f/analysis/">https://www.virustotal.com/de/url/b71976f85ec657d2859ed12a2bd2ebb045b0339954c78b6c3e38d5d7fdefa19f/analysis/</a></p>

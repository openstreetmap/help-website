<p>Hi!</p>
<p>Context: I'm using the Android library Mapsforge to display OSM maps of each Brazilian state inside a native Android application. I get each state map from <a href="http://download.openstreetmap.fr/extracts/south-america/brazil/">here</a> and convert them from .osm.pbf to .map using Osmosis. The app uses only one map, of the state the user is currently living in.</p>
<p>Mapsforge use render themes files like <a href="https://github.com/mapsforge/mapsforge/blob/master/mapsforge-themes/src/main/resources/assets/mapsforge/osmarender.xml">this</a> to determine what color each area should be painted as, how captions should be rendered, etc.</p>
<p>Problem: I'm having problems rendering the <a href="https://www.openstreetmap.org/relation/7888764#map=9/-0.3722/-50.2872">Delta do Amazonas</a> region. All the islands inside that relation are responding to "natural=water" and not responding to "place=island" (therefore being painted as blue, same as the ocean and other bodies of water), while the water part of that relation is being shown as land, not responding to tags like "natural=water", "waterway=*".</p>
<p>I can't figure out if the problem is the render theme file, the <a href="http://download.openstreetmap.fr/extracts/south-america/brazil/north/para.osm.pbf">original map file</a> itself, or maybe the conversion process.</p>
<p><a href="https://ibb.co/N3hthRM">Here's</a> a screenshot of the app showing the Delta do Amazonas region.</p>

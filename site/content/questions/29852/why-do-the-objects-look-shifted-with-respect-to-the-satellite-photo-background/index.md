<div class="answer-body">
<p>That seems to be "the" standard problem with aerial imagery (and multiple sources in general).</p>
<p>See e.g. <a href="/questions/18655/">bing-osm-and-my-waypoints-all-misaligned-who-do-i-trust</a> or the others listed under the same tags (click on the grey words below the question and sort by vote count).</p>
<p>Preliminary advice: Do not move the existing OSM data to match the aerial imagery unless you are quite sure. We need to know the reality – find it out, somehow. Aerial imagery is <em>not</em> necessarily close enough (by today's standards) to the real position. The other way round: OSM data is <em>not</em> necessarily close enough (by today's standards) to the real position.</p>
<p>For discussing (evidence, hints, measurements, observations of) this <em>specific</em> problem the <span>forum</span> is likely the better place (at least for same questions regarding other locations).</p>

<p>… It <em>may</em> also be (and should not be forgotten) that the imagery is not offset but the current OSM data is. That was the situation in my mapping area: the older aerial imagery before 2012 was offset (in addition varying offset due to differing ground elevation), and apparently many mappers used it without correcting the offset leading to offset OSM data.</p>
<p>Moving OSM data (if you/others are quite sure it is offset) is not easy as multiple objects need to be selected (best in JOSM), care must be taken of relative distances/angles, objects which are not offset must not be moved, …</p>

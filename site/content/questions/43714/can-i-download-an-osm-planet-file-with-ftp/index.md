<p>Is there a ftp download or another type of download available? Wouldn't it make sense with such large files?</p>

<p>Your assumption is not true. HTTP downloads <em>can</em> be resumed. This feature has to be supported by both the server and the client but it is quite unusual to <em>not</em> support it (except for many one-click hosters and similar services). Hence you should try to find out why you can't resume your download.</p>
<p>Resuming a HTTP download from <a href="http://planet.osm.org">http://planet.osm.org</a> works fine here.</p>

<p>It is not true that HTTP downloads cannot be resumed (see e..g the <code>--continue</code> option of the <code>wget</code> command line utility, or use some kind of download manager).</p>
<p>There is an FTP mirror of OpenStreetMap data, including old planet files, here: <a href="ftp://ftp.heanet.ie/pub/openstreetmap.org/">ftp://ftp.heanet.ie/pub/openstreetmap.org/</a></p>
<p>Note that if you download the "full history planet" then you can make snapshots for any point in time from that, and don't need to download tons of planet files.</p>

<p>If you use <code>wget</code> on the command line to download the file (e.g.: <code>wget </code><a href="http://planet.openstreetmap.org/pbf/planet-latest.osm.pbf"><code>http://planet.openstreetmap.org/pbf/planet-latest.osm.pbf</code></a>), and it gets interrupted, then you can just pass in the <code>-c</code> option and wget will automatically continue downloading the file. e.g.: <code>wget -c </code><a href="http://planet.openstreetmap.org/pbf/planet-latest.osm.pbf"><code>http://planet.openstreetmap.org/pbf/planet-latest.osm.pbf</code></a>. If that gets interrupted again, you can just call that command again and it'll pick up where it left of (etc. etc.)</p>

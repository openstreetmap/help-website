<div class="answer-body">
<p>There are many more tags than the editors show. E.g. there is <a href="https://wiki.openstreetmap.org/wiki/Proposed_features/ISCED">https://wiki.openstreetmap.org/wiki/Proposed_features/ISCED</a> , which might be what you are looking for. In JOSM, you can just add those tags.</p>
<p>In case you want to create your own tags, it's best to do research whether someone else had the same idea, search our wiki (e.g. 'osm tag XXX' in a search engine will give you good results.</p>
<p>If you cannot find the tag you need, you can come up with your own. But it's best to document it on the wiki, so others understand what you mean and they might start using the tag as well. There is also a formal process to get a tag "approved", see <a href="https://wiki.openstreetmap.org/wiki/Proposal_process">https://wiki.openstreetmap.org/wiki/Proposal_process</a></p>
<p>OSM has no layers. To get new tags added to the standard rendering, you have to create issues on <a href="https://github.com/gravitystorm/openstreetmap-carto">https://github.com/gravitystorm/openstreetmap-carto</a> . Do not have high expectations for rare tags to get rendered in that style.</p>
<p>You might better create your own map, <a href="http://umap.openstreetmap.org">umap</a> might be a good place to start. Especially in combination with Overpass, read e.g. <a href="http://www.mappa-mercia.org/2014/09/creating-an-always-up-to-date-map.html">http://www.mappa-mercia.org/2014/09/creating-an-always-up-to-date-map.html</a></p>
<p>Since this is a lot of information to digest, please do not hesitate to come back and ask more specific questions (preferably as new questions) here on this site.</p>
<p>Good luck</p>

<div class="answer-body">
<p>There are quite a few questions here, so my answer is quite long.</p>
<p>To answer your <strong>last question</strong> first:</p>
<p>OSM has not notion of layers for different types of data (hospitals, schools etc). Everything is held in a common data pool. This may make accessing different types of data a little harder (but see below), but comes with the immense benefit of showing explicit inter-relationships between different objects. Not only can be have a school, but also show which entrances in a school are suitable for a wheelchair user; or enable choice of routes to school which minimise crossing busy roads.</p>
<p>So when adding infrastructure, even if you add schools, hospitals separately, ultimately they should get interconnected as more detail is added.</p>
<p>Now on to <strong>what can be done in JOSM</strong>:</p>
<p>In JOSM it is perfectly possible to start a separate data layer for each type of object. These remain independent until you load them up to OSM.</p>
<p>Additional tags can be added directly in the tag edit panel once you have created the basic element (e.g., school). JOSM will also remember which tags &amp; values you have used recently. It is also possible to <a href="https://wiki.openstreetmap.org/wiki/Customising_JOSM_Presets">configure</a> JOSM to provide specific <a href="https://josm.openstreetmap.de/wiki/TaggingPresets">presets</a> relevant directly to what you are mapping: e.g., to show school types and age ranges. Indeed, suitable preset configurations may already exist.</p>
<p>For r<strong>etrieving data</strong> relating to a particular group of objects, the best tool to use is <a href="https://wiki.openstreetmap.org/wiki/Overpass_turbo">Overpass-turbo</a>, for instance here <a href="http://overpass-turbo.eu/s/iEk">schools</a>.</p>
<p>Finally note that there are already widely used tags for indicating school type (primary, secondary, etc.) and age ranges of the school children.</p>

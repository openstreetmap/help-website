<p>I am using a similar structure. maps/trips.raw maps/trips.a ... where trips.raw are raw gpx-Files</p>
<p>After primary editing I save them in trips.a for further processing. The primary editing consists of assigning general tags (record mode, way-types as road, path and so on) as well as deleting surplus nodes. I am living in a mountain area, where signal quality is often pretty bad. So keeping the original data is always a good idea in order to create layers from several trips and check which 'measurements' reflect best the actual scenery.</p>

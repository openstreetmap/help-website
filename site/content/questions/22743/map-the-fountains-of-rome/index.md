<p>I would like, starting sometime later this summer, to add locations of and information about the public fountains in Rome. Before I go hacking away, is there an expert who could guide me a bit?</p>
<p>I see that some "fountains" have been marked. The scare quotes mean I'm confused over the use of the tags on them. There seem to be 2 types: Amenity Fountain (icon fountain) and drinking water (icon faucet &amp; glass). In Rome, these overlap considerably. Almost all the fountains, which I would mark with a Fountain, because they are decorative, are also potable.</p>
<p>The drinking water designation seems to me more suitable for the nasoni: utilitarian little hydrants. Of which there are a thousand, at least! Not mapping them!</p>
<p>So which is more relevant: decorative Fountain or potable drinking water? And is there a way to give the proper historical names to them? Or add other historical details? Link to Wikipedia? Etc?</p>
<p>Please advise on the best way to go. Thanks!</p>

<div class="answer-body">
<p>The OpenStreetMap database consists of <a href="http://wiki.openstreetmap.org/wiki/Elements">objects</a> with attached <a href="http://wiki.openstreetmap.org/wiki/Tags">tags</a>. Objects can have more than one tag, in fact, most objects do have more than one.</p>
<p>The tag for a fountain is <a href="http://wiki.openstreetmap.org/wiki/Tag:amenity%3Dfountain">amenity=fountain</a> and the tag for drinking water is <a href="http://wiki.openstreetmap.org/wiki/Tag:amenity%3Ddrinking_water">amenity=drinking_water</a>. As they both belong to the same category (<em>amenity</em>), you can't apply both (except if you use the <a href="http://wiki.openstreetmap.org/wiki/Semi-colon_value_separator">semi-colon as a separator</a>). But you don't have to, because the wiki page of <em>amenity=fountain</em> clearly shows that there is also the <em>drinkable</em>=* tag.</p>
<p>So whenever you want to map a fountain which also contains drinkable water you apply both tags (<em>amenity=fountain</em> and <em>drinkable=yes</em>). Your editor might have a preset for the tags but if not then you can usually add them manually (for example in Potlatch 2 via the <em>advanced tab</em>). If you want to map a fountain which has no drinkable water you can also add the tag <em>drinkable=no</em> so that other users will see that this information has been surveyed.</p>
<p>There is also the <a href="http://wiki.openstreetmap.org/wiki/Key:historic">historic tag</a> for historic information. If the information you are searching for is not in the wiki you can also have a look at <a href="http://taginfo.openstreetmap.org/keys/?key=historic#values">other values for historic=* on taginfo</a> because not everything is documented in our wiki. Additionally taginfo also shows <a href="http://taginfo.openstreetmap.org/keys/?key=historic#combinations">commonly used tag combinations</a>.</p>
<p>Names can of course be added with the <a href="http://wiki.openstreetmap.org/wiki/Names">name tag</a>. There are also several name categories like <em>old_name</em> for historic names and <em>loc_name</em> for local names which differ from the official name. And of course names for localization.</p>
<p>If you know the build date of the fountain you can use the tag <a href="http://wiki.openstreetmap.org/wiki/Key:start_date">start_date</a>. It is even possible to specify the <a href="http://wiki.openstreetmap.org/wiki/Key:start_date#Marking_a_civilization.2C_period_and_era">civilization, period or era</a>.</p>
<p><a href="http://wiki.openstreetmap.org/wiki/Key:wikipedia">Links to wikipedia</a> are possible as well.</p>
<p>And there is more. Just search the wiki or taginfo if you are looking for a specific information you want to enter to find the appropriate tags. Not every tag has a preset in your editor and the less often used ones have to be entered manually until someone creates a preset for them.</p>

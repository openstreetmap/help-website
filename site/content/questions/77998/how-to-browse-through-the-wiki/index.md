<blockquote>
<p>Is there a TABLE OF CONTENT page,showing a structure of the Wiki? That would make the bowsing easier.</p>
</blockquote>
<p>Not exactly, because the wiki constantly changes so the people would spend a lot of time updating a ToC. I think the <a href="https://wiki.openstreetmap.org/wiki/Main_Page" title="Main page">Main page</a> shows useful links and you can click them to dive into a topic as in a hierarchical overview.</p>
<blockquote>
<p>If not, what advice would yuo give somebody like me, in order to find my way around?</p>
</blockquote>
<p>I would read <a href="https://wiki.openstreetmap.org/wiki/Searching_the_wiki">Searching the wiki</a> to understand how the search works and have a look at the <a href="https://wiki.openstreetmap.org/wiki/Wiki_organisation#Pages_naming_convention">Page naming convention</a>. The latter is useful when judging search suggestions like the difference between the wiki pages "Railways", "Key:railway", and "Sweden/Railways".</p>

<div class="answer-body">
<p>With it being a wiki, anyone can create a page so the structure can best be described as 'loose', a regrettably for the newbie, quite a few of the pages appear to have been written with the purpose of showing off how much detail the writer knows about a subject, rather than make it clear for a contributor just starting out.</p>
<p>As with many organisation's websites I've found Google's site search option more productive than the built in one. This way you'll come across a few isolated pages which aren't linked.</p>
<p>I wouldn't take what's in the wiki as gospel. it can often be at odds with what's used in the database. If you want to see if a certain tag is popular use this site: <a href="https://taginfo.openstreetmap.org/">https://taginfo.openstreetmap.org/</a> And if you wish to discuss any queries there's this site, or the tagging forum: <a href="https://lists.openstreetmap.org/pipermail/tagging/">https://lists.openstreetmap.org/pipermail/tagging/</a></p>
</div>

<div class="answer-body">
<p>Currently (sept 2011), there is no consensus about tagging dangerous roads for bikers. First, it is some personnal judgment - some would say "subjective stuff". What "Person A" biking with children considers as "dangerous" could be considered as "safe" for Person B driving alone with an mtb, etc.</p>
<p>I know two approaches about this topic : create a scale with numbers or values like "cycle:hazardous=1..5" or "hazard:bicycle=dangerous" (see the <a href="http://taginfo.openstreetmap.org/keys/hazard%3Abicycle">taginfo tool for some statistics</a>). But this solution remains subjective even if you provide a detailed description for each value.<br />
Or just describe <a href="http://wiki.openstreetmap.org/wiki/Template:Map_Features:restrictions">physical attributes</a> precisely with <a href="http://wiki.openstreetmap.org/wiki/Maxspeed">mapxspeed</a>, <a href="http://wiki.openstreetmap.org/wiki/Key:width">width</a>, <a href="http://wiki.openstreetmap.org/wiki/Key:lanes">amount of lanes</a>, <a href="http://wiki.openstreetmap.org/wiki/Key:tunnel">tunnel</a>, etc then let software applications or personal judgment take their own decision about road dangerousness.</p>
<p>You can also search on the <a href="http://wiki.openstreetmap.org/wiki/Mailing_lists">mailing lists archives</a> (searching keywords like "bicycle=no", bike+hazard, etc) and check the different ideas/proposals around this question (for instance, <a href="http://gis.638310.n2.nabble.com/designated-bike-lane-td5883074.html">this one</a>). If you are convinced by one of the proposed solution, just use it. Even better, you can create a proposal on the wiki and call for a review on the main mailing lists (talk@ or tagging@<a href="http://openstreetmap.org">openstreetmap.org</a>)</p>
<p>Don't use "<a href="http://wiki.openstreetmap.org/wiki/Key:access">bicycle=no</a>" which is only for legal restriction.</p>

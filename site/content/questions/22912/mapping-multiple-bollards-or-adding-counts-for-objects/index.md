<p>When mapping <code>barrier=bollard</code> it is not infrequent for there to be several bollards across any particular <code>highway=*</code>. At present I have added <code>bollard:count=#</code> on a single node for a few of these, but this is more of a note for future better tagging.</p>
<p>My questions are:</p>
<ul>
<li>Has anyone got a widely used method tagging counts of bollards? (</li>
<li>Is there any generic tagging technique for adding a count for a given tagged feature?</li>
</ul>
<p><strong>Notes</strong>: (This of course applies both when mapping each individual feature is not useful (as with <code>barrier=bollard</code>), and when a count is used in lieu of more detailed mapping). A number of features often have a count, the one I'm most familiar with is <code>highway=steps</code>, <code>steps=#</code>, but there are no doubt others. I think it would be very useful to have a simple consistent way of adding counts, and I would be surprised if others have not thought so too.</p>

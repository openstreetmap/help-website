<div class="question-body">
<p>Hello,</p>
<p>I wrote an import <a href="https://wiki.openstreetmap.org/wiki/Import/Catalogue/Grotte-RAFVG">wiki</a> to document an import of cave entrances, but I have some difficulties in tagging. I also posted a <a href="https://lists.openstreetmap.org/pipermail/tagging/2021-May/061404.html">message</a> about it on tagging mailing list.</p>
<p>I need help for the following key=value.</p>
<p>General question: for cave entrance morphology (not cave morphology), with key should be used? cave_entrance:type=* ?</p>
<p>About cave_entrance:type values, dataset is in italian language and I don't have skills to translate speleo terms correctly; Below a list with frequency and my guesses:</p>
<ul>
<li>Pozzo 5814 well, pit</li>
<li>Pozzetto 1641 pit, cockpipt</li>
<li>Cunicolo 317 cuniculus</li>
<li>Caverna 263 cavern</li>
<li>Galleria 236 gallery, tunnel</li>
<li>Cavernetta 138 small_cavern</li>
<li>Fessura 124 crack, gap</li>
<li>Scivolo 92 slide</li>
<li>Baratro 34 abyss, chasm</li>
<li>Riparo sottoroccia 23 rock_shelter (a shelter with rocky ceiling)</li>
<li>Meandro 16 meander</li>
<li>Voragine 14 chasm, sinkhole</li>
</ul>
<p>Any suggestions?<br />
Better set a description:it/note tag instead and leave the job for speleologists who (hopefully) will come across?</p>

<ol>
<li><code>cave_entrance=</code> will do. <a href="https://taginfo.openstreetmap.org/keys/cave_entrance">https://taginfo.openstreetmap.org/keys/cave_entrance</a> at least there's 8 instances. Make sure you can actually enter them, as cautioned by <a href="http://gis.19327.n8.nabble.com/Caves-and-speleological-terms-tp5991361p5991365.html.">http://gis.19327.n8.nabble.com/Caves-and-speleological-terms-tp5991361p5991365.html.</a></li>
<li>Usually imports can create their own keys for specific use, like solving this problem you have here by preserving this origin Italian data first. While I have no experience in importing, within existing syntax, you may use at least <code>cave_entrance:description:it=</code> to help ensure no one overwrites your <code>description:it=</code>.</li>
</ol>

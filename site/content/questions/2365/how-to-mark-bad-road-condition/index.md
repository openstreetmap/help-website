<div class="question-body">
<p>Hello,</p>
<p>There is a road that is in main country network - it has marking and description in lots of administrative documents. It looks like it is going to be tagged with highway=secondary, ref=660. However that is completely wrong for the context, because the road condition doesn't respond to its status. Someone could expect a secondary road to be a nice, broad asphalt road. There is about 30km part of this road that is very bad - holes, land slides, asphalt missing, mud and water holes.</p>
<p>It is not a temporary condition since the road is in that shape for many years. It deserves to be marked in some way. It is very misleading for random people to go on that road after they see highway=secondary.</p>
<p>I though to tag it as highway=track, but it has a designated number and it is listed in the adminisrative papers as a secondary.</p>
<p>Should I use the key tracktype=* to describe bad roads? Is this tag only valid in combination with highway=track?</p>
<p>Any other suggestions?</p>

<div class="answer-body">
<p>Openstreetmap maps reality not administrative papers. If the road is not what one would expect from a secondary road, then don't tag it as one. If it is in such poor condition then <code>highway=track</code> <code>tracktype=grade2</code> seems resonably. Suitable <code>surface</code> tags are helpful too. Nevertheless the street should be tagged with <code>ref=660</code> as it is this road.</p>

<p>There are some additional keys for tagging road conditions such as <a href="http://wiki.openstreetmap.org/wiki/Key:smoothness">smoothness</a> and <a href="http://wiki.openstreetmap.org/wiki/Proposed_features/usability">usability</a>, but renderers and probably most routing engines don't support them. You should use them nevertheless, but as already mentioned consider tagging the road as highway=track with an appropriate <a href="http://wiki.openstreetmap.org/wiki/Tracktype">tracktype</a>.</p>

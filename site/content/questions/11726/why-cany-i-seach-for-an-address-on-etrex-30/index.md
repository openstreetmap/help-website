<p>Hello,</p>
<p>Thank to OpenSteetMap and the entire Community!! If it weren't for you Garmin would nickle and dime me to death!!!</p>
<p>I do have a question, I've downloaded the 4000MB-1on -161.15 to 158.95.2012-03-02.gmapsupp.img.torrent from <a href="http://daveh.dev.openstreetmap.org/garmin/Lambertus/2012-03-02/">http://daveh.dev.openstreetmap.org/garmin/Lambertus/2012-03-02/</a> (This is a map of the entire United States) and performed all necessary steps from unzipping it to putting it on my sd card under a file called Garmin and enabling the map on the etrex 30. The map shows up perfectly!!</p>
<p>When I go under, "Where to?" on my etrex 30 I can search All POIs, Cities, Shopping, Food &amp; Drink, Fuel Services, etc--- and all the options work fine...</p>
<p>EXCEPT for "Intersections" &amp; "Addresses". The first question the etrex 30 asks for is, "In what state or province is the intersection?" As soon as I enter the first letter of the state (such as I for Illinois or C for California) the etrex 30 responds, "No Results Found".</p>
<p>Does anyone know why this is happening? Clearly this map has POI's, Airports, Amusement Parks, etc because they show up.</p>
<p>Do I need to download a different map?</p>
<p>Is there a setting on the gps that needs to be changed?</p>
<p>Any help would be very much appreciated.</p>
<p>Thanks in advance!!</p>

<div class="answer-body">
<p>This is probably caused by using an old version of mkgmap (the tool used to generate the .img file) that has <a href="https://github.com/openstreetmap/mkgmap/commit/385d1ea815f79e49a5df5a13bc08113aa07e95f7">issues creating files bigger than 2G</a>, or forgetting to specify an <a href="http://www.mkgmap.org.uk/page/21">option to build the index</a> (thanks IrlJidel for the confirmation and links). So you should either ping the person generating those files, or get the files from someone else.</p>

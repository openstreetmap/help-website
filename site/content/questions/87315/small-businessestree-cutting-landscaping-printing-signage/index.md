<div class="answer-body">
<p>If I were mapping these I'd probably go with a generic <a href="https://wiki.openstreetmap.org/wiki/Tag:office%3Dcompany"><code>office=company</code></a> for the offices themselves with <a href="https://wiki.openstreetmap.org/wiki/Key:craft">craft</a> tags where available:</p>
<ol>
<li>there doesn't seem to be something well established for this. There appears to be <a href="https://taginfo.openstreetmap.org/search?q=tree_surgeon#values">a single</a> <code>craft=tree_surgeon</code> and <a href="https://taginfo.openstreetmap.org/search?q=arborist#values">15</a> <code>craft=arborist</code> according to taginfo so I'd probably go with the latter.</li>
<li><a href="https://wiki.openstreetmap.org/wiki/Tag%3Acraft%3Dgardener"><code>craft=gardener</code></a> with a brief <code>description</code> of scope</li>
<li><a href="https://wiki.openstreetmap.org/wiki/Tag%3Acraft%3Dgardener"><code>craft=gardener</code></a></li>
<li>no good match I'm aware of possibly <a href="https://wiki.openstreetmap.org/wiki/Tag:craft%3Dinterior_decorator"><code>craft=interior_decorator</code></a> or <a href="https://wiki.openstreetmap.org/wiki/Tag:craft%3Dpainter"><code>craft=painter</code></a></li>
<li>either <a href="https://wiki.openstreetmap.org/wiki/Tag%3Acraft%3Dprinter"><code>craft=printer</code></a> or <a href="https://wiki.openstreetmap.org/wiki/Tag%3Acraft%3Dprintmaker"><code>craft=printmaker</code></a> (if small run artistic stuff).</li>
</ol>
<p>You can of course use <a href="https://wiki.openstreetmap.org/wiki/Any_tags_you_like">any tags you like</a>, but if you invent one it is probably a good idea to run it past some people in the <a href="https://community.openstreetmap.org/c/general/tagging/70">community forums</a> in case they can spot any potential misinterpretation or conflicts with existing schemes.</p>

<div class="answer-body">
<p>Hi, You've covered a few bits with your questions, but I'll do my best;</p>
<ol>
<li>If you can view the particular object - such as from when you created or edited it, you should be able to see it's history.</li>
<li>If there is an area you regularly 'look after' it's worth getting a login with ITO - <a href="http://www.itoworld.com/">http://www.itoworld.com/</a> &amp; using one of their tools for OSM mappers. You can select a particular area, and the software will send you a message in a feed when there are any edits or updates for that area.</li>
<li>If someone has deleted data, by accident or otherwise, it is possible to get the changeset reversed - I know of it, but have never used it.</li>
<li>Vandalism can be dealt with in various ways, you just need to highlight the need.</li>
<li>It can be worth signing up for one of the e-mail circulation lists - I've learnt a lot from reading the messages in TalkGB, for instance.</li>
</ol>
<p>Hope my somewhat wordy answer is some help to you. Regards Tallguy</p>

<p>You should really improve your question by specifying if you are speaking about the "license change effect" or "data deleted by someone else".</p>
<p>Because the answer will be different :</p>
<p>For the first, the license transition withdrawn many objects because they did not comply with the new ODbl license. Those data cannot be retrieved and need a new survey (check <a href="https://wiki.openstreetmap.org/wiki/Remapping">the "remapping" task on the wiki</a>).</p>
<p>For the second, it is possible to restore the deleted objects; see the hints provided by other answers on this page.</p>

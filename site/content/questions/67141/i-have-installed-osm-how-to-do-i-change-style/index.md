<div class="answer-body">
<p>In order to change style you can use Kosmtik (<a href="https://ircama.github.io/osm-carto-tutorials/kosmtik-ubuntu-setup/">OSM carto tutorial Kosmtik</a> and <a href="https://github.com/kosmtik/kosmtik">git</a>). It is easy to use and it's great that it shows changes right away.</p>
<p>The other solution would be manual change of .mss files in <code>~/src/openstreetmap-carto</code> (most usual path). You can edit them with your favorite text editor and change every style of map. But before manually changing stuff, I would recommend that you make backup of every .mss file. Also you can use Scripts like <code>generate_road_colours.py</code> and others for easier style change. And then you need to restart renderd.</p>
<p>I usually remove all files at <code>/var/lib/mod_tile</code> -&gt; there are generated tiles in meta file extenstion. Be aware that removal of that files will remove your previous render. Command I use is <code>sudo rm -rf /var/lib/mod_tile/default</code>. You just need to check what is the name of your folder inside mod_tile (mine is default). After that you need to create XML file. Go into <code>~/src/openstreetmap-carto</code> and create XML like this <code>carto project.mml &gt; mapnik.xml</code> (Be aware that mapnik.xml can be named differently and you need to check what is your name of xml file -&gt; it could be style.xml or something else). And after all you need to start new renderd <code>sudo -u 'USERNAME' renderd -f -c /etc/renderd.conf</code> where you swap 'USERNAME' with your username. And that's it, you can refresh your map in browser and check your new style!</p>
<p>If there are errors at command <code>sudo -u 'USERNAME' renderd -f -c /etc/renderd.conf</code> like "Socket bind failed for: /var/run/renderd/renderd.sock", you need to check if you have folder <code>/var/run/renderd</code>. If you don't have it, create one and grant access to your user like this:</p>
<pre><code>sudo mkdir /var/run/renderd
sudo chown &#39;USERNAME&#39; /var/run/renderd</code></pre>
<p>(Swap 'USERNAME' with your account username).</p>
<p>Also if you get an error while renderd is generating, then probably you didn't edit .mss files correctly and you need to check them. Just take your time and read errors &amp; backup data.</p>

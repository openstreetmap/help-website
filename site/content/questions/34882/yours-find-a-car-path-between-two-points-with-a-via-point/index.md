<div class="answer-body">
<p>According to the <a href="https://wiki.openstreetmap.org/wiki/Routing/online_routers">router comparison matrix</a> YOURS supports via-points. It is also possible to specify them via the <a href="http://yournavigation.org/">webinterface</a> (which has horrible bugs). But instead of a single API call for all points the webinterface will run a call for each lat-lon pair instead and then appends them to a single route.</p>
<p>Interestingly when trying to open the permalink on the <em>Info</em> tab it will show a single URL where each via-point is specified with <code>wlat</code> and <code>wlon</code> parameters. But this URL won't work and the route will just use the start and end points, ignoring all via points.</p>
</div>

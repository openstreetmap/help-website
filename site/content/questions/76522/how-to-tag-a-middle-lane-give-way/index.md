<div class="answer-body">
<p>There is a <a href="https://wiki.openstreetmap.org/wiki/Relations/Proposed/Give_way">proposal on using relations</a> for this kind of give ways. It has never left draft status, though. Taginfo shows a <code>type=give_way</code> relation has been used only <a href="https://taginfo.openstreetmap.org/tags/type=give_way">162 times</a> so far and I'd be surprised if any router or map would recognize this construct.</p>
<p>But I'm wondering what the value of tagging this is. Is giving way to oncoming traffic not the standard applicable rule where you found this situation?</p>

<p>The proposal is vague and not very clear in parts. But as I understand it you could do it that way (if you really want to go down that road):</p>
<p>Let's assume your intersection sketch up there is oriented north. Split the north-south road at the intersection. Add the southern part with role=from, the western road with role=to and the northern part with role=give_way. Tag the relation with <code>type=give_way</code> and no other tags.</p>

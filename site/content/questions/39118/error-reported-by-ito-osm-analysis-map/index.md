<p>The ISO-OSM consistency checker reports an error here (<a href="http://www.itoworld.com/product/data/osm_analysis/map_browser?bbox=457651.675781,105760.027396,458155.294844,106068.996146&amp;layers=os_locator&amp;base_style=white,aa_2&amp;clear_map_history=true&amp;referrer=area).">http://www.itoworld.com/product/data/osm_analysis/map_browser?bbox=457651.675781,105760.027396,458155.294844,106068.996146&amp;layers=os_locator&amp;base_style=white,aa_2&amp;clear_map_history=true&amp;referrer=area).</a> Compass Point is the spur off the southern end of Elmhurst Road, and is correctly named in OSM. <li>How, if at all, can I indicate in OSM that the road names in this area are correct and that the comparison data appears to be wrong?</li>

<div class="answer-body">
<p>The dataset is OS Locator which has to be requested from:</p>
<p><a href="https://www.ordnancesurvey.co.uk/opendatadownload/products.html">https://www.ordnancesurvey.co.uk/opendatadownload/products.html</a></p>
<p>Since the data format is a bespoke text description you need bespoke tools to use it (or just inspect it manually since it is plain text).</p>
<p>The Wiki contains further detail about how one may use this data.</p>
<p><a href="https://wiki.openstreetmap.org/wiki/OS_Locator">https://wiki.openstreetmap.org/wiki/OS_Locator</a></p>

<li>If you are confident you know the correct name:</li>
</ol>
<p>not:name=Compass Point</p>
<p>you can also add notes eg:</p>
<p>not:name:note=The individual lanes, where Gosport Road joins the Quay Street Roundabout, are not individually named</p>
<p>source:name=survey</p>
<p>It is not unusual for Ordnance survey (which I assume the OS locator to be) to have a few incorrect street names, whether by error or to check for plagiarism.</p>

<p>Here I asked for a near by search for restaurants around a node or town: <a href="https://help.openstreetmap.org/questions/80912/search-for-restaurants-around-a-distance">https://help.openstreetmap.org/questions/80912/search-for-restaurants-around-a-distance</a></p>
<p>Now it would be helpful to perform the same search for an concrete address.</p>
<p>E. g. let's say all restaurants around a disctance of 1000 Meters "Platz der Republik 1, Berlin" (maybe also with zip code)</p>
<p>Would it be possible to build an address-replaceable query for that task?</p>
<p>I'm new to openstreetmap and it's hard for me to understand the overpass API</p>
<p>This wizard input text from the <a href="https://wiki.openstreetmap.org/wiki/Overpass_turbo/Wizard#Location_Filters">wiki</a> gave me the hint: <code>amenity=restaurant around "1600 Pennsylvania Ave NW, Washington"</code></p>
<p>So I got now this, successfully tested with two different addresses that I know:</p>
<pre><code>/*
This has been generated by the overpass-turbo wizard.
The original search was:
“amenity=restaurant around &quot;Bindlacher Straße 10 95448 Bayreuth&quot;”
*/
[out:json][timeout:25];
// adjust the search radius (in meters) here
{{radius=100}}
// gather results
(
  // query part for: “amenity=restaurant”
  node[&quot;amenity&quot;=&quot;restaurant&quot;](around:{{radius}},{{geocodeCoords:Platz der Republik 1 10557 Berlin}});
  way[&quot;amenity&quot;=&quot;restaurant&quot;](around:{{radius}},{{geocodeCoords:Platz der Republik 1 10557 Berlin}});
  relation[&quot;amenity&quot;=&quot;restaurant&quot;](around:{{radius}},{{geocodeCoords:Platz der Republik 1 10557 Berlin}});
);
// print results
out body;
&gt;;
out skel qt;</code></pre>
<p>But may you review that, please and tell me if it's correct for what I need? It seems to, but I'm not 100 % sure about that.</p>

<p>That looks good to me! If you want, you can shorten it using <em>nwr</em> instead of node, way and relation:</p>
<pre><code>[out:json][timeout:25];
(nwr[&quot;amenity&quot;=&quot;restaurant&quot;](around:100,{{geocodeCoords:Platz der Republik 1, 10557 Berlin}});) -&gt; .results;
.results out center;</code></pre>
<p>On <a href="https://wiki.openstreetmap.org/wiki/Overpass_turbo/Extended_Overpass_Turbo_Queries,">https://wiki.openstreetmap.org/wiki/Overpass_turbo/Extended_Overpass_Turbo_Queries,</a> the geocoding part is briefly outlined: <em>geocodeCoords</em> gives the coordinates of the centre of the first result which the search of the name resulted in. That might cause some problems if the name of the place is not well specified.</p>

<p>If you want this information &amp; know where you are but not the name of any localities, I've found this to be useful. {{center}} returns the co-ordinates of the centre of the OPT screen.</p>
<pre><code>nwr[amenity=restaurant](around:1000,{{center}});
out center;</code></pre>

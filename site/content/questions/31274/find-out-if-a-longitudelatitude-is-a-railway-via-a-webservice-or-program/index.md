<p>Please have a look at</p>
<p><a href="http://overpass-turbo.eu/s/2GN">http://overpass-turbo.eu/s/2GN</a></p>
<p>This queries for all ways that are in a small bounding box around latitude 51.451, longitude 7.01.</p>
<p>You can send these requests also directly to the API endpoint <a href="http://overpass-api.de/api/interpreter">http://overpass-api.de/api/interpreter</a> . The whole API is documented at <a href="http://wiki.osm.org/wiki/Overpass_API">http://wiki.osm.org/wiki/Overpass_API</a></p>

<p>The alternative is importing OSM data into a PostGIS database with osm2pgsql, and then running the query like</p>
<pre><code>SELECT 
    osm_id 
FROM 
    planet_osm_line 
WHERE
    railway IS NOT NULL
    AND st_dwithin(way, st_transform(st_setsrid(st_makepoint(7.01,51.451), 4326), 3857), 10);</code></pre>
<p>This will check for any ways that have a railway tag and are within 10 projection units (approximately 10 metres) of the given point and list the OSM IDs of these ways.</p>

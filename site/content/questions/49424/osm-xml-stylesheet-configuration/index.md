<div class="answer-body">
<p>I'm not really sure what your problem is. However it sounds like you're not sure how to create images/view the rendered map with the OSM XML style. You'll have to <a href="https://switch2osm.org/serving-tiles/">set up a tileserver</a> to do that. The <a href="https://switch2osm.org/serving-tiles/">switch2osm documentation on tileservers</a> will help there.</p>
<p>BTW, the <a href="https://github.com/gravitystorm/openstreetmap-carto">OSM style on openstreetmap.org</a> is developed in carto. You can use <a href="https://www.mapbox.com/tilemill/">TileMill</a> to show images on your desktop using the current OSM style. The <a href="https://github.com/gravitystorm/openstreetmap-carto/blob/master/INSTALL.md">openstreetmap-carto install instructions</a> go into more detail. It's much easier to get started with, and easier to change. That was one of the reasons for moving from XML to Carto.</p>
<p>Of course it all depends on your end game. What do you want to do? If you tell us that, we may be able to guide you more. If you just want to play around, investigate and learn, maybe start with the current OSM-carto style and TileMill.</p>

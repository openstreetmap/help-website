<p>If you are writing this app as a learning exercise, go for it. If not, consider first trying some of the existing apps, and you can use the time you save to create something new and innovative.</p>

<div class="answer-body">
<p>Firstly, you say that you're writing a "GPS tracer" and "need some help to know how to start if I need to update the map". You've probably already read the various <a href="https://wiki.openstreetmap.org/wiki/Beginners%27_Guide">Beginners' Guide</a> pages. Applications that talk to OSM directly to upload data (such as <a href="https://wiki.openstreetmap.org/wiki/API_v0.6#Uploading_Traces">uploading GPS traces</a>) might do so using <a href="https://wiki.openstreetmap.org/wiki/OAuth">Oauth</a>.</p>
<p>You then say "if I need to update the map to host my own copy to guarantee full availability". Does this mean that you want to render OSM data on some sort of map on the phone? If so there are essentially two choices - <a href="https://wiki.openstreetmap.org/w/index.php?title=Special:Search&amp;search=vector+rendering">vector rendering</a> or some sort of map tiles. If you're thinking of using tiles, have a read of the <a href="http://switch2osm.org/serving-tiles/">relevant switch2osm section</a> for more information. If vector rendering, depending on the platform there may be something that you can build upon.</p>
<p>If you think that you'll be making a lot of calls to OSM's APIs, then have a read of the <a href="https://wiki.openstreetmap.org/wiki/API_usage_policy">API usage policy</a>. If you're planning to access OSM tiles directly, read the <a href="https://wiki.openstreetmap.org/wiki/Tile_Usage_Policy">tile usage policy</a>.</p>
<p>The "<a href="http://lists.openstreetmap.org/listinfo/dev">dev</a>" mailing list might also be looking at (it's archives are searchable) and there's also <a href="https://wiki.openstreetmap.org/wiki/Irc">IRC</a> - the #osm, #osm-dev channel might be useful, and there are also many non-English-language OSM IRC channels.</p>
<p>However, I'm not entirely sure what you're asking for help about here - so apologies if I've not understood what you've asked. I'm guessing that English isn't your first language - perhaps it might help to ask in that language? Lots of people here don't speak English as a first language and may be able to help directly.</p>
